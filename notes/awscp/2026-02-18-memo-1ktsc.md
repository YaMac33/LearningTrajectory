## データキュレーション（データカタログのインベントリ整理）とは？

---

### ✅ まず一言で

**「社内にあるデータを見つけやすく・理解しやすく整理すること」**です。

---

## ① 噛み砕いた解説（超初心者向け）

会社の中には、こんなデータがあります：

* 売上データ
* 顧客データ
* アクセスログ
* 商品マスタ
* 在庫情報

これらがバラバラに保存されていると…

> 「このデータ何？」「どこにあるの？」「誰が作ったの？」

となります。

### 🎯 データキュレーションとは

図書館の司書さんのように、

* データの名前を付ける
* 説明を書く
* 分類する
* 保存場所を整理する
* 誰が使えるか管理する

という作業のことです。

---

## ② イメージ図

```
【整理されていない状態】

S3バケット
 ├─ data1.csv
 ├─ final_data_new2.csv
 ├─ test_log_aaa.csv
 └─ untitled.csv

→ 何のデータかわからない


【データキュレーション後】

データカタログ
 ├─ 売上データ（2025年版）
 │   ├─ 作成者：営業部
 │   ├─ 更新頻度：毎日
 │   └─ 保存先：S3://sales-data
 │
 └─ 顧客マスタ
     ├─ 管理者：マーケ部
     └─ 個人情報あり（要アクセス制御）
```

---

## ③ AWSでの実務イメージ

AWSでは主に：

* AWS Glue Data Catalog

を使ってデータを整理します。

企業ではこんな場面で使われます：

### 🏢 例：ECサイト企業

* S3に大量のログデータが保存されている
* データ分析チームが分析したい
* どこに何があるかわからない

👉 Data Catalogで整理
👉 分析ツール（Athenaなど）から検索可能
👉 データ活用が高速化

---

## ④ 例え話

📚 データキュレーション = 図書館のカタログ作成

* 本（データ）
* 本棚（S3などの保存場所）
* 目録カード（データカタログ）
* 司書（データエンジニア）

目録がないと、本を探せませんよね？

---

## ⑤ 試験で問われやすいポイント（CLF-C02）

### ★ 出題ポイント

✔ データを検索可能にする仕組み
✔ メタデータ管理の概念
✔ データレイクとの関係
✔ AWS Glue Data Catalogの役割

---

### 💡 試験でのよくある問題形式

「企業がS3に保存した大量データを分析したい。
データを検索可能にし、メタデータを管理するサービスは？」

→ 正解：AWS Glue Data Catalog

---

## ⑥ 間違えやすいポイント

| サービス           | 役割            |
| -------------- | ------------- |
| S3             | データ保存         |
| Glue           | データ整理・ETL     |
| Athena         | クエリ実行         |
| Lake Formation | データレイクのアクセス管理 |

👉 Data Catalogは**保存場所ではない**
👉 あくまで「目録（メタデータ管理）」です

---

## 🔥 CLF的まとめ

データキュレーションとは：

> 「データを探しやすく、理解しやすく、管理しやすく整理すること」

試験では
✔ データレイク
✔ メタデータ
✔ Glue Data Catalog

とセットで出る可能性が高いです。
