ここは 「論理分離」と「物理分離」の違い がポイントです。

対象：
Amazon Elastic Compute Cloud

⸻

🔷 まず結論

Dedicated Host は
「物理サーバー1台をまるごと専有する仕組み」

⸻

🔷 通常のEC2（共有環境）

物理サーバー
 ├─ あなたのEC2
 ├─ 他社のEC2
 ├─ 他社のEC2
 └─ 他社のEC2

👉 物理マシンは共有
👉 仮想化で分離されているだけ

⸻

🔷 Dedicated Hostの場合

物理サーバー
 ├─ あなたのEC2
 ├─ あなたのEC2
 └─ あなたのEC2

👉 他のAWS顧客は載らない
👉 物理的に分離

⸻

🔷 なぜ必要？

主な理由：
	•	ライセンス制約（Windows / Oracleなど）
	•	コンプライアンス要件
	•	規制業界
	•	物理分離が義務

⸻

🔷 問題文のポイント

「物理的に分離されたEC2インスタンス」

ここがキーワード。
	•	論理分離 → 通常EC2でOK
	•	物理分離 → Dedicated Host

⸻

🔷 Dedicated HostとDedicated Instanceの違い

🟢 Dedicated Instance

物理サーバー
 ├─ あなたのDedicated Instance
 └─ 他アカウントのDedicated Instance

👉 アカウント単位で分離
👉 物理ホストは共有される可能性あり

⸻

🔵 Dedicated Host

物理サーバー
 ├─ あなた専用

👉 ホスト完全専有

⸻

🔷 図で整理

通常EC2 → 共有
Dedicated Instance → アカウント単位分離
Dedicated Host → 物理マシン専有


⸻

🔷 コスト面

Dedicated Hostは：
	•	高額
	•	物理単位で課金
	•	長期利用前提

だから問題文に：

「物理的に分離」

とあれば、ほぼDedicated Host。

⸻

🔷 1文まとめ

Dedicated Hostは、物理サーバーを専有してEC2を実行する仕組みで、物理分離が必要な場合に使用される。
