対象：
	•	Amazon Virtual Private Cloud
	•	AWS Site-to-Site VPN

⸻

🔷 まず結論

カスタマーゲートウェイ（Customer Gateway）は
オンプレミス側のVPN装置

⸻

■ 全体構成図（最重要）

【オンプレミス側】                    【AWS側】

社内ネットワーク
        │
        │
  ┌────────────┐
  │ Customer   │  ← カスタマーゲートウェイ
  │ Gateway    │
  └────────────┘
        │
   (インターネット)
        │
  ┌────────────┐
  │ Virtual    │  ← 仮想プライベートゲートウェイ
  │ Private GW │
  └────────────┘
        │
       VPC
        │
      EC2


⸻

🔷 役割の違い

コンポーネント	どこにある？	役割
Customer Gateway	オンプレ側	VPNルーター
Virtual Private Gateway	AWS側	VPC接続口


⸻

🔷 カスタマーゲートウェイとは？

オンプレミスの：
	•	物理ルーター
	•	VPN装置
	•	ソフトウェアルーター

例：

Ciscoルーター
FortiGate
Linux VPNサーバー

👉 これがCustomer Gateway

⸻

🔷 仮想プライベートゲートウェイとは？

AWS側の接続口。

VPC
   │
Virtual Private Gateway

👉 AWS側のVPN終端

⸻

🔷 接続の流れ

社内PC
   ↓
社内ルーター（Customer Gateway）
   ↓
インターネット
   ↓
Virtual Private Gateway
   ↓
VPC内リソース


⸻

🔷 「インターネットアクセスを可能にするものではない」とは？

重要ポイント：

IGW（Internet Gateway）とは違う。

比較：

IGW → VPCとインターネットを接続
VPN → VPCと社内ネットワークを接続

VPNは「閉じた専用接続」。

⸻

🔷 図で比較

🌍 インターネット接続

VPC
  │
Internet Gateway
  │
Internet


⸻

🏢 オンプレ接続

社内ネットワーク
   │
Customer Gateway
   │
VPNトンネル
   │
Virtual Private Gateway
   │
VPC


⸻

🔷 物理 or ソフトウェアとは？

Customer Gatewayは：
	•	物理ルーターでもOK
	•	LinuxサーバーにVPN構築でもOK

AWSに登録するのは：
	•	公開IP
	•	BGP設定情報

⸻

🔷 超短い覚え方

Customer Gateway = 社内側のVPN装置
Virtual Private Gateway = AWS側のVPN装置

⸻

🔷 1文まとめ

カスタマーゲートウェイは、オンプレミス環境とVPCをVPNで接続する際のオンプレミス側の接続装置であり、インターネットアクセス用ではない。
