学習ログ：仕組みを先に作ると、勉強が続く

最近、「学ぶ内容」より先に「記録する仕組み」を作ったら、学習がかなり続くようになりました。 今日はその気づきをメモします。

やっていること

  • Googleフォームに学習メモを投げる
  • スプレッドシートに溜まったらGASで整形してHTML化する
  • GitHub Pagesに自動投稿して、あとで検索できるようにする

良かった点

  1. 心理的ハードルが下がる：ノートを開くよりフォーム送信の方が軽い
  2. 振り返りが早い：検索できるので「どこに書いたっけ？」が消える
  3. 学習の軌跡が残る：過去の自分の疑問がそのまま資産になる

注意点

仕組み作りに熱中しすぎると、肝心の学習が止まることがあります。 「仕組みは最小で動くところまで作ったら、いったん学習に戻る」のが良さそうです。

次にやりたいこと

  • トップページでツリー表示して、分野の全体像を俯瞰できるようにする
  • タグ検索をもう少し便利にする（例：タグクラウド）
  • 記事のテンプレを増やす（学習ノート / 雑記 / 手順メモ）

今日はここまで。仕組みが整ったので、明日はちゃんと勉強します。
