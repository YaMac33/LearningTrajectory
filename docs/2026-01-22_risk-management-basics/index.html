リスクマネジメントの基本

リスクマネジメント（Risk Management）とは、将来起こり得る不確実な事象を事前に洗い出し、影響を最小化するために管理・対応する活動です。目的は「リスクをゼロにする」ことではなく、「想定内に収める」ことにあります。

リスクマネジメントとは

リスクマネジメントの基本的な考え方は以下の通りです。

  • 起こるかもしれない問題（リスク）を事前に想定し、
  • 発生確率と影響度を評価し、
  • 被害を防ぐ・減らす・受け入れるための方針を決めて管理すること

リスクマネジメントの基本プロセス

ITパスポート試験でも頻出する、リスクマネジメントの4つの基本プロセスを解説します。

① リスクの特定

どんな問題が起こり得るかを洗い出す段階です。具体例として、以下のようなリスクが考えられます。

  • システム障害
  • 要員不足
  • 納期遅延
  • 情報漏えい

② リスクの分析・評価

発生確率と影響度を掛け合わせて重要度を判断します。以下のリスクマトリクスを用いることで、優先的に対応すべきリスクを可視化できます。 

影響度
  高 ┌─────────────────┐
     │                 │
     │   重点管理      │
     │   エリア        │
     │                 │
  低 └─────────────────┘
     低     →     高
          発生確率

「確率も影響も高いもの」を優先的に対応することが重要です。

③ リスクへの対応

リスクへの対応方法は主に4つに分類されます。すべてのリスクに対策する必要はありません。

対応方法 内容 例
回避 リスクの原因をなくす 危険な作業をやめる
低減 確率・影響を下げる バックアップを取る
移転 他者に任せる 保険・外注
受容 起きても許容 軽微なトラブル

④ 監視・見直し

状況変化に応じて再評価を行います。具体的には、以下のような活動が含まれます。

  • 新しいリスクの追加
  • 対策の有効性確認
  • 環境変化に応じた再評価

リスクマネジメントと危機管理の違い

リスクマネジメントと危機管理はよく混同されますが、以下のような違いがあります。

観点 リスクマネジメント 危機管理
タイミング 事前 事後
目的 予防・被害軽減 収束・復旧
例 事前のバックアップ 障害対応

IT・プロジェクトでの具体例

ITシステムにおけるリスクマネジメント

  • データ消失 → バックアップ（低減）
  • サーバ障害 → 冗長化（低減）

プロジェクトにおけるリスクマネジメント

  • メンバー離脱 → 複数人で共有（低減）
  • 仕様変更 → 変更管理ルール（低減）

ITパスポート試験での狙われ方

試験では以下のポイントが問われます。

  • 「リスクを洗い出し、評価し、対策する一連の活動」の理解
  • 回避・低減・移転・受容の区別
  • 「想定外だった」＝リスクマネジメント不十分という理解

リスクマネジメント = 起こる前に考える

実務と試験をつなぐ一言

事前にToDoを作る、失敗パターンを想定する。これも立派なリスクマネジメントです。ITパスポート試験では「事前」「評価」「対応」という3つのキーワードが出てきたら、即座にリスクマネジメントと判断してください。
