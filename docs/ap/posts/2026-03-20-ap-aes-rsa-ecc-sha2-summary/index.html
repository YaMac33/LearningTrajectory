AES, RSA, ECC, SHA-2とは？代表的な暗号アルゴリズムの役割まとめ

#セキュリティ #暗号技術 #AES #RSA #SHA-2

情報セキュリティの学習をしていると必ず登場する、アルファベットの略語たち。これらはすべて、インターネットを安全に保つための「暗号技術（アルゴリズム）」の名前です。
それぞれが「何の略か」と「何をしているか」をセットで整理することで、一気に知識が固まります。

【1行要約：役割分担】
現代の通信（TLSなど）では、AESがデータを守り、RSA/ECCが鍵や身元を守り、SHA-2が改ざんを見破る、という完璧なチームワークで動いています。

1. 4つの主要アルゴリズムの特徴

それぞれの技術が属するカテゴリー（共通鍵か、公開鍵か、ハッシュか）を意識しながら確認しましょう。

共通鍵暗号
📦
AES Advanced Encryption Standard
データをブロック単位に分け、同じ鍵を使ってぐちゃぐちゃに変換（暗号化・復号）する技術です。
  • 現代の共通鍵暗号の「世界標準」
  • 処理が非常に高速で安全
  • TLSなどでの「実際のデータ通信（大容量）」に使われる
公開鍵暗号
🔐
RSA Rivest–Shamir–Adleman
公開鍵と秘密鍵のペアを使い、「巨大な数の素因数分解の難しさ」を安全性の根拠とした技術です。
  • 開発者3人の名前の頭文字
  • 共通鍵の受け渡し（鍵交換）や電子署名に使う
  • 処理が遅いため、データ本体の暗号化にはあまり使わない
公開鍵暗号
📈
ECC Elliptic Curve Cryptography
「楕円曲線上の計算の難しさ」を利用した、RSAに代わる次世代の公開鍵暗号技術です。
  • RSAよりも短い鍵の長さで同じ強度を出せる
  • 計算負荷が低いため、スマホやIoT機器、最新のTLSでよく使われる
  • 鍵交換や電子署名に使う
ハッシュ関数
🔍
SHA-2 Secure Hash Algorithm 2
どんな長さのデータからも、固定長のデタラメな値（ハッシュ値・指紋）を生成する技術です。
  • 一方向関数であり、ハッシュ値から元のデータには戻せない（暗号化ではない）
  • 電子署名やデータの「改ざん検知」で使う
  • SHA-256などが代表的

2. 通信システムにおける一言整理

ハイブリッド暗号方式（SSL/TLS）において、これらのアルゴリズムは以下のように美しく役割を分担しています。

セキュアな通信を守る「3つの守備範囲」
📦
① データを守る（暗号化） ▶ AES
🔐
② 鍵や身元を守る（鍵交換・署名） ▶ RSA / ECC
👀
③ 改ざんを見破る（要約・検知） ▶ SHA-2

試験対策まとめ表

午前試験では、「アルゴリズム名」と「種類」の組み合わせを問う問題が頻出します。

略称 フルネーム 種類 主な役割
AES Advanced Encryption Standard 共通鍵 実際のデータ通信の暗号化（大容量・高速）
RSA Rivest–Shamir–Adleman 公開鍵 共通鍵の安全な受け渡し、電子署名（素因数分解）
ECC Elliptic Curve Cryptography 公開鍵 RSAの進化版。短い鍵で鍵交換や電子署名を行う（楕円曲線）
SHA-2 Secure Hash Algorithm 2 ハッシュ データの要約作成（メッセージダイジェスト）、改ざん検知
