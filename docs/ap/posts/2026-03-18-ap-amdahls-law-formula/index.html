アムダールの法則：2つの公式は同じ？数式アレルギー向けの計算解説

#ハードウェア #CPU #アムダールの法則 #計算問題 #応用情報

「プロセッサを複数並べて処理を並列化したら、どれくらい速くなるか？」を求めるアムダールの法則。参考書によって2つの違う公式が載っていて混乱する人が多いですが、実はこの2つは「全く同じものを違う形で書いているだけ」です。数式アレルギーの方にも直感的にわかるように解説します。

【1行要約：2つの公式の正体】
2つの公式は、全体の処理時間を「並列化できない割合(a)」で見るか、「並列化できる割合(r)」で見るかの違いだけです！
「$a = 1 - r$」の関係を使って変換すれば、完全に同じ式になります。

1. 1つのケーキを2つの視点で見る

あるプログラム全体の処理時間（100% ＝ 1）があるとします。これを、「どうしても順番にやらなきゃいけない部分（a）」と「みんなで手分けして並列にできる部分（r）」に分けます。

全体 ＝ 並列化できない部分 ＋ 並列化できる部分
できない(a)
できる(r)
① Pの式 (ボトルネック視点)
P =
n1 + (n - 1)a
並列化「できない」割合 a を使う式。
② Eの式 (並列化視点)
E =
11 - r + r/n
並列化「できる」割合 r を使う式。

2. 魔法の証明：Eの式からPの式へ変換してみよう

「本当に同じ式なの？」という疑問を解消するため、分数の変形ステップを順番に追ってみましょう。使うルールは「$a = 1 - r$（できない＝1－できる）」と「$r = 1 - a$」だけです。

「Eの式」が「Pの式」に変身するプロセス
スタート
E =
11 - r + r/n
分母を整理
=
1(1 - r) + r/n
a = 1 - r を代入
=
1a + r/n
r = 1 - a を代入
=
1a + (1 - a)/n
分母を n で通分
=
1(an + 1 - a) / n
逆数にする！
=
nan + 1 - a
分母の a をまとめる
=
n1 + a(n - 1)
完成（Pの式）
=
n1 + (n - 1)a

3. 直感的な理解：なぜ性能は「頭打ち」になるのか？

アムダールの法則が伝えたい最も重要なメッセージは、「一部でも並列化できない部分（ボトルネック）があると、プロセッサを無限に増やしても全体のスピードには限界が来る」ということです。

プロセッサ数(n)を増やした時の処理時間の変化
n = 1 (単独)
できない(a)
できる(r)
n = 2 (2分割)
できない(a)
r / 2
n = 4 (4分割)
できない(a)
r / 4
n = ∞ (無限大)
できない(a)
どんなに「できる部分(r)」を大勢で手分けして0秒に近づけても、
「できない部分(a)」の時間は絶対に削れない（これがボトルネックになる）ため、高速化には限界があります。

試験対策まとめ：どっちを覚えるべきか？

結論として、2つの式は相互変換できるので1つ覚えておけば解けますが、応用情報技術者試験ではEの式（並列化できる割合 r で考える式）をメインに覚えておくことを強くお勧めします。
