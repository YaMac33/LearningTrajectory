DNSSECとは？デジタル署名によるDNSの改ざん防止の仕組み

#セキュリティ #ネットワーク #DNS #基本情報 #応用情報

インターネットの名前解決を担うDNSですが、元々の仕組みには「その返事が本当に正しいものか確認できない」という弱点がありました。これを解決するためのセキュリティ拡張が「DNSSEC（DNS Security Extensions）」です。

【試験でのキーワード】
「DNSの応答にディジタル署名を付加することで、応答レコードの正当性と完全性を検証可能にする技術はどれか？」

➡ 答え：DNSSEC

1. なぜDNSSECが必要なのか？（通常のDNSの弱点）

通常のDNSは、問い合わせに対して送られてきたIPアドレスを「無条件で信じて」しまいます。
もし攻撃者が、本物のDNSサーバより先に「嘘のIPアドレス（偽サイトの住所）」を返してくると、利用者は気づかずに偽サイトへ誘導されてしまいます。（＝DNSキャッシュポイズニング攻撃）

通常のDNSの弱点（キャッシュポイズニングの危機）

← 応答待ち ─
本物の
DNSサーバ

攻撃者
「偽のIPだよ！」

利用者は「その応答が本物か、偽物か」を区別する手段を持っていません。

2. DNSSECの仕組み（デジタル署名と検証）

この問題を解決するため、DNSSECでは公開鍵暗号方式を用いた「デジタル署名」の仕組みをDNSに組み込みます。

受け取った署名を、DNS上に公開されている公開鍵を使って検証します。これで「本当に管理者が作ったデータか（正当性）」と「途中で書き換えられていないか（完全性）」を証明できます。

3. 試験で狙われる「引っかけ」ポイント

DNSSECは「改ざん防止」には非常に強力ですが、万能ではありません。

⚠ DNSSECが「やらないこと」

DNSSECは、あくまで「署名」による改ざん検知（完全性・正当性の保証）を行う仕組みです。
そのため、通信内容自体の「暗号化（秘匿性）」は行いません。
（通信内容を盗み見られることや、DDoS攻撃などを防ぐ機能はありません。）

まとめ表

項目 DNSSECの特徴
目的 DNSの応答データの改ざん防止・正当性の確認
防げる攻撃 DNSキャッシュポイズニング、なりすまし応答
使う技術 公開鍵暗号方式によるデジタル署名

検証用の公開鍵： DNSKEY レコード
防げないもの 通信の盗聴（暗号化はしない）、DDoS攻撃

🎯 試験対策の合言葉
「DNSSECは、デジタル署名で『改ざん（ポイズニング）』を防ぐ！でも『暗号化』はしない！」
