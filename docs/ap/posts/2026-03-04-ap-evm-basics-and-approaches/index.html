EVM（アーンド・バリュー・マネジメント）の基礎と問題の解き方

#プロジェクトマネジメント #EVM #基本情報 #応用情報

プロジェクトの「進捗」と「コスト」を同時に管理する手法、EVM（Earned Value Management）。試験で頻出の計算問題や、状況分析問題の解き方のプロセスを整理します。

【今回の分析対象となる状況】
あるプロジェクトにおいて、各指標が以下のようになっている。
CPI = 0.9、 SPI = 1.1、 TCPI = 1.2
この状況から読み取れるプロジェクトの現状と、適切な対策はどれか？

1. まず基礎用語（3つの絶対的指標）

EVMの最大の特徴は、従来の「予定」と「実績（コスト）」だけでなく、「成果（出来高）」を金額換算して同時に評価する点にあります。

PV・EV・ACの関係イメージ
PV
予定通りならここまで終わっているはず
Planned Value
計画価値 (予算)
EV
実際にはここまで完成している
Earned Value
出来高 (成果)
AC
でも、これだけお金を使ってしまった
Actual Cost
実コスト

※ 最終的な総予算のことを BAC (Budget At Completion) と呼びます。

2. 効率指標の読み取り（CPI・SPI・TCPI）

先ほどの3つの値を使って、「コスト効率」と「進捗効率」を計算します。「1.0」を基準に考えるのが鉄則です。

プロジェクト状況ダッシュボード
CPI (コスト効率) EV ÷ AC
0.9
コスト超過
1.0未満 ＝ 悪い
「100円の成果を出すのに、111円使ってしまっている状態」
SPI (進捗効率) EV ÷ PV
1.1
予定より早い
1.0より大 ＝ 良い
「予定より早く作業が進んでいる状態」
TCPI (残作業の必要効率) (BAC-EV) ÷ (BAC-AC)
1.2
効率改善が必要
1.0より大 ＝ 悪い
「総予算内に収めるには、残りの作業を今の1.2倍の効率でやらないとダメ」

3. 状況の分析と適切な「対策」

ダッシュボードから、このプロジェクトは「作業は早いが、お金を使いすぎている（スピード優先でコストが膨らんでいる）」状態であることが分かります。
さらにTCPIが1.2ということは、このままでは予算（BAC）をオーバーするため、残りの作業はかなりの節約（効率化）が必要です。

この状況でとるべき適切な対策
⚙️ コスト効率の向上策 無駄な残業の削減、リソースの最適化、作業工程の見直しなどを行い、コストを抑える。
＋
💰 予備費の検討 効率化だけではカバーできない場合、コンティンジェンシー予備費（想定リスク用の予備予算）の投入を検討する。

思考プロセスのまとめ（試験での解き方）

手順 確認する指標 今回の判断
1 CPI (コスト) が1より大きいか小さいか？ CPI < 1 なので「コスト超過（悪い）」
2 SPI (進捗) が1より大きいか小さいか？ SPI > 1 なので「進捗は早い（良い）」
3 TCPI (残作業) が1より大きいか小さいか？ TCPI > 1 なので「効率改善が必要（ピンチ）」
結論 進捗は良いが、コストがピンチ。したがって「コスト効率の改善」と「予備費の検討」が正解。
