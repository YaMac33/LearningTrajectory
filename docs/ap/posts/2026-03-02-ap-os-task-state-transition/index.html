「休止・終了・待ち」とプリエンプションの違い（タスク状態遷移）

#OS #タスク管理 #スケジューリング #基本情報 #応用情報

OSのタスク状態遷移において、「実行中」のタスクが別の状態に移るケースはいくつかあります。「CPUを一時的に奪われただけ（プリエンプション）」という基本を軸に、他の3つの状態（休止・終了・待ち）との違いを明確に比較します。

1. タスクの「休止（Suspend）」とは？

休止状態とは、システム管理者やユーザの操作によって「明示的に止められた」状態です。

プリエンプション vs 休止（Suspend）
🔁 プリエンプション
実行中 ➡ 実行可能(Ready)
CPUを奪われただけ
  • 順番待ちの列へ戻る
  • 自分の順番が来れば自動で再開
VS
⏸️ 休止 (Suspend)
実行中 ➡ 休止
明示的に停止された
  • スケジューラの対象外になる
  • 「再開操作」があるまで自動では戻らない
状態特徴
プリエンプションCPUを一時的に奪われる
休止明示的に停止・再開待ち
👉 ポイント 休止は「止められて待つ状態」であり、自動では再開しない。

2. 終了状態（Terminate）とは？

終了状態とは、タスクがすべての処理を終えた、またはエラーで異常終了した状態です。

プリエンプション vs 終了（Terminate）
🔁 プリエンプション
実行中 ➡ 実行可能(Ready)
CPUを奪われただけ
  • タスクとしては生きている
  • 後で続きの処理を行う
VS
🏁 終了 (Terminate)
実行中 ➡ 終了
処理完了・消滅
  • 処理が完全に終わった状態
  • タスクはメモリから消滅する
状態意味
プリエンプションCPU使用権の移動
終了状態タスク消滅
👉 ポイント プリエンプションは“順番待ち”、終了は“処理完了”。

3. 待ち状態（Blocked / Wait）とは？

待ち状態とは、キーボードの入力やディスクからのデータ読み込み（I/O）など、何かしらの条件が揃うのを待っている状態です。

プリエンプション vs 待ち（Blocked）
🔁 プリエンプション
実行中 ➡ 実行可能(Ready)
CPUを奪われただけ
  • 処理を進める準備はできている
  • 実行は可能（CPUの空き待ち）
VS
⏳ 待ち (Blocked)
実行中 ➡ 待ち
I/Oやイベント待ち
  • 条件が満たされるまで実行できない
  • 条件成立で「Ready」に戻る
状態意味
待ち状態I/Oやイベント待ち
プリエンプションCPUの一時的な剥奪
👉 ポイント 待ちは「条件待ち」、プリエンプションは「順番待ち」。

まとめ

「実行中」からどの状態へ遷移するかは、「何が原因でCPUを手放したのか」によって決まります。これらを明確に区別できるようにしておきましょう。
