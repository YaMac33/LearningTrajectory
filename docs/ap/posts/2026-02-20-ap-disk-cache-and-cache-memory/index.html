ディスクキャッシュとキャッシュメモリ

応用情報技術者試験 ハードウェア アーキテクチャ

1. 定義と仕組み（2つのキャッシュの違い）

コンピュータを高速に動作させるためには、CPUがデータを待ち時間なく受け取れることが理想です。しかし、各装置の処理速度には大きな差があります。そこで、よく使うデータを「一時保存用の高速な場所」に置いておく技術がキャッシュ（Cache）です。

キャッシュには、配置される場所と目的によって「キャッシュメモリ」と「ディスクキャッシュ」の2種類が存在します。

① キャッシュメモリ（CPUと主記憶の間）

CPUは非常に高速ですが、主記憶（RAM）はそれに比べて遅いため、CPUがデータ待ちになってしまいます。これを解消するため、CPUのすぐ近くに配置され、主記憶のうちよく使うデータの一部をコピーしておくのがキャッシュメモリです。

② ディスクキャッシュ（主記憶とディスクの間）

主記憶（RAM）と磁気ディスク（HDD等）の間には、さらに絶望的な速度差があります。HDDは物理的に円盤を回転させてデータを読み書きするため非常に遅いのです。そこで、ディスクへのアクセス回数を減らすため、HDD側に一時保存用の半導体メモリ（データバッファ）を設けたものがディスクキャッシュです。

2. 応用情報技術者試験（午前）での問われ方

午前試験では、この2つを明確に区別できているかを問う問題が頻出します。「どの装置とどの装置の速度差を埋めるのか」を確実におさえましょう。

比較項目 キャッシュメモリ ディスクキャッシュ
どこを高速化？ CPU ⇄ 主記憶 主記憶 ⇄ ディスク
何をコピーする？ 主記憶データの一部 ディスクデータの一部
主な実装場所 CPU内部、またはCPU周辺 ディスク装置側（コントローラ内など）
試験でのキーワード 「実効アクセス時間」「ヒット率」 「バッファ」「回転待ち時間の隠蔽」

【ひっかけ対策のコツ】
問題文に「CPUと主記憶のアクセス速度の差」とあればキャッシュメモリ、「主記憶と磁気ディスクのアクセス速度の差」とあればディスクキャッシュを選びます。「上の層を速くするのがキャッシュメモリ、下の層を速くするのがディスクキャッシュ」と覚えましょう。

3. 午後問題対策（記述問題の視点）

午後問題（コンピュータシステム分野）では、「実効アクセス時間」の計算や、「なぜキャッシュが有効に働くのか（局所性）」といったより深い理解が問われます。

【想定設問】

キャッシュメモリを導入することでシステム全体の処理速度が向上する理由として、プログラムの実行がもつ「局所性（Locality）」という性質が関係している。この「局所性」とはどのような性質か、時間的・空間的という観点を含めて40字以内で述べよ。

【解答例】

一度アクセスしたデータやその周辺のデータに、近い将来再びアクセスしやすい性質。（39字）

【ポイント抽出の思考プロセス】
1. 局所性とは： 特定のデータばかり偏って使われる性質のこと。
2. 時間的局所性： 「今使ったデータは、すぐ後にもう一度使われる可能性が高い」（例：ループ処理の変数）。
3. 空間的局所性： 「今使ったデータのアドレスの、すぐ近くのアドレスが使われる可能性が高い」（例：配列の連続アクセス）。
これらがあるからこそ、少容量のキャッシュでも「ヒット率」が高くなり、全体として高速化できるという論理を理解しておきましょう。

4. 実務イメージとシステム構成

ITエンジニアの実務において、「キャッシュ」という概念はハードウェアにとどまらず、ソフトウェアアーキテクチャ全体に適用される非常に重要な考え方です。

「遅いストレージの前に、速い一時保存場所を置く」という基本原理は、Webシステムなどでもそのまま応用されています。

Webシステムにおけるキャッシュの応用例

1. メモリキャッシュ (Redis / Memcached)

RDBMS（データベース）はHDDやSSDにアクセスするため遅延が発生します。そこで、よく検索されるユーザー情報などを「インメモリDB」にキャッシュし、高速にレスポンスを返します。（ディスクキャッシュの概念の応用）

2. CDN (コンテンツ配信ネットワーク)

遠くにあるオリジンサーバー（主記憶のようなもの）から毎回画像をダウンロードするのは遅いため、ユーザーの物理的に近いエッジサーバー（キャッシュメモリのようなもの）に画像を一時保存しておきます。

キャッシュは速度を劇的に改善しますが、同時に「元のデータが更新されたとき、いつキャッシュを破棄・更新するか（キャッシュの無効化・整合性維持）」という運用上の難しい課題も引き起こします。試験だけでなく、実務でもこのトレードオフを意識して設計を行うことが求められます。
