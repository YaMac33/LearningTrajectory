アムダールの法則とは？並列処理による性能向上の限界

#Amdahl's Law #性能評価 #並列処理 #計算理論 #応用情報

「CPUの数を10倍にすれば、処理速度も10倍になるはずだ！」…実は、そんなに上手くはいきません。並列処理における性能向上の限界（頭打ち）を示す計算理論、「アムダールの法則（Amdahl's Law）」を解説します。

【法則が伝えたい核心】
プログラムの中には「どうしても並列化できない部分（順番にしか処理できない部分）」が存在するため、いくらCPUを増やしても、その部分がボトルネックとなって全体のスピードアップは限界を迎える。

1. アムダールの法則の公式

CPU1台の時の性能を「1」としたとき、CPUを「n台」に増やしたときの性能向上率（スピードアップ）は、以下の式で表されます。

P =
n 1 + (n - 1)a
P ： スピードアップ（CPU1台に対する性能比）
n ： 使用するCPUの数
a ： プログラムのうち「並列化できない部分」の割合
      (シリアル部分、0〜1の小数で表す)

2. なぜこの式になるのか？（図解で直感的に理解）

プログラム全体の処理を「並列化できない部分（割合：a）」と「並列化できる部分（割合：1－a）」に分けて考えます。

CPU増加による「実行時間」の短縮イメージ
CPU 1台
(時間 = 1)
並列不可(a)
並列可能(1 - a)
⬇ CPUを 4台 (n=4) に増やすと...
CPU 4台
(時間はどうなる？)
そのまま(a)
(1 - a) / n
←この分だけ時間が短縮！

青い部分（並列可能）の時間は「1/n」に縮まりますが、
赤い部分（並列不可）の時間はCPUを増やしても短縮されません。

公式の導出ステップ

上の図の通り、n台のCPUを使ったときの「全体の実行時間 T(n)」は以下になります。
T(n) = a + (1 - a) / n

性能（スピードアップ P）は、「1台の時の時間(1) ÷ n台の時の時間(T(n))」なので、
P = 1 / { a + (1 - a) / n }
この式の分母と分子に「n」を掛けて整理すると、先ほどの公式 P = n / { 1 + (n - 1)a } になります。

3. 衝撃の計算例：CPUを10台にしても…

実際に数字を当てはめて、並列処理の限界を実感してみましょう。

【条件】並列化できない割合 (a) = 0.1 (10%)
CPU数 (n) = 10台
P = 10 / { 1 + (10 - 1) × 0.1 }
P = 10 / { 1 + 9 × 0.1 }
P = 10 / 1.9
約 5.26 倍！

プログラムのたった10%が並列化できないだけで、
CPUを10倍に増やしても、速度は「約5.3倍」で頭打ちになってしまいます。

⚠ 極限状態（無限大）の考え方

もし、CPUの数 (n) を「無限大(∞)」まで増やしたとしたらどうなるでしょうか？
並列可能部分の処理時間は限りなくゼロに近づきますが、並列不可部分(a)は絶対に残ります。したがって、最大性能向上率は「 1 / a 」が限界となります。（例：a=0.1なら、何台CPUを積んでも最大10倍までしか速くならない）

試験対策まとめ

キーワード 意味・試験での出題ポイント
アムダールの法則 複数プロセッサによる並列処理の性能向上の限界を示す法則。
並列化できない部分 (a) 全体の処理時間を決めるボトルネック。この割合が大きいほど、CPUを増やしても効果が薄い。
計算問題のコツ 公式を忘れても、「時間 = a + (1-a)/n」という図のイメージを思い出せば、その逆数で計算できます。
