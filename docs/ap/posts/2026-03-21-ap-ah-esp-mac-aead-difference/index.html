AH/ESPとMAC/AEADの違い：IPsecとTLSの暗号化・改ざん検知まとめ

#セキュリティ #IPsec #TLS #暗号化 #改ざん検知

ネットワーク通信を安全にするための代表的な2大プロトコル「IPsec」と「SSL/TLS」。それぞれの世界の中で、データを守る（暗号化）ことと、書き換えを見破る（改ざん検知）ことを担当している4つの専門用語があります。
アルファベットばかりで混同しやすい「AH・ESP・MAC・AEAD」を、「どのプロトコルの話か？」「役割は何か？」の2軸でスッキリ整理しましょう。

【1行要約：役割の完全一致】
・AH と MAC は、どちらも「改ざん検知だけ」を行う旧方式。
・ESP と AEAD は、どちらも「暗号化＋改ざん検知」を同時に行う現代の主流方式です！

1. まずは「所属する世界（階層）」を分ける

そもそも、これらは守っている「レイヤー（層）」が違います。
IPsec（L3：ネットワーク層）の仲間か、SSL/TLS（L4以上：トランスポート層〜アプリケーション層）の仲間かで、最初に箱を2つに分けましょう。

プロトコルスタックごとの所属グループ
IPsec
ネットワーク層 (L3)
「パケット全体」を守る
AH 改ざん検知
ESP 暗号化 ＋ 改ざん検知
SSL / TLS
トランスポート層〜 (L4+)
「アプリのデータ」を守る
MAC 改ざん検知
AEAD 暗号化 ＋ 改ざん検知

2. 役割の「マトリックス」で対比する

それぞれのグループの中に、「改ざん検知だけ」の機能を持つものと、「暗号化もセット」でやってくれる高機能なものが存在します。この関係性をマトリックスで覚えるのが最強の試験対策です。

機能と所属のマトリックス
👀 改ざん検知のみ
📦 暗号化 ＋ 改ざん検知
IPsec
(L3)
旧方式 AH Authentication Header
  • データは暗号化されない（平文）
  • IPヘッダも保護の対象
  • ※NAT（IP変換）と相性が悪いため、現在はほぼ使われない
現代の主流 ESP Encapsulating Security Payload
  • データを暗号化し、さらに認証（改ざん検知）も行う
  • VPN構築の主役（IPsecの本体）
SSL/TLS
(L4+)
旧方式 MAC Message Authentication Code
  • データと鍵を混ぜて要約（ハッシュ）を作り、改ざんを検知
  • 暗号化は別の処理（AESなど）で行う
  • TLS 1.2までの方式
現代の主流 AEAD Authenticated Encryption with Associated Data
  • 暗号化と認証（改ざん検知）を同時に実行する
  • 非常に高速で安全性が高い
  • TLS 1.3の標準方式

3. IPsecとTLSの「守備範囲」の違い

そもそもなぜ、似たような機能がIPsecとTLSの両方にあるのでしょうか？それは、守る対象の「規模」が違うからです。

それぞれのプロトコルが守る範囲
🏢
IPsec ＝ ネットワーク全体（拠点間）を守る 東京本社と大阪支社の「ルーター間」を丸ごと暗号化する（VPN）ためなどに使われます。通るデータがWeb通信でもメールでも、問答無用で全てパケット単位で暗号化します。
📱
TLS ＝ アプリケーション通信（個別の糸電話）を守る あなたのスマホの「Webブラウザ」とAmazonの「Webサーバー」の間など、特定のアプリケーション間の通信だけを暗号化します。

試験対策：一言で完全整理！

午前試験の選択肢で迷わないよう、それぞれの特徴を最短の言葉で頭に焼き付けましょう。

略語 所属 試験で選ぶべきキーワード・特徴
AH IPsec (L3) 「改ざん検知のみ（暗号化しない）」。NATとの相性が悪い。
ESP IPsec (L3) 「暗号化 ＋ 改ざん検知」。IPsecでデータを保護する本体。
MAC TLS (L4+) 「改ざん検知（メッセージ認証コード）」。暗号化とは別に行う旧方式。
AEAD TLS (L4+) 「暗号化 ＋ 認証（改ざん検知）の一体化」。TLS 1.3の標準となる現代の主流。

🎯 知識の繋がり
「AH と MAC は古い（機能が少ない）」「ESP と AEAD は新しい（全部入り）」とペアで結びつけると、どんな問題が来ても一瞬で対応できます！
