CISCとRISCの違いとは？万能ナイフと専用包丁でわかるCPU設計思想

#ハードウェア #CPU #CISC #RISC #基本情報

パソコンやスマホの頭脳である「CPU」。プログラムの命令をどうやって処理させるかという設計者たちの哲学の違いにより、世界は大きく2つの陣営に分かれています。
前回学習した「FDOE（CPUの料理人）」のイメージを持ったまま、この2大アーキテクチャの違いを見てみましょう。

【1行要約】
ズバリ、CPUの設計思想における「なんでもできる超・多機能な万能ナイフ（CISC）」と、「単純な作業を爆速でこなす、プロの専用包丁セット（RISC）」の対決です！

1. 料理の「レシピ」と「道具」で理解する

CPUが命令をこなす仕組みを、料理人が「カレーを作る」プロセスに例えて比較します。

CPU設計の2大哲学
CISC (シスク)
Complex Instruction Set Computer
🍲✨ 全自動の高級調理家電
  • 「複雑な命令」もCPU側で全部やってあげる。
  • 「カレーを作る」という1つの命令（ボタン）だけで、最後まで自動で仕上げてくれる。
  • 命令が1つで済むため、プログラムの量（レシピ）は短くなる。
  • その代わり、CPUの内部構造はめちゃくちゃ複雑になる。
複雑な命令 可変長命令 Intel Core i系
RISC (リスク)
Reduced Instruction Set Computer
🔪👨‍🍳 手際が異常に良い職人
  • 「単純な命令（Reduced）」だけを持ち、爆速で動く。
  • 全自動ボタンはない。「肉を切る」「炒める」「煮る」という基本命令を組み合わせて作る。
  • 細かく指示を出すため、プログラムの量（レシピ）は長くなる。
  • その代わり、CPUの内部構造はシンプルに作れる。
単純な命令 固定長命令 ARM / Apple Mチップ

2. なぜ「RISC」はパイプライン処理と相性が良いのか？

前回学習した、工場レーンのように命令を並行してこなす「パイプライン処理」。RISCがこの処理で無類の強さを発揮する理由は、「命令の長さ（サイズ）」にあります。

「F・D・O・E」の4拍子のリズムが綺麗に揃うため、パイプライン処理との相性が抜群に良い！
💻 実務イメージ：現代の覇者は「RISC」！
昔は「メモリの容量が少なかったので、プログラム（レシピ）を短く書けるCISC」が有利でした。
しかし現在では、メモリが大容量化したことと、「シンプルで消費電力が少なく、パイプラインで超高速化できるRISC」のメリットが圧倒的に勝るようになり、iPhoneなどのスマホ市場を完全に制覇しました。さらにはパソコン市場（AppleのMacなど）でも、RISCベースのチップが旧来のCISCを圧倒し始めています。
