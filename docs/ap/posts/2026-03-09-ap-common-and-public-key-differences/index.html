共通鍵と公開鍵の違い：暗号方式と必要な鍵の数

#セキュリティ #暗号技術 #公開鍵暗号 #基本情報 #応用情報

インターネット上で安全にデータをやり取りするための暗号化技術。その中核をなす「共通鍵暗号方式」と「公開鍵暗号方式」について、「4人が通信する場合」の図解を用いて違いを徹底比較します。

【試験で最も重要な公式】
「n人が互いに暗号通信を行う場合、システム全体で必要な鍵の数はいくつか？」
➡ 共通鍵なら n(n-1)/2 個
➡ 公開鍵なら 2n 個

1. 図解で見る「鍵の持ち方」の違い

A、B、C、Dの4人がお互いに秘密の通信をする場合、2つの方式で鍵の作り方や数が大きく異なります。

  • 鍵の受け渡しが安全で、管理が容易（100人でも200個）
  • 複雑な数学を使うため、計算処理が遅い（重い）

2. 通信のイメージ（AさんからBさんへ送る場合）

公開鍵暗号方式でAさんがBさんに機密のメッセージを送る場合、以下のように鍵を使います。

  1. Aさんは、世界中に公開されている「Bさんの公開鍵」を使ってメッセージを暗号化する。
  2. 暗号化されたメッセージがBさんに届く。
  3. Bさんは、自分しか持っていない「Bさんの秘密鍵」を使って復号し、メッセージを読む。

※Bさんの公開鍵でロックした箱は、Bさんの秘密鍵でしか開けられないという関係性がミソです。

3. 実務での活用：ハイブリッド方式（SSL/TLS）

実際のインターネット（WebサイトのHTTPS通信など）では、「共通鍵の速さ」と「公開鍵の安全な鍵交換」の**いいとこ取り（ハイブリッド方式）**が使われています。

ハイブリッド方式（SSL/TLSの基本）のフロー
🤝 ① 鍵の受け渡し 安全な「公開鍵暗号」を使って、今回だけ使う「共通鍵」を相手に送る。
➡
⚡ ② 実際のデータ通信 無事に共有できた「共通鍵」を使って、大容量のデータを高速に暗号化通信する。
