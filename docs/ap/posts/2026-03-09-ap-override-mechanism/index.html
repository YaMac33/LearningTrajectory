オーバーライドとは？親クラスの操作を子クラスで上書きする仕組み

#オブジェクト指向 #プログラミング #基本情報 #応用情報

オブジェクト指向プログラミングにおいて、コードの再利用性を高めるための非常に強力な機能「オーバーライド」。親の技を子でアレンジする、その仕組みと試験での頻出ポイントを解説します。

【1行要約】
オーバーライド（Override）とは、親クラスから受け継いだ「操作（メソッド）」の名前はそのままに、子クラス専用の動きに「上書き（再定義）」することです。

1. オーバーライドの仕組み（親の技をアレンジ！）

動物の鳴き声を例にして考えてみましょう。
親である「動物クラス」には「鳴く()」という操作が定義されています。子である「犬クラス」や「猫クラス」はこれを引き継ぎますが、そのままではつまらないので、それぞれ自分専用の鳴き声に中身を上書き（オーバーライド）します。

クラスの継承とメソッドの「再定義」
スーパークラス (親)
🐶 動物クラス
+ 鳴く()
「......」(基本は音なし)
サブクラス (子)
🐕 犬クラス
+ 鳴く() 上書き
「ワンワン！」と出力
サブクラス (子)
🐈 猫クラス
+ 鳴く() 上書き
「ニャー！」と出力

相手が犬でも猫でも、システム側は「鳴く()を実行しろ！」と一言指示するだけで、
それぞれのクラスが自分に合った正しい鳴き声を出してくれます。（＝多相性）

2. 実務イメージ：給与計算システム

実務ではどのように使われるのでしょうか。
例えば、親クラスである「社員」に 給与を計算する() という操作を持たせます。子クラスとして「営業職」や「アルバイト」を作ったとき、営業職なら「基本給＋歩合給」の計算式に、アルバイトなら「時給×時間」の計算式に、それぞれ 給与を計算する() の中身をオーバーライドします。
これにより、雇用形態が増えても呼び出し側のプログラムを変更する必要がなくなります。

3. 【超重要】オーバーロードとの違い（引っかけ対策）

情報処理試験で100%引っかけに使われるのが、名前が似ている「オーバーロード」です。この2つは全くの別物なので、セットで違いを暗記しましょう。

📝 オーバーライド (Override)
関係：親と子（継承）
class 親 {
  攻撃する() { パンチ }
}
class 子 {
  攻撃する() { 炎を吐く } // 上書き
}
意味：上に乗って書き潰す（上書き）。
特徴：名前も、引数（渡すデータ）も全く同じにして、中身の処理だけを書き換えます。
📚 オーバーロード (Overload)
関係：同じクラスの中
class 勇者 {
  攻撃する(剣) { 斬る }
  攻撃する(弓) { 射る } // 別物
}
意味：積みすぎ、多重定義。
特徴：同じ名前の操作を複数作りますが、引数（渡すデータの数や種類）を必ず変えます。渡された道具によって動きを変えるための仕組みです。

試験対策まとめ：このキーワードを探せ！

用語 発生する場所 試験問題で反応すべきキーワード
オーバーライド 親クラスと
子クラス間		 「サブクラスで」、スーパークラスの操作を「再定義する」「変更する」、多相性（ポリモーフィズム）の実現。
オーバーロード 同じクラス内 同じ名前で、「引数の型や数が異なる」メソッドを「複数定義する」。

🎯 覚え方の合言葉
「子クラスで上書きする（Ride）！」「同じクラスで引数を変えていっぱい積む（Load）！」
