クイックソート

応用情報技術者試験 アルゴリズム ソート

1. クイックソートとは？（定義と仕組み）

クイックソートは、「代表（基準値：ピボット）を1人決めて、その人より小さい人と大きい人に分ける」という操作を、グループが1人になるまで繰り返す並べ替え（ソート）アルゴリズムです。このように、大きな問題を小さな問題に分割しながら解決していく手法を分割統治法と呼びます。

試験では「配列の左端を基準値とする」「グループ内の元の順番は変えない（安定かどうかは実装によるが、設問の条件に必ず従う）」といった指定ルールが出題の鍵となります。

アルゴリズムのトレース（左端基準の例）

配列：[2, 3, 5, 4, 1] を昇順に並べ替える

🔷 1回目の分割
初期状態: [ 2, 3, 5, 4, 1 ] ← 基準値(2)は左端
2より小さい → [1]
2より大きい → [3, 5, 4] （※元の順番を維持）
結果: [ 1 ] [ 2 ] [ 3, 5, 4 ]
※この時点では「1, 2, 3, 5, 4」となり、まだ全体は整列されていません。
🔷 2回目の分割
要素が1つの [1], [2] は分割終了。
残りの [ 3, 5, 4 ] を分割します。
対象: [ 3, 5, 4 ] ← 基準値(3)は左端
3より小さい → なし
3より大きい → [5, 4]
全体結果: [ 1 ] [ 2 ] [ 3 ] [ 5, 4 ]

このように、基準値が確定するたびに配列が小さな集団に分かれていき、最終的にすべての要素が1つになった時点で整列が完了します。

2. 応用情報技術者試験（午前）での問われ方

午前試験のソート問題では、アルゴリズムの定義や「途中の状態」を問うトレース問題が頻出です。以下の手順（解くときの型）を意識しましょう。

  1. 基準値を確認する： 左端か？ 右端か？ 中央か？
  2. グループ内の順番ルールを確認する： 基本は「元の配列と同じ順番を維持する」条件を見落とさないこと。
  3. 途中状態か最終状態かを確認する： 「N回目の分割が終わった直後の配列はどれか」という引っ掛けに注意。

混同しやすいソートアルゴリズム

午前試験では、他のソートアルゴリズムと特徴をすり替えた選択肢が出題されます。明確に区別できるようにしておきましょう。

アルゴリズム 特徴（試験でのキーワード） 平均計算量
クイックソート 基準値を選び、それより小さい/大きいグループに分割する。（分割統治法） $O(N \log N)$
バブルソート 隣り合う要素を比較し、逆順であれば交換する。（水面へ浮かぶ泡のよう） $O(N^2)$
選択ソート 未整列の部分から最小値（または最大値）を探して先頭へ移動させる。 $O(N^2)$
マージソート 配列を半分に分割し、それぞれ整列させた後、併合（マージ）する。 $O(N \log N)$

3. 午後問題対策（記述問題の視点）

午後問題（特にアルゴリズムやプログラミング分野）では、クイックソートの「弱点（最悪ケース）」について深く問われることがあります。計算量や再帰処理の仕組みを理解しているかが試されます。

【想定設問】

常に「配列の左端」を基準値として選択するクイックソートにおいて、既に昇順に整列されている配列を処理対象とした場合、平均的な処理時間とならず、最悪の計算量 $O(N^2)$ となってしまう。その理由を、分割のされ方に着目して40字以内で述べよ。

【解答例】

基準値が常に最小値となり、データが偏って分割されず再帰の深さが最大になるから。（39字）

【ポイント抽出の思考プロセス】
1. 前提の確認： 昇順の配列 `[1, 2, 3, 4, 5]` で左端を基準値にすると、基準値は常にそのグループの「最小値」になります。
2. 生じる現象： 「基準値より小さいグループ（0個）」と「大きいグループ（残り全部）」に分かれてしまい、要素が半分になりません。
3. 結果（なぜ遅いか）： $N$個のデータを分割する回数（再帰処理の深さ）が $\log N$ 回ではなく、$N$ 回になってしまい、処理効率がバブルソート等と同じ $O(N^2)$ に落ちてしまいます。
※実務ではこの問題を回避するため、左端・右端・中央の値の「中間値」を基準値に選ぶ工夫（Median-of-Three）が取られます。

4. 実務イメージとシステム構成

ITエンジニアの実務において、クイックソートの概念はどこで活かされているのでしょうか。

クイックソートは、その名の通り平均的に最も速い（$O(N \log N)$）ソートアルゴリズムの一つであり、メモリ使用量も少ない（インプレースな並べ替えが可能）という強力な特徴を持っています。

実務での利用シーン

1. プログラミング言語の標準関数

多くの言語の sort() 関数（C言語の qsort など）の内部実装のベースとして使われています。（※現在はクイックソートの弱点を補った「イントロソート」が多く採用されています）

2. データベースの内部ソート

RDBMSで ORDER BY 句を実行した際、インデックスが使えない場面（巨大なメモリ上での整列作業）において、高速に処理を完了させるために利用されます。
