RISCの5段階パイプラインとハザードの仕組み

#ハードウェア #CPU #アーキテクチャ #基本情報 #応用情報

CPUが命令を効率よく処理するための仕組み「パイプライン」。命令の処理を5つの段階（ステージ）に分け、工場でのベルトコンベア作業のように並行して実行する技術について解説します。

【よくある問題パターン】
「RISCプロセッサにおいて、命令の実行を5つのステージに分割して処理するパイプライン方式で、正しいステージの順番はどれか？」

➡ 答え：IF → ID → EX → MEM → WB

1. パイプライン処理とは？（逐次処理との違い）

通常、1つの命令が終わってから次の命令を開始する「逐次処理」では、CPUの各部品に「待ち時間」ができてしまいます。
パイプライン処理では、処理の各段階を少しずつずらしながら重ねて実行することで、全体のスループット（処理量）を大幅に向上させます。

5段パイプラインなら、理論上は処理速度が最大で約5倍になります。

3. パイプラインを乱す「ハザード」

パイプライン処理は「命令が途切れることなく流れる」ことが前提ですが、処理の都合で流れが止まってしまうことがあります。これをパイプラインハザードと呼びます。

3種類のパイプラインハザード
⏳ データハザード
前の命令の「計算結果」が出ないと、次の命令が計算を始められず、待ち時間が発生する現象。
例：命令1の結果(R1)を、命令2がすぐ使う場合
🔀 制御ハザード
「条件分岐」が発生し、次に実行する予定で先読みしていた命令が破棄されてしまう現象。
対策：分岐予測（どっちに進むかあらかじめ予想する）
💥 構造ハザード
複数の命令が、同時に同じハードウェア（メモリやALUなど）を使おうとして競合する現象。
対策：命令メモリとデータメモリを分ける（ハーバードアーキテクチャ）

試験対策まとめ

略称 日本語名 役割のキーワード
IF 命令フェッチ 命令メモリから命令を読み出す（PCの更新）
ID 命令デコード 命令の意味を解析し、レジスタの値を読む
EX 実行 ALUで演算処理を行う、アドレスを計算する
MEM メモリアクセス データメモリへの読み書き（ロード/ストア時のみ重要）
WB 書き戻し 計算結果をレジスタへ保存（書き込む）
