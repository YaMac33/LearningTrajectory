「A+B×C」の2分木（構文木）と逆ポーランド表記法の作り方

#アルゴリズム #データ構造 #木構造 #基本情報 #応用情報

「A＋B×C」という普段見慣れた数式。これをコンピュータに正しく計算させるために、枝分かれした「2分木」の形に変換したものを「構文木（こうぶんき）」と呼びます。その描き方のルールと、試験で頻出の表記法の変換テクニックを解説します。

【構文木を描くための「絶対ルール」】
「計算の優先順位が低い（後回しになる）記号ほど、木の上（根っこ）にくる」
これさえ覚えておけば、どんなに複雑な数式でも正しく木を描くことができます。

1. なぜその形になるのか？（ツリーの描き方）

算数のルールでは、足し算（＋）よりも掛け算（×）を先に計算します。
つまり「A ＋ (B × C)」という順番になるため、一番最後に計算する「＋」が木の頂点（根）になります。

← 最後に計算する記号
← 先に計算するカタマリ
💻 コンパイラでの実務イメージ
プログラマーが書いた「A + B * C」というソースコードをコンピュータ（CPU）が理解できる機械語に翻訳する際、コンパイラは必ずメモリ上にこの「構文木」を作成します。
この木を下から上へとたどることで、「まずはBとCを掛ける命令を作って…その結果にAを足す命令を作ろう」と、正しい計算順序を組み立てているのです。

2. 午前試験頻出！3つの表記法と「木のなぞり方」

午前試験では、描かれた木構造の図を見て「この木から逆ポーランド表記法を作れ」という問題が頻出します。この問題は、木をぐるっと左回りに一周なぞる時の「文字を拾うタイミング（左・下・右）」だけで一瞬で解けます。

ポーランド表記法 (前置) 拾う場所：ノードの【左側】
＋ A × B C
各ノードの左側を通った瞬間に文字を書き出します。演算子がデータの前に来る表記法です。
通常の数式 (中置) 拾う場所：ノードの【下側】
A ＋ B × C
ノードの下側（真下）を通った瞬間に拾います。私たちが普段使っている数式そのままの形になります。
逆ポーランド表記法 (後置) 拾う場所：ノードの【右側】
A B C × ＋
ノードの右側を通り過ぎる瞬間に拾います。
「Aを置いて、BとCを掛けて(×)、それを足す(＋)」という、コンピュータが計算する手順そのままの並びになります。

3. 関連する木構造との違い（混同注意）

「2分木（枝が最大2本に分かれる木）」には、目的によっていくつか種類があります。試験で選択肢として並んだ時に迷わないよう、特徴を整理しておきましょう。

名称 配置のルールと主な目的
構文木 計算の優先順位が低いものほど上（根）にする。
目的：数式の評価やコンパイラの構文解析。
2分探索木 「左の子 ＜ 親 ＜ 右の子」の順に大小関係を揃える。
目的：データを高速に検索（バイナリサーチ）するため。
ヒープ木 「親の値は、必ず子よりも大きい（または小さい）」というルール。
目的：最大値（または最小値）を素早く取り出すため（ヒープソートなど）。

🎯 逆ポーランド表記法の試験対策
「逆（後置）」と聞かれたら、木を左回りに一周なぞって、文字の【右側】を通過した瞬間に書き出す！これだけでパズルのように解けます。
