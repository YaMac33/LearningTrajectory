HTTPはなぜ危険なのか？盗聴された生データの見え方とバケツリレー

#セキュリティ #ネットワーク #HTTP #パケットキャプチャ #基本情報

「暗号化されていないから危険」と頭ではわかっていても、実際に自分のパスワードがネットワーク上をどう流れているのかを想像するのは難しいものです。
今回は、インターネット通信の「バケツリレー」という根本的な仕組みと、ハッカーのパソコンに映し出される**「恐ろしい生データ」**の画面を図解します。

【1行要約：なぜ丸見えなのか？】
ズバリ、HTTPはあなたが入力したパスワードやクレジットカード番号を**「一切暗号化せず、世界中誰でも読める標準ルールの文字データ（平文）のまま、電波やケーブルに流しているから」**です！

1. 通信は「何十回ものバケツリレー」である

コンピュータの通信の仕組みそのものは、元々インターネットが生まれた頃の「性善説（みんな悪いことはしない前提）」で作られています。
あなたのスマホから飛び出たデータは、直接Webサーバーに届くわけではありません。何十個もの見知らぬ中継地点（ルーター）を経由して運ばれます。

このバケツリレーの途中に、もし悪意のある人間が管理するルーター（偽のフリーWi-Fiなど）が混ざっていたら、通過するデータは簡単に横取りされてしまいます。

2. 盗聴者には「どう見えている」のか？

HTTPで送られているデータは「世界共通の標準語（ASCIIなど）」です。盗聴者は、パソコンに入っている「パケットキャプチャ（通信を覗き見するソフト）」を起動するだけで、流れてきた0と1の信号を一瞬で元の文字に翻訳し、画面に表示させることができます。

実際に盗聴されたHTTPの生データ（平文）
Packet Capture - sniffer.exe
# Target detected: User clicked login button.
# Capturing raw HTTP traffic...

POST /login.php HTTP/1.1
Host: www.shopping-example.com
User-Agent: Mozilla/5.0 (iPhone; CPU iPhone OS 16_0 like Mac OS X)
Accept-Language: ja-JP
Content-Type: application/x-www-form-urlencoded
Content-Length: 43

--- Body Data ---
user_id=taro.yamada&password=secret12345&submit=Login

👿 ハッカー「なるほど、IDは『taro.yamada』で、パスワードは『secret12345』だな。いただき！」

このように、HTTP通信は「宛先」も「使うブラウザの種類」も、そして「送るパスワード」も、すべてが整理されたただのテキスト（文字）として流れています。これが「透明なハガキ」と呼ばれるゆえんです。

💻 実務イメージ：Wireshark（ワイヤーシャーク）
情報システムの現場でも、ネットワークの故障調査などで通信の中身を見るために「パケットキャプチャツール（Wiresharkなど）」を使います。
もし自社の社内システムが古いHTTPで作られていた場合、このツールを起動するだけで、同僚が入力した社員IDやパスワードが、管理者のパソコン画面にハッキリと英数字のまま流れていくのが見えてしまいます。非常に恐ろしい状態です。

試験対策まとめ：HTTPとHTTPSの違い

午前試験で必ず問われるのは、この「HTTPの弱点」と「ポート番号」の結びつきです。暗号化の有無と扉の番号をセットで暗記しましょう。

プロトコル 暗号化 ポート番号 特徴・試験でのキーワード
HTTP なし 80 平文（ひらぶん）で通信されるため、パケットキャプチャによる盗聴や改ざんに極めて弱い。
HTTPS あり 443 HTTPの通信を、SSL/TLSの仕組みで包み込み、通信経路全体を暗号化したもの。（SecureのS）

🎯 次のステップへの繋がり
この「誰でも読めちゃうテキストデータ」を、送信する直前（スマホの中）で「絶対に解読できないぐちゃぐちゃのデータ」に変換してからバケツリレーに流すのが、前回学習した「SSL/TLS（ハイブリッド暗号）」の役割です！
