変更管理委員会の解説

システム変更に伴うリスクを審議・承認する重要な組織、CABについて解説します。

# 1. 用語の意味（結論）

👉 システムに対する変更が、業務やサービスに悪影響を与えないかを審議・承認する組織

（Change Advisory Board：略してCAB）

# 2. なぜ必要なのか（背景）

システム変更には、必ずリスクが伴います。

  • ⚠️ 予期せぬシステム障害が起きる
  • ⚠️ 重要な業務が止まってしまう
  • ⚠️ セキュリティ事故につながる

特に、プログラム修正、サーバ構成変更、ネットワーク設定変更などは影響が大きいため、「勝手な変更を防ぐ仕組み」としてCABが設置されます。

# 3. 何をする組織なのか（役割）

変更管理委員会では、主に以下の項目について審議します。

変更の必要性
なぜその変更が必要か？
影響範囲
どこまで影響が出るか？
リスクの大きさ
失敗したらどうなるか？
対応策（バックアウト）
失敗時の切り戻し手順は？

# 4. 仕組みのイメージ（承認フロー）

👤
システム担当者
「変更要求」を出します
CAB（変更管理委員会）
審議
❓ 変更は妥当か？
❓ リスクは許容範囲か？
❓ 影響範囲はどこまでか？
承認 → 実施
否認 → 中止
👉 CABの承認なしに変更は行わないのが原則

# 5. ITILとの関係（試験で重要）

変更管理委員会は、ITサービスマネジメントのベストプラクティス集であるITILに基づく組織です。

CAB ⊂ ITILの変更管理プロセス

# 6. 具体例（実務イメージ）

基幹システムのバージョンアップを例に、CABでどのように意見が調整されるか見てみましょう。

💻
開発担当
「新機能を追加したいので、来週アップデートしたいです！」
⚙️
運用担当
「平日昼間に止められると困ります。業務への影響が大きすぎます。」
🔒
セキュリティ担当
「その新機能、脆弱性診断は済んでいますか？リスク評価が必要です。」
⚖️ CABの判断

「セキュリティ診断を完了させることを条件に、業務影響の少ない休日深夜に実施することで承認します。」

# 7. ITパスポート試験でのポイント

❌ よくある間違い

「CABは変更作業を実施する組織である」
（作業するのはエンジニアや担当者です）

⭕ 正解

「CABは変更の可否を判断・承認する組織である」
（意思決定機関としての役割が問われます）
