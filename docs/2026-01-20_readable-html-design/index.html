Markdownのように読めるHTMLデザイン

この記事は、生のHTMLでありながら、まるでMarkdownエディタで書いたような「読みやすさ」を重視したデザインのサンプルです。ブログやドキュメント作成に最適です。

1. なぜ読みやすいのか？

読みやすいWebページにはいくつかの共通点があります。このテンプレートでは以下のCSS設定を行っています。

  • 適切な行間 (Line-height): 日本語は英語よりも文字の密度が高いため、1.8程度の広めの行間を設定しています。
  • 読みやすい文字幅 (Max-width): 一行が長すぎると目が疲れます。幅を800pxに制限し、中央揃えにしています。
  • 視覚的な階層構造: 見出し（H1, H2）の下に細い線を入れることで、話題の区切りを明確にしています。

2. コードの表示例

エンジニアブログには欠かせないコードブロックの表示も、Githubのようなスタイルにしています。

// JavaScriptのサンプルコード
function helloWorld() {
    console.log("Hello, simple world!");
}

helloWorld();

文中の短いコードは const readability = "high"; のようにハイライトされます。
