AWSの最強データベース「Aurora」

  • Aurora
  • AWS
  • データベース
  • 性能
  • クラウド

Amazon Aurora（オーロラ）は、AWSが本気を出して開発した、「AWS史上最強のデータベース」です。RDSの一部ではあるのですが、中身が凄すぎて別格扱いされています。試験でも「性能」「可用性（壊れにくさ）」の面で正解になりやすい選択肢です。

一言で言うと、「オープンソース（MySQLなど）の使いやすさと、商用DB（Oracleなど）の高性能・高信頼性を併せ持った、AWS専用のデータベース」です。

1. Auroraの正体：何が「最強」なのか？

Auroraは、ゼロから新しく作られた言語ではありません。既存の人気データベースと互換性があります。

  • MySQL 互換
  • PostgreSQL 互換

つまり、今までMySQLで使っていたアプリやツールが、そのままAuroraでも使えます。なのに、AWSのクラウドインフラに最適化されるように中身が改造されているため、性能が桁違いです。

  • MySQLの 最大5倍 高速
  • PostgreSQLの 最大3倍 高速

試験ポイント： 「MySQL互換で、より高速で可用性の高いDBを選べ」と言われたら、迷わず Amazon Aurora です。

2. 最大の特徴：常識破りの「データの守り方」

普通のRDSとは、データの保存方法が根本的に違います。これがAuroraの強さの秘密です。

6つのコピーを3つのAZに (6 copies across 3 AZs)

あなたがデータを1つ書き込むと、Auroraは裏側で自動的に6個のコピーを作ります。

AZ-a
1
2
AZ-b
3
4
AZ-c
5
6

計6個のコピーを常時維持

ここが凄い：

  • 「AZが1つ丸ごと火事で消滅」しても、他のAZにデータがあるので止まりません。
  • 「ディスクが壊れた」としても、自動検知して勝手に直します（自己修復機能）。

3. Auroraだけの特殊能力

RDS（標準のMySQLなど）にはない、Auroraならではのメリットが試験に出ます。

① ストレージが「勝手に増える」

RDSでは最初に容量を決めますが、Auroraはデータが増えるに合わせて、自動で容量が拡張されます（最大128TBまで）。「今後のデータ量が予測できない」という場合に最適です。

② リードレプリカが「多い・速い」

RDSは最大5台ですが、Auroraは最大15台まで増やせます。しかも、メインとの時差（ラグ）が極めて少ない（ミリ秒単位）ため、大規模な読み取りアクセスを処理できます。

③ Aurora Serverless

アプリからのアクセスがない時は自動で停止（シャットダウン）し、アクセスが来たら数秒で起動するモードがあります。開発環境や、たまにしか使われない社内システムでコストを劇的に削減できます。

4. RDS と Aurora の比較表（試験直前チェック）

試験では「どっちを使うべき？」という比較問題が出ます。

特徴 Amazon RDS (標準) Amazon Aurora (最強)
エンジン MySQL, Postgres, Oracle等 MySQL互換, PostgreSQL互換
速度 普通 5倍 / 3倍 速い
データのコピー マルチAZ設定で 2箇所 (1+1) 標準で 6箇所 (3AZ × 2)
ストレージ 自分でサイズを決める必要あり 自動拡張 (Auto Scaling)
コスト 安い (t3.microなどが選べる) 少し高い (性能が良い分、単価は高め)
向いている用途 コスト重視、小規模、Oracle必須 性能重視、大規模、絶対に止めたくない

5. Aurora Global Database（世界展開）

DynamoDBのグローバルテーブルのように、Auroraも世界中にデータを高速コピーできます。

  • シナリオ： 東京リージョンで書き込んだデータを、1秒以内に米国リージョンで読めるようにしたい。
  • 用途： グローバルなサービス展開や、国レベルの災害対策（DR）。

6. まとめ

  • AWSネイティブ： クラウドのために再設計されたDB。
  • 互換性： MySQL(5倍速い) / PostgreSQL(3倍速い)。
  • 防御力： データは 3つのAZに6個コピー される（標準仕様）。
  • 柔軟性： ストレージ容量は自動で増える。
  • サーバーレス： Aurora Serverlessなら使わない時は停止できる。

「性能」「信頼性」「スケーラビリティ」を求められたら、とりあえずAuroraを検討する。それがAWSの設計の基本です。
