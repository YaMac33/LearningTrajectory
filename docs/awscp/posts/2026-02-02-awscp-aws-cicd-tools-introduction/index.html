AWS CI/CDツールの紹介

#AWS #CI/CD #DevOps #自動化 #ソフトウェア開発

AWS Codeシリーズは、ソフトウェア開発の「作る（保存）→ 組み立てる（ビルド）→ 配る（デプロイ）」という一連の流れを全自動化するためのサービス群です。

これらをまとめて「CI/CD（継続的インテグレーション／継続的デリバリー）ツール」と呼びます。

一言で例えると、「全自動のパン工場（ベルトコンベア）」です。人間は「レシピ（コード）」を投入口に入れるだけで、あとは機械が勝手にパンを焼き、包装し、お店に並べてくれます。それぞれの役割を「パン工場」に例えて解説します。

2. 各サービスの詳しい役割

① AWS CodeCommit（コードコミット）

役割：ソースコード管理（Gitリポジトリ）

  • 工場の役割: 「巨大な冷蔵庫（倉庫）」です。
  • プログラマーが書いたコード（設計図）を安全に保存し、「誰がいつ書き換えたか」という履歴を管理します。
  • GitHub や GitLab のAWS版だと思ってください。プライベート（非公開）なリポジトリを安全に持てます。

② AWS CodeBuild（コードビルド）

役割：ビルド＆テスト（コンパイル・テスト実行）

  • 工場の役割: 「調理オーブン」です。
  • 倉庫からコードを取り出し、機械が読める形に変換（コンパイル）します。
  • 同時に、「味が変じゃないか？」というテスト（単体テスト）を自動で実行します。
  • 特徴はサーバーレスであること。ビルドする時だけ動くので、待機時間は無料です。

③ AWS CodeDeploy（コードデプロイ）

役割：自動デプロイ（本番環境への配布）

  • 工場の役割: 「配送トラック」または「陳列スタッフ」です。
  • 完成したソフトを、実際のサーバー（EC2, Lambda, Fargate, オンプレミス）に自動で配置します。
  • 手作業でのファイルコピー（FTPなど）によるミスを撲滅します。「Blue/Greenデプロイ」などの高度な技を使い、サービスを止めずに切り替えることも可能です。

④ AWS CodePipeline（コードパイプライン）

役割：オーケストレーション（自動化の進行役）

  • 工場の役割: 「工場長」 または 「ベルトコンベアそのもの」です。
  • 「CodeCommitにコードが保存されたら、すぐにCodeBuildを動かせ。終わったらCodeDeployに渡せ」という指示出しを行います。
  • これがあるおかげで、人間は「コードを保存（Commit）」するだけで、数分後には本番環境が更新されるという完全自動化が実現します。

まとめ表（試験対策）

試験では「どの工程を担当するか」が問われます。

サービス名 役割 フェーズ たとえ
CodeCommit コードの保存 Source 倉庫 (Git)
CodeBuild テスト・コンパイル Build 調理場 (テスト実行)
CodeDeploy サーバーへの配置 Deploy 配送係 (自動更新)
CodePipeline 全体の自動化 Pipeline ベルトコンベア (司令塔)

補足：AWS CodeStar（コードスター）

たまに選択肢に出るのが AWS CodeStar です。

  • これは、上記の4つ（Commit, Build, Deploy, Pipeline）を「一括でセットアップしてくれるテンプレート機能」です。
  • 「Webアプリ用の環境を一瞬で作りたい」という時に、これら4つのサービスをまとめて自動構築してくれます。
  • 試験対策: 「CI/CDパイプラインを素早く構築したい」と言われたら CodeStar です。

これで、AWSの主要な開発者用ツール（DevOpsツール）もコンプリートです。
