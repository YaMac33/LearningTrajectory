AWSウェル・アーキテクテッド入門

  • AWS
  • Well-Architected
  • フレームワーク
  • 設計原則
  • クラウド

これまで数多くのAWSサービス（EC2, S3, RDSなど）を学んできましたが、これらはあくまで「武器」や「材料」に過ぎません。良い材料を持っていても、「どう配置し、どう組み合わせれば、最強の城（システム）になるか？」を知らなければ意味がありません。

そのための「設計図」や「評価基準」となるのが、AWS Well-Architected Framework（ウェル・アーキテクテッド・フレームワーク）です。試験では「この行動は、どの柱に基づいているか？」という問題が頻出します。

1. 6つの柱（The 6 Pillars）

このフレームワークは、6つの視点（柱）で構成されています。それぞれの目的とキーワードを整理しましょう。

⚙️ 1. 運用上の優秀性
(Operational Excellence)
自動化 IaC 失敗から学ぶ

システムを運用・監視し、改善し続ける能力。
行動例： CloudFormationでインフラをコード化する。小さな変更を頻繁に行う。

🛡️ 2. セキュリティ
(Security)
多層防御 最小権限 データ保護

情報やシステムを保護する能力。
行動例： IAMで権限を絞る。S3の暗号化を有効にする。CloudTrailでログを取る。

🔄 3. 信頼性
(Reliability)
復旧 冗長化 スケーリング

障害から自動復旧し、サービスを止めない能力。「すべては壊れる」が前提。
行動例： マルチAZ配置。Auto Scaling。復旧手順のテスト。

🚀 4. パフォーマンス効率
(Performance Efficiency)
サーバーレス 適材適所

リソースを無駄なく効率的に使う能力。
行動例： CloudFrontで配信を高速化。Lambdaを使ってサーバー管理をなくす。

💰 5. コスト最適化
(Cost Optimization)
支出把握 需要と供給

無駄な出費を削り、コストを最小化する能力。
行動例： Cost Explorerで分析。スポットインスタンスの利用。未使用リソースの削除。

🌱 6. サステナビリティ
(Sustainability)
省エネ カーボンフットプリント

環境負荷（エネルギー消費）を最小限にする能力。
行動例： 省電力なAWS Gravitonプロセッサの採用。不要データの削除。

2. 一般的な設計原則

6つの柱全体に通じる、AWS設計における「心構え」です。試験の選択肢で頻出のフレーズです。

🛑 キャパシティの推測をやめる (Stop guessing capacity) 「たぶんサーバーは3台くらい必要かな？」という勘をやめ、Auto Scalingで需要に合わせて自動追従させます。
🧪 本番規模でテストする (Test systems at production scale) クラウドなら本番と同じ環境を一瞬で作れます。テスト時だけ本番相当の環境を用意し、終わったら消すことで低コストに実現できます。
🧱 アーキテクチャの実験を簡単にする インフラを自動化しておけば、失敗してもすぐに作り直せます。これにより、新しい構成への挑戦が容易になります。
📊 データに基づいて決定する CloudWatchなどのデータ（事実）に基づいて、アーキテクチャの判断を行います。

3. AWS Well-Architected Tool

自分のシステムがこのフレームワークに沿っているかを診断するための、無料のツールがAWSコンソールに用意されています。

  • 機能： いくつかの質問に答えるだけで、「セキュリティに高リスクがあります」「信頼性が不十分です」といったレポートを出してくれます。

4. まとめと学習の完了

これで、AWSクラウドプラクティショナー（CLF）合格に必要な主要知識のレクチャーは完了です。

  • クラウドの概念： クラウドのメリット、責任共有モデル、Well-Architected。
  • セキュリティ： IAM、WAF、Shield、Artifact、KMS。
  • テクノロジー： EC2, S3, RDS, DynamoDB, Lambda, VPC, Route53, CloudFront, ELB。
  • 請求と料金： コスト管理、サポートプラン。

これらの知識がつながった今、あなたは「AWSという巨大な道具箱」の中身を理解し、適切な道具を選べるようになっています。
