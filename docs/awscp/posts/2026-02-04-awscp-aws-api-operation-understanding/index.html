AWSのAPIと操作の理解

#AWS #API #マネジメントコンソール #CLI #CloudTrail

AWS操作は画面でもコマンドでも同じAPIとして処理される仕組みと、APIキーの役割について解説します。

Q. 「画面（マネジメントコンソール）からの操作も、コマンド（CLI）からの操作も、すべて『API』として処理されます」という言葉の意味は？
A. 普段プログラミングで使う「APIキー」のイメージがあると混乱しますよね。ここでのAPIとは、「AWSへの正式な『注文書』」のことだと考えてください。

AWSの裏側の仕組みを、「巨大なレストラン」に例えると一発で理解できます。

1. レストランの厨房（AWS）から見た景色

まず、AWSという巨大なレストランの厨房をイメージしてください。

  • 厨房 (AWS本体): ここにはシェフ（EC2やS3などのサービス）がいます。
  • ルール: シェフは、「決まった形式の注文伝票（APIリクエスト）」しか受け取りません。「おいしいやつ適当に！」と叫んでも無視されます。「ハンバーガーセット、ピクルス抜きで」と正確に書かれた伝票が必要です。

さて、客（あなた）がこの伝票を厨房に届けるには、3つの方法があります。

① マネジメントコンソール（GUI） = 「タッチパネル注文」

あなたは、席にあるタブレット（ブラウザ画面）を見て、写真付きのメニューから「EC2起動」ボタンを指でポチッと押します。
すると、タブレットの内部で自動的に「注文伝票」が作られ、厨房に送信されます。

  • あなたの感覚: ボタンを押しただけ。
  • 実際の処理: 裏でAPI（注文伝票）が飛んでいる。

② CLI（コマンド） = 「手書きのメモを渡す」

あなたは、黒い画面に aws ec2 run-instances ... と文字（コマンド）を打ち込みます。
これも、エンターキーを押した瞬間に、①と全く同じ形式の「注文伝票」に変換され、厨房に送信されます。

③ SDK（プログラム） = 「電話注文」

プログラムコードを使って指示を出します。これも結局、同じ「注文伝票」を送っています。

2. 図解：すべては「1つの窓口」に集まる

これが「すべてAPIとして処理される」の正体です。入り口（見た目）が違うだけで、最終的にAWSに届くデータ（JSON形式のテキストデータ）は全く同じなのです。

すべての操作は「注文伝票」になる
📱 コンソール (GUI)
📝 コマンド (CLI)
📞 プログラム (SDK)
📄 API注文伝票 (JSONデータ)
➡
👨‍🍳 AWSの厨房
(Endpoint)

3. 「APIキー」のイメージはどうすればいい？

質問者様が仰る「APIキー」は、このレストランでいう「会員証（身分証明）」のようなものです。
注文伝票（API）には、必ず「誰からの注文か？」という署名（サイン）が必要です。

  • CLIの場合: あなたが設定した「アクセスキー（APIキーの一種）」が自動的に伝票にハンコを押してくれます。
  • コンソール（画面）の場合: ログインした時に、ブラウザが裏側で「一時的な会員証（セッション）」を受け取っています。ボタンを押すたびに、ブラウザが勝手にその会員証を伝票に貼り付けて送ってくれているのです。

つまり、「画面操作であっても、裏では見えないAPIキー（認証情報）が毎回使われている」のです。

4. なぜこの理解が重要なのか？（CloudTrailの話）

この仕組みを理解すると、AWS CloudTrail（操作ログ記録サービス）の凄さがわかります。
「画面操作もコマンド操作も、結局は同じ『注文伝票（API）』が届く」ということは…
「届いた注文伝票さえ全部コピーして保存しておけば、誰がどんな手段で操作しても、すべての記録が残せる」ということです。

CloudTrailに残るログのイメージ
{
  "who": "Alice",
  "action": "RunInstances (EC2起動)",
  "time": "12:00:00",
  "source": "console" (または cli)
}

画面でボタンを押しても、コマンドを打っても、記録されるログの形式は同じになります。

まとめ

  • 「APIとして処理される」の意味: 画面のボタンクリックも、裏側では「文字で書かれた命令書（HTTPリクエスト）」に変換されてAWSに送られているということ。
  • APIキーの疑問: 画面操作の場合も、ログイン情報という「見えないAPIキー」が裏で自動的に使われている。

「AWSというロボットは、ボタンを押されても、コマンドを打たれても、最終的には同じ言葉（API）で命令を受け取っている」というイメージです！
