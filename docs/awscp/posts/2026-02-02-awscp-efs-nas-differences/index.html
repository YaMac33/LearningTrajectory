EFSとNASの違い

#EFS #NAS #クラウド #ストレージ #比較

「Amazon EFS」と、オフィスや自宅にある「通常のNAS（ネットワーク・アタッチド・ストレージ）」は、「ネットワーク経由でみんなで共有できるフォルダ」という機能面では全く同じです。

しかし、「運用（面倒くささ）」と「耐久性（壊れにくさ）」が天と地ほど違います。

一言で言うと、「通常のNASは『自分で世話をするペット』、EFSは『蛇口をひねれば出る水道』」のような違いがあります。詳しく比較解説します。

運用の手間の違い
🐶
通常のNAS (ペット)
  • ハード購入・設置が必要
  • 故障したら自分で治療 (交換)
  • 容量が足りなくなったら引越し
  • 定期的な世話 (パッチ適用)
🚰
Amazon EFS (水道)
  • 契約不要ですぐ使える
  • 裏側で勝手に直る
  • 容量は無限に湧いてくる
  • メンテナンス不要

2. EFS が圧倒的に優れている 3つの点

① 容量設計がいらない（Elastic = 伸縮自在）

通常のNASを買う時、「将来これくらい使うかな？」と予想して、大きめの4TBなどのHDDを買いますよね？

  • 通常のNAS: データが一杯になったら、新しいHDDを買ってきて、データを移行するという地獄の作業が待っています。
  • EFS: 「容量」という概念がありません。ファイルを入れたら入れた分だけ勝手に広がり、ペタバイト（PB）クラスまで無限に大きくなります。ファイルを消せば勝手に縮み、料金も安くなります。

② 災害に強すぎる（Multi-AZ）

これがクラウドを使う最大の理由です。

  • 通常のNAS: オフィスに置いてあります。もしオフィスが火事になったり、停電したりしたら、データは終わりです。
  • EFS: あなたがファイルを保存した瞬間、裏側で「3つの異なるデータセンター（AZ）」にコピーされます。もし東京の一部が壊滅的な被害を受けても、データは生き残ります。これを自前のNASでやるのは不可能です。

③ メンテナンス時間ゼロ

  • 通常のNAS: ファームウェアの更新や再起動のために、「今夜はサーバー止めるからアクセスしないでね」とアナウンスする必要があります。
  • EFS: メンテナンスによる停止はありません。24時間365日、いつでも使えます。

3. 通常のNASの方が良い点（デメリット）

EFSも万能ではありません。通常のNASに負ける点もあります。

  • Windowsで使えない:
    EFSはLinux専用です。オフィスのWindows PCから手軽にアクセスするファイルサーバーとしては使えません（※その場合は FSx for Windows を使います）。
  • コスト（単価）:
    「1GBあたりの単価」だけで見ると、HDDを買ってきて自作するNASの方が圧倒的に安いです。EFSは「管理の手間賃」や「3拠点バックアップ代」が含まれているため、純粋なディスク単価は高めです。
    ※ただし、EFS IA（低頻度アクセス）などの割引機能を使えば、かなり安く抑えられます。

まとめ

  • 通常のNAS: 「箱」を買うこと。自分でHDDを交換したり、バックアップを管理したりする手間を楽しめる（またはコストを極限まで下げたい）場合。
  • Amazon EFS: 「サービス」を買うこと。「容量不足」や「故障」という悩み自体をお金で解決し、ビジネス（アプリ開発など）に集中したい場合。

企業がAWSを使う場合、「ハードウェアの管理をしたくないから」という理由が大きいため、ほとんどの場合でEFS（またはFSx）が選ばれます。
