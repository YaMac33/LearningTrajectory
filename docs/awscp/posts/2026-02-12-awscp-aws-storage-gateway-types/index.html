AWS Storage Gateway の種類まとめ

#AWS #Storage Gateway #ファイルゲートウェイ #ボリュームゲートウェイ #テープゲートウェイ

オンプレミスとAWSのストレージを繋ぐ「Storage Gateway」の3つの種類と使い分けを解説します。

AWS Storage Gateway は、オンプレミスのサーバーからAWSのストレージ（S3など）を、まるでローカルのディスクのように扱えるようにするハイブリッドストレージサービスです。
試験対策で最も重要なのは、「3つの種類（ファイル、ボリューム、テープ）」の違いと、ボリュームゲートウェイ内の「保管型」と「キャッシュ型」の違いを理解することです。

1. 全体像：3つのゲートウェイ

どの「通信プロトコル（接続方式）」を使って、何を保存したいかによって種類を選びます。

Storage Gatewayの種類
📁 ファイルゲートウェイ NFS / SMB
オンプレから「共有フォルダ」としてS3にファイルを保存する。既存アプリの改修が不要。
💽 ボリュームゲートウェイ iSCSI
「ブロックストレージ（ハードディスク）」としてS3を利用する。保管型とキャッシュ型の2種類がある。
🎞️ テープゲートウェイ iSCSI (VTL)
「仮想テープライブラリ」として機能。既存のバックアップソフトをそのまま使ってS3/Glacierに保存する。

2. ボリュームゲートウェイの2つの型（超重要）

ボリュームゲートウェイには、「どこにデータをフルで持っておくか（主：プライマリー）」によって2つのタイプがあります。
ここが試験で一番狙われるポイントです。

保管型（Stored）とキャッシュ型（Cached）の違い
① 保管型 (Stored Volume)

オンプレ側に全データを置く

🏢 オンプレミス
(ローカルディスク)
⬇ スナップショット
☁️ Amazon S3
(バックアップ用)
② キャッシュ型 (Cached Volume)

S3側に全データを置く

☁️ Amazon S3
(メインストレージ)
⬇ よく使うデータのみ
🏢 オンプレミス
(キャッシュ用・小容量でOK)
  • 保管型（Stored）: データはすべてオンプレミスにあり、そのバックアップをS3に逃がす方式。オンプレ側の大容量ディスクが必要です。
  • キャッシュ型（Cached）: データはすべてS3にあり、よく使う分だけをオンプレに保持する方式。「オンプレの容量不足を解消したい」「コストを削減したい」という要件に最適です。

3. 試験用・超まとめ表

種類 接続方式 主な保存先 (Primary) 用途・特徴
ファイルゲートウェイ NFS / SMB S3 共有フォルダのクラウド移行。
ボリューム（保管型） iSCSI オンプレミス ローカルアクセス速度重視 ＋ S3バックアップ。
ボリューム（キャッシュ型） iSCSI S3 オンプレの容量節約。S3をメインストレージ化。
テープゲートウェイ iSCSI (VTL) S3 / Glacier 物理テープの置き換え。既存ソフト流用。

4. 最重要ひっかけポイント（一問一答）

問題文の以下のキーワードを見逃さないようにしましょう。

  • Q. 「NFSやSMBを使ってファイル共有したい」 → A. ファイルゲートウェイ
  • Q. 「プライマリーデータをオンプレミスに保持したい」 → A. ボリュームゲートウェイの 保管型 (Stored)
  • Q. 「オンプレミスのストレージ容量が足りない。S3をメインにしたい」 → A. ボリュームゲートウェイの キャッシュ型 (Cached)
  • Q. 「既存のバックアップソフトウェアを変更せずに使いたい」 → A. テープゲートウェイ

「iSCSI」というプロトコルが出た場合は、「ファイル」ではないことが確定するので、「ボリューム」か「テープ」のどちらかに絞り込めます。
