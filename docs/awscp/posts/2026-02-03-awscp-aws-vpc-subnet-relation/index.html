AWSのVPCとサブネットの関係

#AWS #VPC #サブネット #インスタンス #クラウド

VPC、サブネット、インスタンスの関係を例えで理解。階層構造が重要です。

Q. VPCの中にサブネットがあって、その中にインスタンスがあるんだっけ？
A. その認識で100点満点です！

イメージとしては「マトリョーシカ」や「住所」のように、大きい箱の中に小さい箱が入っている構造になっています。
この階層構造は、AWSのネットワークを理解する上で一番の基礎になります。絶対にブレてはいけない部分ですので、図と例えで確実に固めておきましょう。

1. 階層構造の図解

上から順に、箱が小さくなっていきます。

AWSネットワークの包含関係
🌏 AWS (世界)
🇯🇵 リージョン (国:東京など)
🏡 VPC (私有地)
🚪 サブネット (部屋 / AZ単位)
💻 インスタンス (家具 / 住人)

2. 「家」で例えると完璧にわかります

この構造を「一軒家」に例えてみましょう。

VPC = 私有地、サブネット = 部屋
VPC (あなたの家)

※ここはあなただけの敷地。部外者は勝手に入れません。

🪟 パブリックサブネット
(リビング・玄関)
Webサーバー

窓(IGW)があり、外とやり取りできる。

🔒 プライベートサブネット
(寝室・金庫室)
DBサーバー

窓がない。外からは見えない安全な部屋。

  • VPC ＝ 「あなたの家（敷地）」
    • AWSという広い世界の中に、あなただけが使える「私有地」を確保しました。
    • ここには、部外者は勝手に入ってこれません。
  • サブネット ＝ 「家の中の『部屋』」
    • 家（VPC）の中を、壁で仕切って部屋を作ります。
    • パブリックサブネット: 「玄関やリビング」。窓があって、外の人（インターネット）とやり取りできる部屋。
    • プライベートサブネット: 「寝室や金庫室」。窓がなく、外から覗かれない安全な部屋。
  • インスタンス (EC2) ＝ 「部屋に置く『家具（パソコン）』」
    • 家具は、必ず「どこかの部屋」に置きますよね？「廊下（VPC直下）」にポツンと置くことはできません。
    • 「どの部屋（サブネット）に置くか」によって、その家具が外から見えるかどうかが決まります。

3. 試験で問われる「鉄の掟」

この階層構造には、いくつか絶対に破れないルールがあります。

  • インスタンスは、必ず「サブネット」に入れること
  • サブネットは、1つの「アベイラビリティーゾーン (AZ)」にしか作れない
  • VPCは、複数のAZにまたがれる
  • 「インスタンスは必ずサブネットに入れる」
    • 「VPCの中だけど、サブネットには属していない」というインスタンスは作れません。家具は必ず部屋に置きます。
  • 「サブネットはAZをまたげない」
    • 1つの部屋（サブネット）が、東京と大阪にまたがって存在することはできません。必ず「東京のA棟」のように場所が決まっています。
  • 「VPCはAZをまたげる」
    • 家（VPC）自体は大きくて、複数の棟（AZ）にまたがって建てることができます。

まとめ

VPC ⊃ サブネット ⊃ インスタンス

この包含関係さえ頭に入っていれば、ネットワークの問題は怖くありません。
「インスタンスをインターネットにつなぎたい」と思ったら、「インスタンスそのもの」を設定するのではなく、「インスタンスが入っている部屋（サブネット）」を玄関（パブリック）にする必要がある、という発想になります。

この調子でいくと、VPCのセキュリティ（セキュリティグループとネットワークACL）の違いも、スムーズに理解できるはずです！
