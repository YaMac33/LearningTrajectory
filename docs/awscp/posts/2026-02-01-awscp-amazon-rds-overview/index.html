Amazon RDSの概要と特徴

  • Amazon RDS
  • DB管理
  • マネージドサービス
  • Aurora
  • マルチAZ
  • リードレプリカ

Amazon RDS (Relational Database Service) は、S3やEC2と並んでAWSの根幹を支える最重要サービスの一つです。一言で言うと、「面倒な管理をAWSが全部やってくれる、伝統的なデータベース（Excelのような表形式のデータ）」です。試験では「何が便利なのか（マネージド）」と「2つのコピー機能（マルチAZとリードレプリカ）の違い」が問われます。

1. なぜEC2の中にDBを入れないのか？

「EC2の中にMySQLをインストールすればいいじゃないか」と思うかもしれません。もちろん可能ですが、それは「自炊」のようなもので、以下の手間が発生します。

  • OSのアップデート
  • DBソフトのパッチ当て
  • 毎日のバックアップ
  • 故障した時の復旧作業

RDSは、これらを全てAWSが肩代わりしてくれる「マネージドサービス」です。あなたは「データの読み書き」だけに集中できます。

2. RDSで使える「エンジン」（種類）

RDSはあくまで「枠組み」の名前です。中身のソフト（エンジン）は以下から選べます。

  • オープンソース系： MySQL, PostgreSQL, MariaDB
  • 商用系： Oracle, Microsoft SQL Server

試験で頻出のAWS独自エンジン

  • 特徴： MySQLやPostgreSQLと互換性がありながら、AWSクラウドに特化して改造された爆速データベース。
  • 性能： MySQLの5倍、PostgreSQLの3倍高速。
  • 価格： 少し高いが、信頼性が段違いに高い（データが3つのAZに計6個コピーされる）。
  • 試験対策： 「クラウドネイティブ」「高性能」という言葉が出たらAuroraです。

3. 試験の最難関：2つの「コピー機能」の違い

RDSには、データを別の場所にコピーする機能が2つあります。「何のためにコピーするのか？」という目的の違いが必ず出題されます。

4. RDSプロキシ (RDS Proxy)

これも最近よく出ます。

  • 課題： Lambdaなどの「サーバーレス」機能からRDSに大量に接続すると、接続処理（コネクション）が溢れてRDSがダウンすることがあります。
  • 解決策： RDS Proxyを間に挟むと、接続をうまくまとめて（プーリングして）管理してくれます。
  • キーワード： 「Lambdaからの接続過多」「データベース接続の効率化」。

5. 間違いやすい「他のDBサービス」との比較

試験では「この要件に最適なDBはどれか？」と聞かれます。RDSと迷うのが以下の2つです。

6. まとめ

  • マネージドサービス： バックアップやパッチ当てはAWSにお任せ。
  • Aurora： AWS最強のDB。速い・強い。
  • マルチAZ： 「予備機」を用意して、故障に備える（高可用性）。
  • リードレプリカ： 「コピー機」を用意して、負荷を分散する（性能向上）。
  • VPC内： 基本的にプライベートサブネットに置く（外から直接アクセスさせない）。

これで、「RDS（かっちりしたデータ）」と、対になる「S3（ファイルなんでも）」の違いがはっきりしたと思います。
