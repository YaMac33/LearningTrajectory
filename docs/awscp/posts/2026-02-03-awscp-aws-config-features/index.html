AWS Configの基本と機能

#AWS #Config #構成管理 #コンプライアンス #監査 #変更履歴

AWS Configはリソースの設定履歴を管理し、変更を追跡・監視するサービスです。

AWS Config は、AWSリソースの設定（Configuration）を記録し、「いつ、どう変わったか」を管理するサービスです。
一言で言うと、「AWS環境全体の『設定のタイムマシン』兼『校則違反チェッカー』」です。

試験では、以下のキーワードが出たら間違いなく AWS Config です。

  • 構成管理
  • コンプライアンス監査
  • 変更履歴の追跡

1. なぜ AWS Config が必要なのか？

ある日突然、システムが動かなくなりました。「誰かが設定を変えたのか？」「昨日の時点ではどうだったのか？」を知りたくなります。

トラブル発生時の調査アプローチ

CloudTrail

🎥

「犯人」を特定する

「誰が」APIを実行したか？
(例: user-AがDeleteSecurityGroupを実行)

AWS Config

📸

「現場」を検証する

設定が「どう変わったか」？
(例: 昨日15時にポート22が開放された状態になった)

  • CloudTrail: 「誰が（APIを）実行したか」はわかりますが、「具体的に設定値がどう変わったか」までは直感的にわかりにくいです。
  • Config: 「変更前」と「変更後」の状態（スナップショット）を記録しているので、具体的な設定値の変化が一目で分かります。

① 構成履歴の記録（タイムマシン）

AWSリソース（EC2, S3, RDSなど）の設定変更を自動で検知し、タイムラインとして保存します。
「過去のある時点」の構成に戻って確認できるため、トラブルシューティングに最強です。

② ルールによるコンプライアンスチェック（校則違反チェック）

あらかじめ「ルール（校則）」を決めておき、違反しているリソースがないか監視します。

  • ルール例:
    • 「EBSボリュームは暗号化されていること」
    • 「S3バケットは公開設定になっていないこと」
    • 「SSH（ポート22）が全開放されていないこと」
  • 検知: 違反しているリソースを見つけると、「非準拠（Non-compliant）」としてマークし、通知します。

③ 自動修復（Remediation）

ルール違反を見つけた時に、AWS Systems Manager (SSM) Automation と連携して、自動で直すことができます。
例えば、「SSH全開放のセキュリティグループ」を見つけたら、「自動的にそのインバウンドルールを削除する」というアクションを実行させることが可能です。

3. CloudTrail との違い（超重要）

この2つは「監視系サービス」として非常によく似ていますが、役割が違います。
トラブル時は、まず Config で「何が変わったか」を特定し、その時刻を元に CloudTrail で「誰がやったか」を特定する、という合わせ技が定石です。

特徴 AWS CloudTrail AWS Config
注目点 「人 (Who)」と「行動 (Action)」 「モノ (What)」と「状態 (State)」
記録内容 API呼び出し履歴 (操作ログ) リソースの設定変更履歴 (構成ログ)
質問への答え 「誰が EC2を止めた？」 「EC2の設定はどう変わった？」
たとえ 防犯カメラの映像 (犯人が映る) ビフォーアフターの写真 (変化がわかる)

4. 試験に出る重要キーワード

  • コンプライアンス監査: 「社内規定（ルール）を守っているかチェックしたい」と言われたらConfig。
  • 構成変更の追跡: 「トラブル原因調査のために、過去の設定を知りたい」と言われたらConfig。
  • AWS Config ルール:
    • マネージドルール: AWSが用意してくれている定番ルール（設定不要で使える）。
    • カスタムルール: Lambdaを使って自分で書く独自ルール。
  • アグリゲータ (Aggregator): 複数のAWSアカウントや、複数のリージョンのConfigデータを「一箇所に集約」して見れる機能。

まとめ

  • 役割: リソースの「設定」を監視・記録・評価するサービス。
  • 機能: 設定のタイムライン（履歴）が見れる。ルール違反（非準拠）を検知・自動修復できる。
  • 比較: CloudTrail は「操作」を記録、Config は「設定（状態）」を記録。

CloudWatch（性能監視）、CloudTrail（操作ログ）、Config（構成管理）の「監視3兄弟」を使い分ければ、AWS環境は鉄壁です。
