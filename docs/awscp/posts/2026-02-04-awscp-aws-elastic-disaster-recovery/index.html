AWS Elastic Disaster Recoveryの魅力

#AWS #Disaster Recovery #低コスト #復旧時間 #DRS

低コストで迅速なシステム復旧が可能なAWS DRSの特徴と利点を解説します。

AWS Elastic Disaster Recovery (DRS) は、「災害が起きた時だけ、AWS上にコピー環境を立ち上げるサービス」です。
一言で言うと、「維持費が激安な『スペアタイヤ』」です。オンプレミスのサーバーが壊れたり、データセンターが停電したりした時（災害時）に、数分でAWS上にシステムを復旧（フェイルオーバー）させることができます。

1. 従来のDR（災害対策）との違い

これまでの災害対策（DR）は「お金」との戦いでした。「いつ起こるかわからない災害」のために、高いコストを払い続ける必要があったからです。

  • 従来の方式（ホットスタンバイなど）:
    • 本番と同じスペックのサーバーを、予備として常に起動しておく必要があります。
    • デメリット: 何も起きていない平和な時も、2倍の料金がかかり続けます。
  • AWS DRSの方式（パイロットライトの進化版）:
    • 普段は、データを保存する「ディスク（EBS）」と、データを受け取る「極小のサーバー」しか使いません。
    • 本番スペックのEC2は、災害が起きて「復旧ボタン」を押すまで起動しません。
    • メリット: 平常時のコストを激安（従来の数分の一）に抑えられます。

2. 仕組み： 「ステージングエリア」が肝

このサービスの賢いところは、「ステージングエリア（準備室）」という考え方です。

  • 平常時（レプリケーション）:
    • オンプレミスや他社のクラウドにあるサーバーに「エージェント」を入れます。
    • データは常にAWSの「ステージングエリア」に同期され続けます。ここは軽量・安価なリソースで動いています。
  • 災害時（フェイルオーバー）:
    • あなたがAWSコンソールで「復旧」ボタンを押します。
    • AWS DRSが、ステージングエリアのデータを使い、自動的に本番スペックのEC2インスタンスへと変換・起動します。

3. AWS Backup との違い（試験のポイント）

「データを守る」という意味では似ていますが、「復旧スピード（RPO/RTO）」が段違いです。

4. 試験対策キーワード

試験で以下の要件が出たら、AWS Elastic Disaster Recovery (DRS) が正解です。

  • 「オンプレミスのサーバーをAWSへ迅速に復旧（フェイルオーバー）させたい」
  • 「RPO（目標復旧時点）は数秒、RTO（目標復旧時間）は数分以内にしたい」
  • 「災害対策のコストを最小限に抑えたい（アイドル状態のインスタンスにお金を払いたくない）」

まとめ

  • AWS Elastic Disaster Recovery (DRS): 低コスト・爆速復旧を実現するDRサービス。
  • ポイント: 普段は「更地（ディスクのみ）」で待機し、災害時だけ「ビル（EC2）」を建てるから安い。

「AWS Backupじゃ復旧が間に合わない！でも常時2台動かすお金はない！」という時の切り札です。
