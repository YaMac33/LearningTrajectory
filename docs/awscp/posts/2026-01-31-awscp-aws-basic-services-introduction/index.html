AWS基本サービスの紹介

  • AWS
  • EC2
  • S3
  • RDS
  • クラウド
  • サービス

AWSには200以上のサービスがありますが、その中でも「三種の神器」とも言える基本サービスがあります。これらはITパスポートなどで学ぶ一般的なIT知識と紐付けるとイメージしやすくなります。まずはこの3つを完璧に押さえましょう。

1. Amazon EC2（イーシーツー）

EC2は、「クラウド上の仮想サーバー（パソコン）」です。自分のPCの代わりに、インターネット上にWindowsやLinuxのPCを借りて、そこでプログラムを動かす場所と考えれば分かりやすいでしょう。

EC2のイメージ：仮想のレンタルPC
あなたのPC
↓ リモート接続
Amazon EC2
アプリケーション OS (Windows / Linux) CPU / メモリ
  • 役割： 自由に使えるコンピューティングリソース（計算能力）。
  • 試験ポイント： サーバーの電源管理や、OSのアップデート（パッチ当て）は「利用者の責任」で行います。AWSはハードウェアのみ管理します。

2. Amazon S3（エススリー）

S3は、「容量無制限のデータ保管庫」です。PCに繋がっているハードディスクではなく、インターネット上に独立して存在する巨大な「物置」です。GoogleドライブやDropboxの「システム用」のようなものとイメージしてください。

S3のイメージ：巨大なファイル倉庫
スマホ
PC
↓ インターネット (HTTP/HTTPS)
Amazon S3 バケット（入れ物）
画像.jpg 動画.mp4 ログ.txt
  • 役割： 画像、動画、ログファイルなどを「オブジェクト」として保存します。
  • 試験ポイント： 容量は無制限（使った分だけ課金）です。Webサイトの静的コンテンツ（画像など）置き場として頻繁に使われます。

3. Amazon RDS（アールディーエス）

RDSは、「面倒な管理をAWSがやってくれるSQLデータベース」です。EC2の中に自分でDBソフトをインストールすることも可能ですが、バックアップや設定が大変です。RDSはその「面倒な作業」をAWSが肩代わりしてくれます。

RDSのイメージ：管理お任せDB
アプリ
(EC2など)
→ SQL命令
Amazon RDS
DBエンジン (MySQL等) 自動バックアップ パッチ適用・自動復旧
  • 役割： 構造化データの管理（SQLを使用）。
  • 試験ポイント： 「マネージドサービス」の代表例です。利用者は「データの中身」の管理に集中し、インフラ運用はAWSに任せます。

4. この3つはどう連携する？

実際のシステム開発やAWS認定試験では、この3つを組み合わせた構成が頻出です。例えば「ショッピングサイト」を作る場合、以下のような役割分担が鉄板パターンとなります。

EC2 Webサイトの
プログラム（レジ係）
画像を保存 ➔
S3 商品写真（商品棚）
注文履歴を保存 ➔
RDS 顧客リスト（帳簿）

このように、「処理はEC2」「ファイルはS3」「データはRDS」という役割分担をイメージできれば、AWSの基礎的なアーキテクチャは理解できたと言えます。
