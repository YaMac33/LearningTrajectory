Amazon OpenSearch Serviceの概要

#OpenSearch #検索 #ログ分析 #AWS #Elasticsearch #可視化

OpenSearchは検索とログ分析のためのAWSサービス。Elasticsearchから派生し、2つの使い道があります。

Amazon OpenSearch Service（旧称：Amazon Elasticsearch Service）は、「全文検索」と「ログ分析」のためのサービスです。
一言で言うと、「あなたのサイト専用の『Google検索エンジン』を作ったり、大量のログを『グラフ』で見える化したりするツール」です。

試験でも実務でも非常に重要ですが、名前が変わったりして少しややこしいので、3つのポイントで整理します。

1. 生い立ち：「Elasticsearch」からの独立

もともと、世界中で大人気のオープンソース検索エンジン「Elasticsearch（エラスティックサーチ）」というものがありました。
しかし、ライセンス（権利関係）の問題でAWSがそれを自由に使えなくなってしまったため、AWSが独自に分岐（フォーク）させて作ったのが、この OpenSearch です。

サービスの成り立ち

※古い試験問題では「Elasticsearch Service」と書かれている場合があります。

2. OpenSearch の「2つの顔」

このサービスには、全く異なる2つの使い道があります。試験ではどちらのパターンで出ても答えられるようにしてください。

主なユースケース
🔍 ① 検索エンジン
Webサイトやアプリに
「強力な検索窓」をつける。

例: ECサイトの商品検索
(あいまい検索、予測変換)
📊 ② ログ分析ツール
大量のログデータを
「グラフで可視化」する。

ツール: OpenSearch Dashboards
(旧 Kibana)

① 「検索エンジン」としての顔

Webサイトやアプリに、強力な検索機能を実装したい時に使います。
「スニーカー 赤」のようなあいまいな検索や、オートコンプリート（予測変換）、ランキング機能などが簡単に作れます。

② 「ログ分析ツール」としての顔

実は、AWSではこちらの用途で使われることの方が多いです。
サーバーやアプリから出る大量のログデータをOpenSearchに投げ込み、セットになっている「OpenSearch Dashboards（旧 Kibana）」というツールで見ると、かっこいいグラフやダッシュボードでリアルタイムに可視化できます。

3. CloudSearch との違い

AWSにはもう一つ、Amazon CloudSearch という古い検索サービスがあります。試験で迷わないように比較します。

特徴 Amazon OpenSearch Service Amazon CloudSearch
人気・主流 圧倒的に主流 現在はあまり使われない
機能 多機能 (検索 + ログ分析) シンプル (検索のみ)
可視化ツール Dashboards (Kibana) が付属 なし
選び方 「ログ分析」「可視化」「最新の検索」ならこれ 「とにかく管理を楽にしたい」「単純な検索でいい」時

まとめ

  • 正体: Elasticsearch から派生した、検索と分析のエンジン。
  • 用途A: アプリに「検索機能」をつける（Googleのような検索窓）。
  • 用途B: ログを収集して「可視化・分析」する（Kibana / Dashboards）。

試験で「アプリケーションログをリアルタイムで分析・可視化したい。どのサービスを使うべきか？」と聞かれたら、Amazon OpenSearch Service が正解です！
