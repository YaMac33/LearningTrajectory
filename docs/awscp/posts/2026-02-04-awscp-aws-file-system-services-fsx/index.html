AWSのファイルシステムサービスFSx

#AWS #FSx #ファイルサーバー #Windows #HPC

Amazon FSxはWindows向けのファイルサーバーサービス。高い互換性と複数のタイプを提供します。

Amazon FSx（エフエスエックス）は、「Windowsやハイパフォーマンスな計算処理（HPC）でよく使われる『特定のファイルシステム』を、AWS上でそのまま使えるようにしたサービス」です。
一言で言うと、「『Windowsファイルサーバー』や『超高速ファイルサーバー』のレンタルサービス」です。

AWSには他にも Amazon EFS というファイル共有サービスがありますが、EFSは「Linux専用」です。「じゃあWindowsはどうするの？」という声に応えるのが、この FSx です。

1. なぜこれが必要なの？（EFSとの違い）

一番の理由はOSとの「互換性」です。LinuxとWindowsでは話す言葉（プロトコル）が違うため、EFSだけではカバーできません。

OSによる使い分け
🐧 Linux
NFSプロトコル
Amazon EFS
🪟 Windows
SMBプロトコル
Amazon FSx
  • Amazon EFS: Linux標準の仕組み（NFS）で動いています。Windowsからはうまく繋がりません。
  • Amazon FSx: 「Windows Serverそのもの」や「Lustreという専用ソフト」をAWSが裏で動かしてくれています。だから、今使っている環境をそのままAWSに持ってこれます。

2. 試験に出る「4つのタイプ」

FSxには、用途に合わせて4種類の「味（タイプ）」があります。試験では特に最初の2つが頻出です。

FSxの4つのラインナップ
🏢 FSx for Windows 超頻出
本物のWindows Serverが裏で動く。
「共有フォルダ（Zドライブ）」やActive Directory連携ならこれ。
🚀 FSx for Lustre HPC
スーパーコンピュータ並みの速度。
科学計算、動画処理、機械学習向け。
S3と連携してデータを読み書きできる。
🗄️ FSx for NetApp ONTAP
高機能ストレージ「NetApp」を完全再現。
オンプレからの移行や高度なデータ管理に。
🐧 FSx for OpenZFS
Linux向けの高速ファイルシステム。
EFSよりも高い性能が必要な場合に選択。

① Amazon FSx for Windows File Server （超重要！）

社内のファイル共有、Active Directory（AD）連携、Windowsアプリの移行に使います。
「会社のデスクの横にある『共有フォルダ（Zドライブ）』を、そのままクラウドに置いたもの」とイメージしてください。

② Amazon FSx for Lustre （ラスター）

科学技術計算、動画のレンダリング、機械学習など、ミリ秒以下のレイテンシーが求められる世界で使います。
「S3にあるデータを、使う時だけ超高速なLustreに読み込んで、計算が終わったらまたS3に戻す」という連携機能が強力です。

3. EFS と FSx for Windows の使い分け（鉄板）

試験で「ファイルストレージを選べ」と言われたら、「どのOSから使うか？」を見て即決してOKです。

使うOS (クライアント) 選ぶべきサービス プロトコル (通信語)
Linux のサーバー Amazon EFS NFS
Windows のサーバー Amazon FSx for Windows SMB
計算用スパコン (Linux) Amazon FSx for Lustre -

まとめ

  • FSx: 特殊なニーズ（Windowsやスパコン）に対応したファイルサーバー。
  • FSx for Windows: Windows環境のファイル共有ならこれ一択。SMB と AD連携 がキーワード。
  • FSx for Lustre: HPC（科学計算）や機械学習で、爆速の読み書きが必要ならこれ。S3連携 が便利。

「WindowsならFSx、LinuxならEFS」と覚えておけば、基本問題は解けます！
