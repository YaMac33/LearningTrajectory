AWS CodeGuru 概要

#AWS #CodeGuru #機械学習 #コードレビュー #パフォーマンス分析

AWS CodeGuru はコードの品質と性能をAIで分析するサービスです。

AWS CodeGuru は、機械学習（ML）を使って、コードの品質問題やパフォーマンス問題を自動で見つけてくれるAWSのAIサービスです。
一言で言うと、「AIがあなたの書いたコードをレビューし、アプリの健康診断をしてくれる専属コーチ」です。

1. CodeGuru は「2つ」に分かれている（超重要）

CodeGuruには2つの異なる機能があります。試験ではここを区別して理解することが重要です。

2つの機能の違い
🧐 CodeGuru Reviewer 静的解析 (Static)
「書いたコードを読む」
実行せずに、ソースコードの書き方をチェックする。
バグやセキュリティリスクを発見。
⏱️ CodeGuru Profiler 動的解析 (Dynamic)
「動くアプリを測る」
実行中のアプリを監視する。
CPU使用率や遅延の原因を可視化。

① CodeGuru Reviewer （静的解析）

コードを実行せず、書かれたテキスト（ソースコード）として分析します。GitHubやCodeCommitと連携し、プルリクエストが出されたタイミングで自動的にチェックします。

  • 見つけるもの:
    • バグの温床（nullチェック漏れなど）
    • セキュリティの脆弱性
    • AWS SDKの非効率な使い方
  • アウトプット: 「ここの書き方は危険です」といったコメントを自動投稿します。

② CodeGuru Profiler （動的解析）

実際に稼働しているアプリケーション（本番環境やテスト環境）を分析します。エージェントを組み込んで、CPUやメモリの使用状況をリアルタイムで計測します。

  • 見つけるもの:
    • 「どのメソッド（処理）がCPUを一番食っているか」
    • 「レスポンスが遅い原因のボトルネック」
  • アウトプット: ヒートマップ（処理の重さを色で表示）などで可視化し、「この行を直せばコスト削減できます」と提案します。

2. どんな時に使う？

利用シーン
🤖

自動コードレビュー

シニアエンジニアが忙しくてレビューできない時、AIが一次チェックを代行する。

🔥

パフォーマンス改善

「なんかアプリが重いけど原因がわからない」という時に、重い処理を特定する。

💰

コスト削減

無駄にCPUを使っている処理を見つけて直し、EC2のサイズを下げる。

3. 試験対策キーワードと注意点

AWS試験では、他の監視サービスとの違いが問われます。

サービス 対象 役割
AWS CodeGuru ソースコード / アプリ内部 コード品質向上、性能改善の「提案」
Amazon CloudWatch インフラ / ログ 死活監視、メトリクス収集、アラート
AWS X-Ray リクエスト経路 分散トレーシング（どこで止まったか追跡）

よくあるひっかけ ❌

  • ❌「CodeGuruはコードを自動修正する」 → 提案するだけです。直すのは人間です。
  • ❌「CodeGuruはサーバーのダウンを検知する」 → それは CloudWatch の仕事です。

まとめ

  • CodeGuru: 機械学習で開発者を支援するAIサービス。
  • Reviewer: 「コードを見て」バグを見つける（静的）。
  • Profiler: 「動作を見て」遅い場所を見つける（動的）。

「AIの力で、より良いコードを書き、より速いアプリを作るためのツール」と覚えておけばOKです！
