APIの種類と特徴

#API #REST #HTTP #WebSocket #AWS #開発

APIの種類（REST、HTTP、WebSocketなど）とその特徴を解説し、用途を紹介します。

APIの種類について理解することは、システム設計やAWS認定試験において非常に重要です。
特に、「REST、HTTP、WebSocket」の3つは、Amazon API Gateway で作成できるAPIのタイプそのものでもあります。
これらを中心に、現代の開発で使われる主要なAPI形式を詳しく解説します。

1. 主要な3つのAPI（API Gateway対応）

AWSでも中心となる3つのAPIタイプを、身近な「注文」や「会話」に例えて比較します。

① REST API (Representational State Transfer)

「Web APIの王様」です。世界中で最も普及しています。
「リソース（データ）」に対して、HTTPメソッド（GET, POSTなど）を使って操作します。

  • 特徴: ルールが明確で、ステートレス（サーバーは前のやり取りを覚えない）。
  • AWSでの位置づけ: API Gatewayの「REST API」は、WAF連携やAPIキー管理などを含む「高機能・フルスペック版」です。

② HTTP API

「REST APIの『軽量・高速・安価』版」です。
技術的にはRESTの一種ですが、AWSにおいては「機能を削ぎ落として、安さと速さを追求したもの」を指します。

  • 特徴: REST APIに比べて料金が約70%も安く、レイテンシー（反応速度）が低いです。ただしWAFが使えないなどの制限があります。

③ WebSocket API

「リアルタイムの双方向通信」です。
これまでの2つは「手紙（一方通行）」でしたが、これは「電話」です。

通信の流れの違い

2. その他、知っておくべき重要なAPI

AWSサービスとも深く関わる、その他のAPI形式です。

その他のプロトコル
🍽️
GraphQL (グラフQL) 「ビュッフェスタイル」
欲しいデータだけを指定して取る。「名前だけ欲しい」と言えばそれだけ返ってくる。
(AWS AppSync)
📜
SOAP 「封蝋付きの公式書簡」
XMLを使う昔ながらの厳格なプロトコル。銀行やレガシーシステムで現役。
🚄
gRPC 「熟練スタッフの暗号会話」
バイナリ形式で爆速通信。主にサーバー同士の裏側の会話（マイクロサービス）に使われる。

3. API Gateway における比較まとめ

試験対策として、特に重要な最初の3つを整理します。

種類 特徴 料金 典型的なユースケース
REST API 高機能・全部入り 高め 外部公開API、WAFが必要な場合、複雑な管理が必要な場合
HTTP API シンプル・低価格 安い Lambdaへのプロキシ、コスト重視、WAF不要な場合
WebSocket 双方向・リアルタイム 接続時間課金 チャット、通知、ダッシュボード

まとめ

  • 「コストを抑えてLambdaを公開したい」なら HTTP API。
  • 「チャットアプリを作りたい」なら WebSocket API。
  • 「多機能な管理が必要」なら REST API。

用途に合わせて「レストラン」か「ファストフード」か「電話」かを選びましょう！
