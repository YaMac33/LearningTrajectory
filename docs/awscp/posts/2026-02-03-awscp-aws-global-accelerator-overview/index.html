AWSグローバルアクセラレーターの概要

#AWS #グローバルアクセラレーター #通信 #高速化 #クラウド

AWSの専用ネットワークで通信を加速・安定化するサービスの解説。

AWS Global Accelerator（グローバルアクセラレーター）は、世界中にいるユーザーが、あなたのアプリ（AWS上のサーバー）にアクセスする際、「インターネットという荒れた公道」を使わず、「AWS専用の高速道路」を通らせることで、通信を爆速かつ安定させるサービスです。
一言で言うと、「国境を越える通信のための『どこでもドア（専用トンネル）』」です。

1. 抱えている課題と解決策

通常、海外のユーザーが日本のサーバーにアクセスしようとすると、インターネットという「公道」を通ります。しかし、公道は混雑したり、遠回りしたりと不安定です。

通信経路の違い
🚦
インターネット (公道)
信号あり、渋滞あり、遠回り。
結果：「遅い」「切れる」「反応が悪い」
🚀
AWS Global Accelerator (専用道)
最寄りの入口からAWSバックボーンへ直行。
結果：「速い」「安定」「パケットロスなし」

Global Accelerator を使うと、ユーザーは「一番近くのAWSエッジロケーション（入口）」から、AWSの敷地内に入ります。そこから先は、AWSが地球規模で張り巡らせている「専用の光ファイバー網」を通って、サーバーまで一直線に進みます。

2. 試験に出る「3つの最強機能」

① 固定IPアドレス（Static IP）がもらえる

これを使うと、2つの静的なIPアドレス（Anycast IP）が提供されます。世界中のどこにいても、ユーザーはこのIPアドレスにアクセスするだけです。
バックエンド（EC2やALB）のIPが変わっても、ユーザー側の設定を変える必要がありません。

② 瞬時のフェイルオーバー（障害切り替え）

もし「東京リージョンのサーバー」がダウンしたら、Global Acceleratorはそれを瞬時に検知し、ユーザーの通信を自動的に「大阪リージョン」や「米国リージョン」へ切り替えます。
DNS切り替え（Route 53）よりも圧倒的に早い（数秒以内）のが特徴です。

③ TCP / UDP プロトコルに対応

Webサイト（HTTP/HTTPS）だけでなく、ゲーム（UDP）やIoT（MQTT）、ボイスチャット（VoIP）など、あらゆる通信を高速化できます。

3. CloudFront との違い（ここが一番の山場！）

「世界中に配信して高速化する」というと、Amazon CloudFront（CDN）と何が違うの？と必ず混乱します。
明確な使い分けの基準は、「キャッシュ（コピー）できるか？」です。

高速化のアプローチ比較
🏪
Amazon CloudFront
「近所のコンビニに置く」

キャッシュ（コピー）を近くに配る。
画像、動画、CSSなど。
誰が見ても同じもの向け。
🚅
Global Accelerator
「新幹線で直通させる」

道のり（経路）を最適化する。
ゲーム、株価、チャットなど。
リアルタイムで変わるもの向け。
特徴 Amazon CloudFront AWS Global Accelerator
仕組み キャッシュ (コピー) 経路最適化 (トンネル)
得意なデータ 静的コンテンツ
(画像, 動画, Webページ)		 動的コンテンツ
(ゲーム位置情報, IP電話)
プロトコル HTTP / HTTPS メイン TCP / UDP (何でもOK)

まとめ

  • 役割: インターネット（公道）を避けて、AWSの専用線（高速道路）を使って通信を安定・高速化させる。
  • 機能: 世界共通の固定IPを提供し、障害時は瞬時に別のリージョンへ逃がす。
  • 使い分け:
    • キャッシュしたいなら CloudFront。
    • キャッシュできない（UDPや動的通信）なら Global Accelerator。

試験で「IPアドレスを固定したい」「キャッシュできない動的な通信を高速化したい」「UDPを使いたい」というキーワードが出たら、迷わず Global Accelerator を選んでください！
