クラウドの費用モデル比較

#クラウド #AWS #CapEx #OpEx #PAYG

CapEx、OpEx、PAYGの違いとAWSの利点を解説します。

CapEx（キャペックス）とOpEx（オペックス）。この2つの用語は、「クラウドを使うと、お金の使い方がどう変わるのか？」を理解するための最重要キーワードです。
一言で言うと、「家を買う（CapEx）」か、「ホテルに泊まる（OpEx）」かの違いです。

1. CapEx (Capital Expenditure) = 設備投資

「最初にドカンと払うお金（固定費）」のことです。

  • オンプレミス（自社運用）の場合:
    • サーバー機器、ルーター、エアコン、データセンターの建設費などを、システムを使い始める前に多額の現金を用意して「購入」する必要があります。
  • デメリット:
    • キャッシュフロー悪化: 最初に大金が出ていく。
    • リスク: 「予想が外れてシステムを使わなかった」場合でも、買ったサーバーは返品できず大損になります。

2. OpEx (Operating Expenditure) = 運用費

「毎月ちょこちょこ払うお金（変動費）」のことです。

  • クラウド（AWS）の場合:
    • 初期費用は不要（または極小）で、毎月の請求書払いになります。
  • メリット:
    • 手軽: すぐ始められる。
    • 低リスク: 嫌になったら解約すれば、その時点から支払いは止まります。「辞められる」というのは最強のリスクヘッジです。

3. PAYG (Pay-As-You-Go) = 従量課金

OpEx（運用費）を実現するための「AWSの支払いルール」のことです。「使った分だけ払う」という仕組みです。

  • イメージ: 「電気代」や「水道代」、「タクシーのメーター」。
  • AWSの場合: EC2を1時間動かしたら1時間分払う。止めたら支払いはゼロになる。これが PAYG です。

【試験に出る！】 クラウドのメリット

AWSの試験では、以下のフレーズが正解として頻出します。

「CapEx（設備投資）を OpEx（変動費）に転換する」
(Trade capital expense for variable expense)

これはつまり、「最初にサーバーを買う『固定費のギャンブル』をやめて、使った分だけ払う『経費』に変えることで、会社の資金繰りを良くしましょう」という意味です。

まとめ

用語 日本語 たとえ AWSとの関係
CapEx 設備投資 家を買う オンプレミス（昔）のやり方。
OpEx 運用費 ホテルに泊まる AWS（クラウド）のやり方。
PAYG 従量課金 電気代 OpExを実現する課金モデル。

「AWSにすると、CapEx がなくなって OpEx になる（借金して家を買わなくて済む）」と覚えておけばバッチリです！
