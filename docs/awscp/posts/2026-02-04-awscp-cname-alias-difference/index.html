CNAMEとエイリアスレコードの違い

#CNAME #エイリアス #AWS #Route 53 #DNS

CNAMEの特徴と弱点、エイリアスレコードを活用したAWSでの転送方法を解説します。

CNAME（シーネーム）とは、一言で言うと「ドメインの『転送』や『あだ名』」のことです。
「www.example.com にアクセスが来たら、実は google.com のことだから、そっちに案内して！」というように、ドメイン名を別のドメイン名に転送する仕組みです。

しかし、AWS試験（特にRoute 53の問題）において、このCNAMEには「致命的な弱点」があります。それを補うのが、AWS独自の「エイリアスレコード」です。詳しく解説します。

1. CNAME（Canonical Name）の仕組み

「あだ名（別名）を、本名に転送する」レコードです。

名前解決の流れ
blog.mysite.com
(あだ名)
➡ 転送 ➡
hatenablog.com
(本名 / 転送先)
  • 特徴: IPアドレス（数字）ではなく、ドメイン名（文字）に紐付けます。
  • CNAMEの「致命的な弱点」:
    • 実は、CNAMEは「ドメインの根っこ（Zone Apex / Naked Domain）」には設定できないというインターネット界の絶対的なルールがあります。
    • ○ www.example.com （サブドメイン）にはCNAMEを使える。
    • × example.com （ルートドメイン）にはCNAMEを使えない。

これが困るのです。「example.com と打ったら CloudFront（d123...cloudfront.net）に転送したい」と思っても、CNAMEではルール違反で設定できません。

2. そこで登場、「エイリアスレコード」

Route 53には、このCNAMEの弱点を克服した「AWS専用のスーパーCNAME」があります。それがエイリアスレコードです。
CloudFrontやELB（ロードバランサー）への紐付けには、CNAMEではなくエイリアスレコードを使うのが鉄則です。理由は2つあります。

ルートドメインへの対応
CNAME レコード

標準的なDNSの機能

example.com (ルート)
⬇
× 設定不可
エイリアスレコード

AWS Route 53 独自機能

example.com (ルート)
⬇
◎ 設定可能！

理由①：ルートドメイン（Zone Apex）でも使える

CNAMEでは禁止されていた「example.com → CloudFront」という転送が、エイリアスレコードなら設定可能です。これはAWSが裏側でうまく処理してくれているからです。

理由②：無料でお得

  • CNAMEの場合: 名前解決のたびにRoute 53のクエリ料金がかかります。
  • エイリアスの場合: AWSのリソース（CloudFront, ELB, S3など）に向けたクエリは、なんと無料になります。

3. 試験対策まとめ

試験で「CloudFront」や「ELB」が出てきたら、反射的に以下の表を思い出してください。

特徴 CNAME レコード エイリアスレコード (Alias)
仕組み 標準的なDNSの機能 AWS Route 53 独自の機能
ルートドメイン(example.com) 設定不可 (×) 設定可能 (◎)
料金 有料 AWSリソース宛なら無料
CloudFrontへの紐付け 使えるけど、非推奨 絶対こっちを使う (推奨)

まとめ

「AWSを使うなら、CNAMEの上位互換であるエイリアスを使いなさい（安いし便利だから）」というのが正解です。
