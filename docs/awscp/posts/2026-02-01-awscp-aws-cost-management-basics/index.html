AWSコスト管理の基本

  • AWS
  • コスト管理
  • Budgets
  • Cost Explorer
  • クラウド

クラウドは「使った分だけ払う（従量課金）」のがメリットですが、裏を返せば「使いすぎても誰も止めてくれない」という怖さがあります。試験では、「AWS Budgets」と「Cost Explorer」の役割の違いが必ず問われます。それぞれの役割を一言で表すと以下のようになります。

🚨 AWS Budgets

「アラーム（警告）」
予算を決めて、超えそうなら知らせる。
守りのツール。

📊 Cost Explorer

「家計簿（分析）」
過去の支出をグラフで見て、無駄を見つける。
分析のツール。

1. AWS Budgets（バジェット）：予算管理とアラート

「今月は1万円までに抑えたい」という目標（予算）を設定し、それを超えそうになったら通知してくれるサービスです。

  • 予算の設定： 「月額 100ドルまで」といった予算を作成します。
  • アラート通知：
    • 実績ベース： 「実際に80ドル（80%）に達したらメールする」
    • 予測ベース： 「このペースだと月末に120ドルになりそうだよ（予測）」という段階でもメールを飛ばせます。
  • アクション（応用）： 予算を超えたら、自動的にEC2を停止させる（ChatbotやLambdaと連携）ことも可能です。

イメージ： クレジットカードの「使いすぎ防止アラート」。

2. AWS Cost Explorer（コストエクスプローラー）：可視化と分析

AWSの請求データを、棒グラフや折れ線グラフで「見える化」するツールです。「先月、なんでこんなに高かったんだ？」と原因を突き止める時に使います。

  • グラフ化： 「サービス別（EC2でいくら、RDSでいくら）」や「日別」のグラフを表示します。
  • フィルタリング： 「東京リージョンの、本番環境のタグがついているEC2だけ」といった細かい絞り込みができます。
  • 予測（Forecast）： 過去のデータに基づいて、「来月はこれくらいかかりそうです」という未来の予測グラフを出してくれます（最大12ヶ月先まで）。

イメージ： 詳細な「家計簿アプリ」や「クレジットカードの明細分析画面」。

3. 【最重要】Budgets と Cost Explorer の違い

試験で「どっちを使う？」と迷ったら、この判断基準を思い出してください。

目的 使うサービス 判断基準
「予算を超えたら知らせてほしい」 AWS Budgets アクション (通知) が欲しい時
「なぜ高いのか原因を知りたい」 Cost Explorer 分析・確認 がしたい時
「来月の請求額を予測したい」 Cost Explorer 傾向分析からの予測
「予算オーバーしそうか予測したい」 AWS Budgets 予算に対するアラート

4. コスト配分タグ (Cost Allocation Tags)

「AWSの請求書を見たら、EC2代が100万円だった。でも、どの部署がいくら使ったか分からない！」
こんな時に使うのが「タグ」です。

EC2 A Dept: Sales
EC2 B Dept: Dev
RDS Dept: Sales
⬇ フィルター集計
営業部 (Sales) のコスト： ¥700,000
  • 仕組み： EC2などのリソースに Department: Sales や Project: Website といったタグ（名札）を付けておきます。
  • 効果： Cost Explorerなどで「Sales部署だけのコスト」を抽出できるようになります。
  • 試験対策： 「部門ごと、プロジェクトごとにコストを分類するには？」→ コスト配分タグを有効化する。

5. その他のコスト関連サービス

選択肢に混ざってくる脇役たちです。

AWS Cost and Usage Report (CUR)

  • 役割： 「最も詳細な」生の請求データ（CSVファイル）です。
  • 用途： Cost Explorerよりもさらに細かく、1時間ごとの明細をExcelやBIツールで分析したい時に使います。
  • 試験対策： 「最も詳細なデータ」「生データ」ときたらこれ。

AWS Pricing Calculator

  • 役割： 「見積もり」ツール。
  • 用途： AWSを使う前に、「この構成だといくらになるかな？」と試算するために使います（昔のSimple Monthly Calculatorの後継）。

6. まとめ

  • AWS Budgets： 「守り」。予算を決めて、超えたらアラート（通知）。
  • Cost Explorer： 「分析」。グラフで見て、原因特定や将来予測。
  • コスト配分タグ： 請求を「部署ごと」「プロジェクトごと」に分けるための名札。
