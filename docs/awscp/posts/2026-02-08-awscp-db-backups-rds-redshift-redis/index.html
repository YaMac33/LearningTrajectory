AWSデータベースサービスのバックアップ

tags: #AWS #バックアップ #RDS #Redshift #ElastiCache #試験対策

RDS、Redshift、ElastiCache（Redis）の自動バックアップ機能を整理。CLF-C02（AWS Certified Cloud Practitioner）目線で、保持期間やPITRの可否、試験でのひっかけ対策までを短くまとめます。

補足メモ：原稿時刻 2026-02-07 15:07

結論（試験での覚え方）

  • RDS・Redshift・ElastiCache（Redis）は「自動バックアップあり」。ただし性質と前提が異なる。
  • Memcachedはバックアップ不可（ElastiCache＝常にバックアップ可、ではない）。
試験では「自動バックアップの有無」と「誰が管理するか（マネージドか）」を問われやすい。

バックアップの全体像（図解）

【サービス別の自動バックアップのイメージ】
RDS
  ├─[日次スナップショット]
  └─[トランザクションログ継続取得] ──→ 目的時点へ復元（PITR）
                      ↑ 保持1〜35日

Redshift
  └─[自動スナップショット→S3保存] ──→ スナップショット時点へ復元
                      ↑ 既定1日（変更可）

ElastiCache（Redis）
  └─[スナップショット（RDB）] ──→ 取得時点へ復元
ElastiCache（Memcached）
  └─[バックアップ機能なし]（再起動で揮発）

各サービスの扱い（CLF-C02準拠）

Amazon RDS

  • 自動バックアップ：あり（デフォルト有効）
  • 方式：トランザクションログ＋日次スナップショット
  • 保持期間：1〜35日（デフォルト7日、変更可）
  • PITR（ポイントインタイムリカバリ）：可能

試験ポイント

  • 「RDSは自動バックアップされる」は頻出。利用者はOSやバックアップ設計を個別実装しない＝マネージド。
  • PITRが可能であることを押さえる。

Amazon Redshift

  • 自動バックアップ：あり
  • 方式：S3へ自動スナップショット保存
  • 保持：既定1日（延長可、リージョン間コピーや手動スナップショットも可）
  • 用途：分析（OLAP）向けDWH

試験ポイント

  • Redshiftもマネージド。自動スナップショットがある。
  • 「分析用DWH＋自動バックアップ」で記憶。

Amazon ElastiCache（Redisのみ）

  • 自動バックアップ：あり（Redisのみ。Memcachedは不可）
  • 方式：スナップショット（RDB）を指定ウィンドウで取得
  • 復元：スナップショット時点に復元（PITRは不可）

試験ポイント（超重要）

  • 「ElastiCache＝常にバックアップ可能」は誤り。Redisのみ可能、Memcachedは不可。

まとめ表（試験直前チェック用）

サービス 自動バックアップ 補足
RDS ⭕ PITRあり（1〜35日保持）
Redshift ⭕ S3に自動スナップショット（既定1日、変更可）
ElastiCache（Redis） ⭕ スナップショット取得。Memcachedは対象外
ElastiCache（Memcached） ❌ 揮発性。再起動で消える設計

ひっかけ対策

  • 「ElastiCacheはバックアップできる」→ Redis限定。Memcachedは不可。
  • 「インメモリDB＝バックアップ不可」→ Redisは可能（RDBスナップショット）。
  • 「RDSは手動でバックアップ設計が必要」→ デフォルトで自動バックアップ有効。

運用の実務ヒント

  • RDS：保持期間は復旧要件（RTO/RPO）とコストで決定。重要DBは手動スナップショットのライフサイクル管理も併用。
  • Redshift：スナップショットのリージョン間コピーで災害対策。負荷の低い時間帯にウィンドウを合わせる。
  • Redis：スナップショット頻度と性能のバランスに注意。永続化要件が強い場合はRDS等の永続DBと役割分担。
# 例：RDSの自動バックアップ保持期間を14日に変更（AWS CLI）
aws rds modify-db-instance \
  --db-instance-identifier mydb \
  --backup-retention-period 14 \
  --apply-immediately

# 例：ElastiCache（Redis）スナップショット自動取得の有効化（例）
aws elasticache modify-replication-group \
  --replication-group-id my-redis \
  --snapshotting-cluster-id my-redis-0001-001 \
  --snapshot-window 03:00-04:00 \
  --snapshot-retention-limit 7
