AWS WAFの基礎知識

  • AWS
  • WAF
  • セキュリティ
  • 攻撃防止
  • Web ACL

AWSのセキュリティを支える強力な番人、AWS WAF（ワフ）について解説します。これも試験（CLF-C02）では「どのサービスがどの攻撃を防ぐか」という区別で非常によく出題されます。

1. AWS WAFとは？

WAFは Web Application Firewall の略です。一言で言うと、「Webサイト（アプリ）に届く悪意のある通信を、ルールに基づいてブロックするフィルター」です。一般的なファイアウォール（ネットワークの門番）が「誰が通るか（IPアドレス）」をチェックするのに対し、WAFは「どんな内容の通信を送ってきたか（悪いメッセージが書かれていないか）」という中身までチェックします。

🚪 普通のファイアウォール
(セキュリティグループ等)
「オートロック」
不審な人物（怪しいIPアドレス）を入り口でシャットアウトする。
チェック対象：送信元IP、ポート番号
👮‍♂️ AWS WAF
(Web Application Firewall)
「手荷物検査員」
住人のような顔をしていても、カバンの中に爆弾（悪意のあるコード）を持っていないか中身を検査する。
チェック対象：通信の中身 (URL, Body)

2. WAFが防いでくれる主な攻撃

試験によく出る「Web特有の攻撃」の名前を2つセットで覚えましょう。

  • SQLインジェクション： 入力フォームなどに変な命令を書き込んで、データベースの情報を盗み出す攻撃。
  • クロスサイトスクリプティング（XSS）： サイトに罠を仕掛けて、他のユーザーの情報を奪う攻撃。

3. AWS WAFの仕組み：Web ACLとルール

WAFを使うときは、「Web ACL（アクセスコントロールリスト）」というルールのリストを作ります。このリストに合致した通信を「通す（Allow）」か「弾く（Block）」か決めます。

😈 攻撃者
「変なSQLコードを送ってやる！」
↓ 攻撃通信
✅ ルール1: 日本からのアクセス → 許可
❌ ルール2: "SELECT * FROM" を含む → 拒否
❌ ルール3: 悪質IPリスト → 拒否
Block!
↓ 正常な通信のみ通過
保護対象のサービス
CloudFront / ALB / API Gateway
  • ホワイトリスト： 「この人（日本国内からのアクセスなど）だけ通してOK！」
  • ブラックリスト： 「この国からのアクセスや、この変な文字列が含まれる通信は拒否！」
  • マネージドルール： AWSの専門家が作った「これを入れておけば安心」というセットメニュー。初心者はこれを選ぶだけで強力な防御ができます。

4. 実務イメージ：企業のAWS利用シーン

ある通販サイト（ECサイト）を運営している会社を想像してください。

  • 悩み： 海外からの不審なアクセスが急増し、偽の注文フォームを送りつけられる攻撃に遭っている。
  • 解決策： AWS WAFを導入。「特定の海外IPからのアクセスを遮断」し、さらに「フォームにプログラムコードを書き込むような怪しい通信」を自動でブロックする設定にします。
  • 結果： 本物の顧客だけがスムーズに買い物できるようになり、管理者は安心して夜眠れるようになります。

5. 試験で問われやすいポイント！

WAFと他のセキュリティサービスを混同しないようにしましょう。

特徴 AWS WAF
守るレイヤー 第7層（アプリケーション層）
HTTP/HTTPSの中身を見る
関連付ける先 CloudFront, ALB, API Gateway, AppSync
キーワード SQLインジェクション、XSS、Web ACL、IP制限

「ネットワーク層（第3/4層）での攻撃」を防ぐのは AWS Shield や Network Firewall の役割です。WAFはあくまで「Webアプリの中身」を守る番人です。
