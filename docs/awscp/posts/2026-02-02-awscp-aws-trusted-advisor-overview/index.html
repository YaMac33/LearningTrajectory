AWS Trusted Advisorの全貌

  • AWS
  • Trusted Advisor
  • セキュリティ
  • コスト最適化
  • パフォーマンス

AWS Trusted Advisor（トラステッド・アドバイザー）は、あなた専属の「敏腕コンサルタント」です。AWS環境を自動的にスキャンして、「お金を使いすぎていないか？」「セキュリティは甘くないか？」などのチェックを行い、「こう直したほうがいいですよ」とアドバイス（推奨事項）をくれるサービスです。

先ほどの「AWS Health Dashboard」が「運行情報（外の状況）」なら、Trusted Advisorは「家計・防犯の見直し（家の状況）」をしてくれる存在です。

1. Trusted Advisor の「5つのチェック柱」

試験では、以下の5つのカテゴリのどれに関するアドバイスをくれるかが問われます。

💰 ① コスト最適化
・停止中のEC2
・使ってないEBS
・アイドリング状態のDB
🛡️ ② セキュリティ
・ルートユーザーのMFA
・ポート22全開放
・S3の公開設定
🧩 ③ 耐障害性
・EBSスナップショット
・ELB配下の台数不足
・マルチAZの設定
🚀 ④ パフォーマンス
・Route 53の最適化
・EC2のCPU張り付き
・スループット不足
🛑 ⑤ サービス制限
・VPC作成数の上限
・EC2起動数の上限
(パンク防止)

2. サポートプランによる「格差」（試験のひっかけ）

ここが試験で非常に重要です。Trusted Advisorは全機能を使うには、「お金（サポートプラン）」が必要です。

ベーシック / 開発者プラン
制限あり

以下の2カテゴリのみチェック可能：
1. コアセキュリティチェック (7項目)
2. サービス制限

※「コスト最適化」などは見られません！

ビジネス / エンタープライズ
フル機能 (All Checks)

5つの柱すべてのチェックが可能。
コスト削減やパフォーマンス向上のアドバイスを受けたいならこちら。

※CloudWatchアラームとの連携なども可能。

  • 試験対策： 「コスト削減のアドバイスをTrusted Advisorから受けたい」という要件がある場合、ビジネスプラン以上の契約が必要です。

3. 他のサービスとの「違い」総まとめ

似たような「チェック・監視」系サービスとの違いを整理します。これが分かれば迷いません。

サービス名 役割 (メタファー) 見ているもの
Trusted Advisor コンサルタント 「設定ミス」や「無駄遣い」
(ベストプラクティスとの乖離)
Inspector 脆弱性スキャナ 「OSの中身」や「ネットワークの穴」
(CVEなどの脆弱性)
CloudWatch 健康診断医 「今の数値」
(CPU使用率、エラーログ)
Health Dashboard 運行情報板 「AWS側の障害」
(物理メンテナンス情報)

具体的な違いの例：

  • 「ポート22（SSH）が世界中に開けっ放しだよ」と設定を注意するのは Trusted Advisor。
  • 「OSのセキュリティパッチが当たってないよ」と中身を見るのは Inspector。
  • 「今、CPUが100%だよ」と数値を出すのは CloudWatch。

4. まとめ

  • 役割： AWS環境が「ベストプラクティス（AWS推奨の設定）」になっているかを自動チェックする。
  • 5つの柱： コスト、セキュリティ、耐障害性、パフォーマンス、サービス制限。
  • プラン： 全機能（特にコスト系）を使うには、ビジネスプラン以上が必要。
  • セキュリティ： 無料プランでも「ルートユーザーのMFAチェック」などのコアチェックは使える。
