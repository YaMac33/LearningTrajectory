AWS IoT Greengrassの概要

#AWS #IoT #エッジコンピューティング #Greengrass #クラウド

AWS IoT Greengrassは、エッジでの処理を実現するIoTソフトウェアです。

AWS IoT Greengrass（アイオーティー・グリーングラス）は、一言で言うと「クラウドの頭脳を、現場のデバイス（エッジ）に持ち込むためのソフトウェア」です。
通常、IoT（モノのインターネット）は「データをクラウドに送って、クラウドで考えて、命令を返す」のが基本ですが、Greengrassを使うと「デバイス自身がその場で考えて判断する」ことができるようになります。これを「エッジコンピューティング」と呼びます。

1. なぜこれが必要なの？（抱えている課題）

例えば、工場のベルトコンベアで不良品を見分けるロボットアームを想像してください。すべてをクラウド（AWS）で処理しようとすると、問題が起きます。

  • 課題A（遅延）: カメラ映像を全部クラウドに送って判定していたら、「あ、不良品だ！」と気づいた時には、もう商品は通り過ぎてしまっています（通信のタイムラグ）。
  • 課題B（回線切れ）: もし工場のWi-Fiが切れたら、ロボットが止まってしまい、ライン全体が停止します。

2. Greengrass の解決策：「現場監督を置く」

Greengrass をインストールしたデバイス（ゲートウェイ）は、クラウドの出張所（ミニ・クラウド）のような働きをします。

司令塔と現場監督の関係
☁️ クラウド (本社) 🏢
AWS IoT Core
司令塔
全体のデータ収集・分析
間欠的 接続OK ⬍
🏭 エッジ (現場) 👷‍♂️
Greengrass
現場監督
その場で即判断

※ ネットが切れても「現場」だけで動き続けられる！

  • その場で処理: AWS Lambda（プログラム）をデバイスの中で動かせます。「不良品だ！」という判断を0.01秒で下せます。
  • オフライン対応: インターネットが切れても、手元のデバイス同士で通信し、動き続けます。ネットが復旧したら、溜まっていたデータをクラウドに同期します。

3. 具体的に何ができるの？

Greengrassの3大機能
⚡
ローカル Lambda
クラウドで作ったプログラムを配信し、デバイス上で実行できる。
📡
デバイス間通信
ネットがなくても、センサーとライトがGreengrass経由で会話できる。
🧠
機械学習 (ML)
クラウドで鍛えたAIモデルを持ち込み、カメラ映像の解析などを現場で行う。

4. 試験対策キーワード

試験で以下の状況が出たら、迷わず AWS IoT Greengrass を選んでください。

  • 「インターネット接続が断続的（不安定）な環境でも、デバイスを稼働させたい」
  • 「データをクラウドに送る前に、ローカル（エッジ）で処理・フィルタリングしたい」
  • 「ミリ秒単位の超低遅延で反応する必要がある」
