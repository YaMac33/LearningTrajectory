EC2のインスタンスファミリーとテナンシー

#AWS #EC2 #インスタンス #テナンシー #試験対策

インスタンスファミリーはEC2の用途別分類で、テナンシーは物理サーバーの共有方法を示します。

EC2を選ぶときに必ず出てくる「ファミリー」と「テナンシー」。一見難しそうですが、要するに「車の種類（ファミリー）」と「住み方（テナンシー）」の話です。

  • インスタンスファミリー: EC2の「用途・性能タイプ」の分類。（スポーツカーか、トラックか）
  • テナンシー: EC2が「誰と物理サーバーを共有するか」の設定。（シェアハウスか、一戸建てか）

1. インスタンスファミリーとは？

AWSは用途ごとに最適なEC2を用意しており、それをアルファベットで分類しています。

主なインスタンスファミリー
M, T ⚖️ 汎用
バランス型。
Webサーバーや開発環境に最適。
C 🧠 コンピューティング
CPU強化型。
バッチ処理や科学計算向け。
R, X 💾 メモリ最適化
メモリ大量。
データベースやインメモリ処理向け。
I, D 🗄️ ストレージ最適化
高速I/O。
ビッグデータ分析やNoSQL向け。
P, G 🎮 高速計算 (GPU)
グラフィック・AI。
機械学習や動画処理向け。

試験では「用途に合ったファミリーを選べるか」が問われます。「DBならメモリ最適化」「Webサーバーなら汎用」が定石です。

2. テナンシーとは？

EC2を動かすための「物理サーバー」を、他のAWSユーザーとどう分け合うかという設定です。

住み方（物理サーバーの使い方）の違い
🏢 共有 (Shared)
「マンション」
他の住人（他社）とビルを共有。
基本はこれ。一番安い。
🏠 専有 (Dedicated)
「一戸建て（貸家）」
自分専用の物理サーバー。
他人は入ってこない。
コンプライアンス要件に対応。
🏰 専有ホスト
「土地ごと購入」
物理サーバーを丸ごと確保。
BYOL（ライセンス持ち込み）用。
  • 共有 (Shared): デフォルト設定。最も安く、一般的です。セキュリティ機能（VPC等）で守られているので、共有だからといってデータが漏れることはありません。
  • 専有 (Dedicated Instance): 「物理的に他社と同じサーバーになることを禁止する」設定です。厳しいコンプライアンス要件がある場合に使います。
  • 専有ホスト (Dedicated Host): 物理サーバーを一台丸ごと予約します。「CPUソケット単位のライセンス」を持ち込みたい場合（Windows Serverなど）に使います。

3. RI（リザーブドインスタンス）との関係

先ほどのRIの話題と繋がりますが、RI購入時にはこれらを指定する必要があります。

  • スタンダードRI: 一度決めたら、「ファミリー」も「テナンシー」も変更できません。
  • コンバーティブルRI: 後から変更可能です。

よくある誤解 ❌

  • ❌「テナンシー＝セキュリティ機能」 → いいえ、あくまで物理配置の話です。共有でも安全です。
  • ❌「インスタンスファミリー＝サイズ」 → ファミリーは「m5」などの「m」の部分です。サイズは「large」などの部分です。

まとめ

この2文だけ覚えておけば試験はバッチリです。

  • インスタンスファミリー ＝ EC2の「用途別分類（得意分野）」。
  • テナンシー ＝ EC2を配置する「物理サーバーの共有方法」。
