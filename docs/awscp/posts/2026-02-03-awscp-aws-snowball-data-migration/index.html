AWS Snowballの概要と利点

#AWS #データ移行 #Snowball #クラウド #試験対策

AWS Snowball（スノーボール） は、一言でいうと「大量のデータを、インターネットを使わずにAWSへ運ぶための『物理的な箱（ハードウェア）』」です。

Cloud Practitioner試験（CLF-C02）では、「データ移行」や「ネットワーク回線が遅い場合の解決策」として頻出のサービスです。

1. なぜ「物理的な箱」が必要なのか？

通常、AWSにデータを送るときはインターネット経由でアップロードします。しかし、データ量が「数十テラバイト〜ペタバイト」になると、回線が遅すぎて何週間もかかってしまうことがあります。

そこで、「データを箱に入れて、宅急便（トラック）で送ったほうが速い！」という発想が生まれました。

引っ越しの例え：どっちが速い？
🚲
インターネット転送
「バイク便」
少量なら速いが、
家具全部を送るには何万往復も必要。
🚚
AWS Snowball
「引っ越しトラック」
家具（大量データ）を
一度にまとめて運べる。
結果的に圧倒的に速い。

3. 試験で問われるポイント

以下のキーワードが出たら、Snowballを疑ってください。

  • ネットワークが遅い / 不安定: ネットを使わないため、回線状況に関係なく移行できます。
  • 大量データ (TB〜PB級): テラバイト、ペタバイトクラスのデータを移行したい時。
  • セキュリティ (暗号化): 物理的な盗難リスクに備え、データは強力に暗号化されます。端末自体も頑丈なケースに入っています。

4. 似ているサービスとの違い（ひっかけ注意）

「データをAWSに移す」サービスは他にもありますが、Snowballの最大の特徴は「オフライン（物理転送）」であることです。

サービス名 転送手段 おすすめの状況
AWS Snowball 物理デバイス (配送) ネット回線が遅い、データ量が莫大(TB〜PB)。
AWS DataSync インターネット / 専用線 ネット回線が高速、定期的にデータを同期したい。
Amazon S3 Transfer Acceleration インターネット (AWS独自網) 海外拠点など遠くからS3へアップロードするのを速くしたい。

試験の合言葉:
「回線が遅い」「大量データ」「ネットを使いたくない」
👉 迷わず AWS Snowball

5. ファミリー紹介 (Snow Family)

Snowballには、サイズ違いの兄弟がいます。

  • Snowcone (スノーコーン): お弁当箱サイズ（小）。ドローンに積んだり、リュックに入れて運べる。数TB向け。
  • Snowball Edge: スーツケースサイズ（中）。一番標準的。データ移行だけでなく、その場で簡単な計算（EC2実行）もできる。
  • Snowmobile (スノーモービル): 本物の巨大トラック（特大）。エクサバイト級のデータ移行用。データセンターごと引っ越すレベル。

まとめ

  • 役割: 大量データを物理的にAWSへ運ぶサービス。
  • メリット: ネットワーク回線が遅くても高速に移行できる。
  • 保存先: 主に Amazon S3 にデータが入る。
  • キーワード: 「物理デバイス」「オフライン転送」「ペタバイト規模」
