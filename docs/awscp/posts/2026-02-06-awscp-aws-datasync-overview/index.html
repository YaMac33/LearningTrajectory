AWS DataSyncの概要と特徴

#AWS #DataSync #データ転送 #高速 #同期

AWS DataSyncは高速データ転送を可能にするサービスで、定期的な同期もサポートします。

AWS DataSync は、一言で言うと「大量のデータを、インターネット（または専用線）経由で『爆速』でコピー・同期する引っ越しサービス」です。
Storage Gateway や Snowball と似ていますが、これは「オンラインで」「一度きりではなく定期的にも」「とにかく速く」データを送りたい時に使います。

1. なぜこれが必要なの？（既存ツールの限界）

オンプレミスのサーバーから、AWS（S3やEFS）へ数テラバイト〜数ペタバイトのデータを送ろうとした時、普通に cp コマンドやフリーソフトを使うと、遅すぎて終わりません。また、途中で回線が切れたらやり直しになったり、ファイルの整合性（ちゃんと送れたか）の確認が大変です。

  • DataSyncの解決策:
    • 専用プロトコル: 普通の通信（オープンソースの転送ツールなど）よりも最大10倍高速な、AWS独自の通信方式を使います。
    • 全自動: ネットワークが切れたら再接続し、送ったデータの整合性チェックまで勝手にやります。

2. 仕組み： 「エージェント」が運び屋

DataSyncを使うには、オンプレミス側に「エージェント（運び屋ソフト）」を入れる必要があります。

  • オンプレミス側: サーバーの近く（VMwareなど）に DataSyncエージェント（仮想マシン）をインストールします。
  • AWS側: マネジメントコンソールから「タスク（同期の指示）」を作ります。
  • 転送: エージェントがデータを読み取り、圧縮・暗号化して、AWSに猛スピードで投げ込みます。

3. 他のサービスとの使い分け（試験の超重要ポイント！）

「データをAWSに運ぶ」サービスは主に3つあります。試験ではこの区別が命です。

データ転送サービスの使い分け

サービス 接続方法 主な用途
AWS DataSync オンライン 数TB〜数PBのデータを、短期間で送りたい。または定期的に同期（バックアップ）したい。
Storage Gateway オンライン 転送というより、「普段使いのファイルサーバー」として繋ぎっぱなしにしたい。
Snowball オフライン ネット回線が細すぎて送れない。数週間かかってもいいから大量に運びたい。

4. 試験対策キーワード

試験で以下の文脈が出たら、AWS DataSync が正解です。

  • 「オンプレミスのストレージから、S3やEFSへ、データを『高速に』移行したい」
  • 「独自のデータ転送プロトコルを使用して、ネットワーク帯域を最大限に活用したい」
  • 「定期的なデータ転送タスクをスケジュールしたい」
  • 「転送中のデータ検証（整合性チェック）を行いたい」

まとめ

  • AWS DataSync: ネット経由の「高速データ転送ツール」。
  • メリット: 普通のコピーより10倍速い。スケジュール設定ができる。

「引っ越し用トラック（Snowball）を呼ぶほどではないけど、手作業（cpコマンド）では無理な量のデータを、ネットの土管を使って高圧洗浄機のように流し込むサービス」とイメージしてください！
