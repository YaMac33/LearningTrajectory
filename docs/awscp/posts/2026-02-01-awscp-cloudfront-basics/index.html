CloudFrontの基本とメリット

  • CloudFront
  • AWS
  • S3
  • EC2
  • キャッシュ
  • 高速化

Amazon CloudFront（クラウドフロント）は、これまでに学んだS3やEC2と組み合わせることで、Webサイトを「爆速」にするためのサービスです。一言で言うと、「世界中にばら撒かれた『キャッシュ（コピー）置き場』」です。

これは「コンビニの物流」に例えると非常に分かりやすくなります。

  • S3 (オリジン)： 「北海道にある巨大な工場」。商品はここにしかありません。
  • CloudFront (エッジ)： 「あなたの家の近所のコンビニ」。

沖縄のユーザーが北海道まで商品を買いに行くのは大変ですが、近所のコンビニに置いてあれば一瞬で手に入ります。

1. 仕組みと用語（エッジロケーション）

CloudFrontを理解するには、AWSの物理的な場所の区別が重要です。

  • リージョン： 東京、大阪など（データセンターの集まり）。世界に30箇所以上。
  • エッジロケーション： CloudFront専用のキャッシュサーバー。世界に600箇所以上（リージョンより遥かに多い！）。

2. CloudFrontの3つのメリット

試験では以下のポイントが問われます。

① レイテンシー（遅延）の低下

物理的にユーザーに近い場所からデータを返すので、表示速度が劇的に上がります。動画配信や画像が多いサイトで必須です。

② セキュリティ向上（DDoS対策）

CloudFrontが「最前線の盾」になります。攻撃者が大量のアクセスを送ってきても、世界中のエッジロケーションが分散して受け止めるため、大元のS3やEC2が守られます。AWS Shield というDDoS保護機能が標準で組み込まれています。

③ サーバー負荷の軽減

2人目以降のアクセスはエッジが返してくれるので、オリジン（EC2やS3）へのアクセスが激減します。結果としてEC2の台数を減らせたり、S3のリクエスト料金を節約できたりと、コスト削減に繋がります。

3. 試験に出る重要キーワード

CloudFrontの問題が出たら、以下の単語に注目してください。

用語 意味・試験ポイント
Origin
(オリジン)		 データの発生源。S3バケット、EC2インスタンス、ELB、あるいはオンプレミスのサーバーも指定可能。
Distribution
(ディストリビューション)		 CloudFrontの設定そのもの。「どこのオリジンを使うか」「ドメインは何にするか」などをまとめた設定単位。
TTL
(Time To Live)		 キャッシュの有効期限。これを「0」にすると毎回取りに行く。「24時間」にすると1日は古いまま。情報の鮮度に関わる。
OAI / OAC 「S3の中身をCloudFront経由でしか見せない」ようにする設定。S3のURLを直接叩いても見られないようにロックを掛ける（セキュリティ）。

4. あなたの「ブログ」との関係（HTTPS化）

GitHub PagesやS3で公開する静的ブログにおいて、CloudFrontは最強の相棒になります。

  • S3単体： 独自のドメインで「https (鍵マーク)」をつけるのが大変です。
  • S3 + CloudFront： CloudFrontを使うと、簡単に無料でSSL証明書（https）を使えます。

5. 間違いやすい「S3 Transfer Acceleration」との違い

似たような機能で、試験でひっかけとして出るものがあります。

  • CloudFront： コンテンツを「配る（下り）」ためのキャッシュ。
  • S3 Transfer Acceleration： 遠くのS3にデータを「アップロード（上り）」するのを速くする機能。エッジロケーションを使って、アップロードの近道を作ります。

6. まとめ

CloudFrontは「世界中にコピーを置くことで、速く、安全に、安くする」サービスです。これで主要な「攻め」のサービス（EC2, S3, RDS, DynamoDB, CloudFront, Route 53）はほぼ網羅しました。
