ACMの無料と自動更新の仕組み

  • AWS
  • ACM
  • SSL証明書
  • 自動更新
  • クラウド

AWS Certificate Manager (ACM) の「無料」かつ「自動更新」という特徴は、AWSを利用する最大のメリットの一つです。なぜそんなにお得で便利なのか、その裏側にある「賢い仕組み」と制約事項について解説します。

1. なぜ「発行料が無料」なの？

昔は、SSL証明書といえば年間数万円を払って購入するのが当たり前でした。ACMが無料なのは、「AWSの特定のサービス（出口）で使うこと」を条件に、Amazon自体が認証局となって証明書を配っているからです。

証明書が使える場所

AWSサービス
(ALB / CloudFront / API Gateway)
📜 ¥0 (無料)

AWS内にセットして使うならタダ！

外部サーバー
(他社サーバー / オンプレミス)
📜 持ち出し不可

証明書データのダウンロードはできません。

  • 条件： 証明書の「秘密鍵」などのデータ自体をユーザーがダウンロードすることはできません。あくまで「AWSのリソースにセットして使う」形でのみ提供されます。
  • 実質0円： スマホ本体が実質0円で配られるのと似ています。「AWSの回線（CloudFrontなど）を使ってくれるなら、証明書代はサービスします」というモデルです。

2. なぜ「自動更新」ができるの？

通常の証明書は1年ごとの更新作業が面倒ですが、ACMは「DNS検証」という仕組みを使って、これを全自動化しています。

仕組み：AWSとDNSの連携プレー

あなたが「このドメインの持ち主です」と証明するために、AWSは「合言葉」を指定します。

🔐 ACM
CNAME
合言葉
Route 53
(DNS)
✅ 合言葉を確認 ➔ 勝手に更新
  1. セットアップ： 最初に一度だけ、Route 53（DNS）に「AWS指定のCNAMEレコード（合言葉）」を登録します。
  2. 定期チェック： AWSは定期的にそのDNSを見に行きます。「あ、まだ合言葉が残ってるな。この人はまだドメインの持ち主だ」と確認します。
  3. 自動更新： 有効期限が近づくと、AWSが勝手に新しい証明書を発行し、ALBなどの裏側でこっそり差し替えます。

あなたが寝ている間に、証明書は常に最新に保たれます。

3. 試験で問われやすいポイント

CLFやSAA試験での重要ポイントです。

項目 内容
無料の範囲 パブリック証明書をAWSサービスで使う場合のみ。
更新方法 DNS検証が推奨（自動更新できるから）。
Eメール検証だと、メールのリンクをポチッと押す手動作業が必要になり、自動更新されません。
使える場所 ELB (ALB/NLB)、CloudFront、API Gatewayなど。
EC2インスタンスの中に直接インストールすることはできません。

4. 実務イメージ：導入のビフォーアフター

企業がWebサイトを立ち上げる際の手間が劇的に変わります。

  • Before（従来）：
    1. 証明書ベンダーに数万円払って申請。
    2. サーバーにログインしてファイルを配置し、Webサーバー再起動。
    3. カレンダーに「1年後に更新！」とメモする（忘れるとサイトが見られなくなる）。
  • After（ACM）：
    1. AWS画面でポチポチっとリクエスト。
    2. Route 53ならボタン一つで検証完了。
    3. 以後の作業は一生なし。

5. まとめ

  • ACM = 証明書管理の自動化ツール。
  • 無料： AWSの出口で使うならタダ。
  • 自動更新： DNS検証にしておけば、更新作業から解放される。
  • EC2には入れない： 手前のロードバランサー（ALB）等にセットする。
