RDSとは何か

  • RDS
  • AWS
  • データベース
  • リレーショナル
  • マネージド

RDS（Relational Database Service）は、AWSを利用する上で避けては通れない主要サービスの一つです。一言で言えば、「面倒なデータベース（DB）運用をAWSが肩代わりしてくれる、マネージドなリレーショナルデータベース」です。

1. RDSとは何か

  • マネージドサービス： Amazon Web Services（AWS）が提供する、リレーショナルデータベース（RDB）の管理サービスです。
  • 運用の自動化： サーバーの構築、OSやDBエンジンのパッチ適用、バックアップといった「差別化につながらない重労働」をAWSが代行します。
  • 開発への集中： アプリケーション開発者は、インフラ管理ではなく、アプリケーションロジックやDBの利用そのものに集中できます。

2. RDSで使えるDBエンジン

RDSでは、以下の6つの主要なデータベースエンジンを選択して利用できます。特にAmazon AuroraはAWS独自の技術であり、試験でも頻出です。

  • Amazon Aurora（AWS独自・高性能・高可用性）
  • MySQL
  • PostgreSQL
  • MariaDB
  • Oracle
  • SQL Server

ポイント： AuroraはMySQLやPostgreSQLと互換性を持ちながら、クラウドに最適化されたAWS史上最も成長しているサービスの一つです。

3. 「マネージド」とは何をAWSがやるのか

「マネージドサービス」といっても、全てをAWSがやってくれるわけではありません。これを理解することは、AWSの「責任共有モデル」を理解する上でも重要です。

担当 具体的な作業内容
AWSがやること
  • OSやDBエンジンのインストール
  • パッチ適用（メンテナンス）
  • 自動バックアップ
  • ハードウェア障害時の復旧
  • ホストの監視
利用者がやること
  • スキーマやテーブルの設計
  • SQLクエリの作成とチューニング
  • アプリケーションの設計
  • データそのものの管理と暗号化設定

6. RDS vs EC2上のDB（試験定番）

「EC2インスタンスの中に自分でMySQLをインストールする場合」と「RDSを使う場合」の違いは、AWS認定試験の定番問題です。

観点 RDS (マネージド) EC2上のDB (セルフ)
運用負荷 低い（AWSにお任せ） 高い（すべて自分）
パッチ適用 自動（メンテ枠で適用） 自分で実施
バックアップ 自動設定が可能 スクリプト等で自作
自由度 OSログイン不可など制限あり OSレベルで完全制御可能

基本的には「運用負荷を減らしたいならRDS」を選びます。特定のOS設定が必要な場合など、特殊な要件がある場合のみEC2を選択します。

7. RDS vs DynamoDB（混同注意）

同じくAWSの代表的なDBサービスであるDynamoDBとの使い分けも重要です。

観点 RDS DynamoDB
データ型 リレーショナル (RDB) NoSQL (Key-Value)
クエリ言語 SQL 専用API (SQL不可)
構造 厳密なスキーマあり スキーマレス
トランザクション 強力・複雑な結合が可能 限定的・高速な単純アクセス向け

業務システムの移行など、「SQLが必要」「複雑な結合が必要」な場合はRDSが適しています。

8. AWS CP試験でのひっかけポイント

Cloud Practitioner試験などでよく問われる間違いやすいポイントです。

  • ❌ RDSはサーバーレスである
    → ⭕ 基本はサーバー（インスタンス）が存在します。Aurora Serverlessという例外はありますが、RDS自体はサーバーレスではありません。
  • ❌ Multi-AZにすると処理性能が2倍になる
    → ⭕ Multi-AZは「可用性（落ちないこと）」のためであり、待機系は普段動きません。性能向上には「リードレプリカ」を使います。
  • ❌ トラブル対応でOSにSSHログインして調査する
    → ⭕ RDSはOSへのログイン権限がありません。マネージドサービスのためAWSが管理します。

9. まとめ

RDSを一言で表すなら、「DBは欲しいが、DB管理（バックアップやパッチ当て）はしたくない人のためのサービス」です。AWSでシステムを構築する際、データベースの第一選択肢となる非常に重要なサービスです。
