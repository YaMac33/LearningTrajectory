AWS静的サイトの最強構成

  • AWS
  • 静的ウェブサイト
  • CloudFront
  • S3
  • Route 53

AWSにおける静的Webサイトホスティングの「ゴールデン構成」、つまり最強の組み合わせについての解説です。この構成は、実務で「最も安く、最も速く、最も安全に」Webサイトを公開する鉄板パターンであり、AWS認定試験（CLF/SAA）でも頻出中の頻出です。

1. 役割分担：なぜ3つ必要なのか？

「S3だけでもWebサイトは公開できる（静的ウェブサイトホスティング機能）」と習ったかもしれませんが、それだけでは「セキュリティ」と「ドメイン」が弱いです。以下の3人のスペシャリストを組み合わせることで、完全なWebサイトになります。

  • Amazon S3（倉庫）： HTML、CSS、画像ファイルを置いておく場所。単体だとURLが長く、HTTPS化が面倒という弱点があります。
  • Amazon CloudFront（配送センター兼ガードマン）： S3のデータをキャッシュして爆速配信し、HTTPS（SSL証明書）を付与します。
  • Amazon Route 53（案内人）： あなたが取得した「かっこいいドメイン（例: myblog.com）」を、CloudFrontの複雑なURLに紐付けます。

3. この構成の「3つの最強ポイント」（試験対策）

なぜS3単体ではなく、わざわざこの構成にするのか？ 試験で問われる理由は以下の3点です。

① セキュリティ (HTTPSとOAC)

S3単体では基本的にHTTP（暗号化なし）になりますが、CloudFrontを使えば無料で簡単にHTTPS化できます。また、S3バケット自体は「パブリックアクセスをすべてブロック（非公開）」にし、OAC（Origin Access Control）を使って「CloudFront経由のアクセスだけ許可する」ことで、S3の中身が勝手に覗かれるのを防ぎます。

② パフォーマンス (爆速表示)

S3単体だと、例えば東京リージョンのS3に置いたデータは、ブラジルの人には表示が遅くなります。CloudFrontを使えば、世界中のエッジロケーションから配信されるので、地球の裏側でも爆速で表示されます。

③ コスト (実は安い)

CloudFrontからのデータ転送量は、S3から直接配信するよりもボリュームディスカウントで安くなるケースが多いです。また、S3からCloudFrontへのデータ転送は無料です。

4. 設定のキモ（ここを間違えると動かない）

実務や試験の設定問題で問われる、間違いやすいポイントです。

S3の設定
  • 静的ウェブサイトホスティング機能は OFF にする。（ここが意外なひっかけ！）
  • バケットは プライベート（非公開） にする。
  • S3は単なる「データ置き場」として振る舞います。
CloudFrontの設定
  • OAC (Origin Access Control) を作成し、S3バケットポリシーに書き込む。
  • デフォルトルートオブジェクト に index.html を指定する。（/ でアクセスされた時にトップページを表示するため）
Route 53の設定
  • CloudFrontへの紐付けには、CNAMEではなく 「エイリアスレコード」 を使う。
  • 理由：エイリアスレコードを使うと、Route 53へのクエリ料金が無料になるからです（試験頻出！）。

5. まとめ

「S3に置いて（鍵をかける）、CloudFrontで配る（HTTPS化）、Route 53で名前をつける」。これが、現在AWSで静的Webサイト（ブログやLPなど）を作る際の事実上の標準（デファクトスタンダード）です。
