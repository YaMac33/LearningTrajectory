AWSインスタンス役割一覧

#AWS #インスタンス #試験対策 #Web #AP #DB

システム構築の基本形である「Web層・AP層・DB層」の3層構造に対して、AWSのどのサービス（インスタンス）が使われるのかを整理します。

試験では「Webサーバーとして適切なものは？」「バッチ処理を行うAPサーバーは？」といった問われ方をするため、各層の役割と代表選手を紐づけて覚えることが重要です。

1. 3層アーキテクチャとAWSサービス

典型的なWebシステムの構成と、そこに配置される主要なサービスの一覧です。

2. 各層の詳細と試験ポイント

① Webサーバー層（プレゼンテーション層）

ユーザーのブラウザに表示される「画面」を配信する役割です。

  • EC2: 最も基本。ApacheやNginxをインストールしてWebサーバーにします。
  • Lightsail: 「WordPressをすぐに立ち上げたい」といった小規模・個人利用向け。

② APサーバー層（アプリケーション層）

「在庫があるか確認する」「決済処理を行う」といった、システムの頭脳となる処理を行います。

  • EC2: JavaやPHP、Rubyなどのプログラムを動かします。
  • Lambda: サーバー管理不要（サーバーレス）でコードを実行します。短い処理に最適。
  • ECS / EKS / Fargate: コンテナ（Docker）を使ってアプリを動かす場合の主役です。

③ DBサーバー層（データ層）

大切なデータを保存する場所です。

  • RDS / Aurora: 顧客情報や注文履歴など、整合性が重要なデータ（RDBMS）。
  • DynamoDB: カート情報やゲームのログなど、大量・高速なデータ（NoSQL）。
  • ElastiCache: よく使うデータをメモリに一時保存して、表示を速くする（キャッシュ）。

まとめ

ECサイトで例えると、以下のようになります。

  • Web層 (EC2/Lightsail): 「商品ページ」をお客さんに見せる係。
  • AP層 (Lambda/Fargate): 「在庫確認」や「お会計」を計算する係。
  • DB層 (RDS/DynamoDB): 「注文台帳」や「商品リスト」を金庫に保管する係。

この「役割分担」をイメージできれば、どのサービスを選べばよいか自然と導き出せます。
