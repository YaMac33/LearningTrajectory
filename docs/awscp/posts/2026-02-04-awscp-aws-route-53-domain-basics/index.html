ルートドメインとサブドメインの基本

#AWS #ドメイン #Route 53 #試験対策

AWSにおけるルートドメインとサブドメインの役割を解説。試験対策にも最適です。

「ルートドメイン」と「サブドメイン」は、AWSに限らず、CLF-C02などの試験でネットワークやRoute 53の文脈で必ず登場する重要用語です。
一言で言うと、以下の関係になります。

  • ルートドメイン: そのWebサイトの「本体となる名前」（例: example.com）
  • サブドメイン: ルートドメインの前に文字を付けて作る「用途別の名前」（例: blog.example.com）

1. ルートドメインとは？

インターネット上の住所の「本籍地」です。
企業の公式サイトなどで使われる、最も短くてシンプルな名前です。

ルートドメインのイメージ
example.com
東京都〇〇区1-1 (一軒家の住所)
  • 実務イメージ: 会社のメインサイト（HP）に使われます。
  • 試験ポイント: ドメイン名の中心となる部分であり、Route 53（Hosted Zone）で管理する対象です。
  • 注意: 「www」はルートドメインの一部ではありません。

2. サブドメインとは？

ルートドメインを用途ごとに部屋分けした名前です。
同じ `example.com` という土地の上に、異なる役割の窓口を作るイメージです。

ビルの階層としてのサブドメイン
3F: 開発 api.example.com
2F: ブログ blog.example.com
1F: 受付 www.example.com
ビル本体: example.com

実務での使い分け例

用途 サブドメイン例 アクセス先 (AWSリソース)
Webサイト www.example.com CloudFront (会社HP)
ブログ blog.example.com WordPress (EC2)
API api.example.com API Gateway / ALB
管理画面 admin.example.com 社内用ALB (認証付き)

3. 構造の図解とAWSとの関係

Route 53（DNSサービス）では、ルートドメインを親として、ぶら下がるようにサブドメインを管理します。

ドメインの階層ツリー
example.com (ルート)
www Webサイト
blog ブログ
api システム

試験でのひっかけポイント

  • ❌ 「www.example.com がルートドメインである」
    → 間違いです。ルートドメインは `example.com` です。`www` もあくまで「よく使われるサブドメインの一種」にすぎません。
  • ❌ 「サブドメインを作るには別契約が必要」
    → 間違いです。ルートドメイン（example.com）さえ持っていれば、Route 53の設定だけでサブドメインは自由に（無料で）何個でも作れます。

まとめ

用語 意味 例
ルートドメイン ドメインの本体・本籍地 example.com
サブドメイン 用途別に分けた名前 blog.example.com
www Webサイト用の慣習的なサブドメイン www.example.com

試験で「example.com を管理しつつ、api.example.com を別のAWSリソースに向けたい」という問題が出たら、「Route 53でサブドメインのレコードを作成する」が正解です！
