AWS RDSのデータベースエンジン

#AWS #DBエンジン #Aurora #MySQL #PostgreSQL #SQL Server

Amazon RDS（Relational Database Service）では、用途や予算に合わせて6種類のデータベースエンジン（心臓部）から好きなものを選ぶことができます。

これらは大きく分けて、「AWS独自」「オープンソース」「商用」の3つのグループに分類すると、特徴がスッキリ理解できます。試験でも実務でも、この分類が非常に重要です。

1. AWS独自（最強のクラウドネイティブ）

① Amazon Aurora（オーロラ）

「AWSがクラウドのために再設計した、最強のデータベース」です。RDSの一部ですが、別格の存在として扱われます。

  • 互換性: MySQL と PostgreSQL に互換性があります（既存アプリをそのまま移行可能）。
  • 速度: 標準的なMySQLの5倍、PostgreSQLの3倍高速です。
  • コスト: 商用DB（Oracle等）の1/10のコストで、同等のパフォーマンスが出せると謳っています。

2. オープンソース（無料で使える定番）

これらはソフトウェア自体の利用料（ライセンス料）がかからないため、安価で人気があります。

② MySQL（マイエスキューエル）

世界で最も利用されているオープンソースデータベースです。

  • Web開発（WordPressなど）の標準です。
  • 「LAMP環境（Linux, Apache, MySQL, PHP）」のMです。

③ MariaDB（マリアディービー）

MySQLから派生した（フォークされた）兄弟のようなデータベースです。

  • MySQLの開発者が、Oracle社に買収されたMySQLの将来を案じて作ったものです。
  • 基本的にはMySQLとほぼ同じ使い勝手ですが、性能が改善されている部分もあります。

④ PostgreSQL（ポストグレスキューエル）

「ポスグレ」と呼ばれ、MySQLと並ぶ人気者です。

  • MySQLよりも「複雑な処理」や「堅牢性」に強いとされています。
  • Oracleとの互換性が高いため、Oracleからの移行先として選ばれることが多いです。
  • PostGIS という拡張機能を使った「地図情報（位置情報）」の処理が得意です。

3. 商用データベース（ライセンスが必要）

企業が開発・販売しているデータベースです。利用にはライセンス料が必要です。

⑤ Oracle（オラクル）

データベース界の絶対王者。大企業の基幹システムで長年使われています。

  • License Included: AWSがライセンス料込みで貸してくれるモデル。
  • BYOL (Bring Your Own License): 会社ですでに持っているライセンスをAWSに持ち込むモデル。

⑥ Microsoft SQL Server（エスキューエルサーバー）

マイクロソフト製のデータベースです。

  • Windows Server や .NET（C#）で作られたアプリとの相性が抜群です。
  • Oracle同様、BYOLが可能です。

試験のポイント

  • 「商用データベース（Oracle/SQL Server）の移行」
    → AWS Database Migration Service (DMS) を使って、Aurora に移行するのがAWSの推奨パターン（コスト削減のため）です。
  • 「ライセンス持ち込み」
    → BYOL (Bring Your Own License) という単語を覚えてください。OracleとSQL Serverで可能です。
  • 「性能重視」
    → 何も考えずに Aurora を選びましょう。
