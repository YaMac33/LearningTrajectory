AWS Storage Gatewayの概要

#AWS #Storage Gateway #クラウド #ハイブリッド #ソリューション

企業のオンプレとAWSを繋ぐハイブリッドクラウドの橋渡しサービスの詳細。

AWS Storage Gateway は、一言で言うと「会社のオフィス（オンプレミス）と、AWSの倉庫（S3など）を繋ぐ『魔法のトンネル』」です。
これは「ハイブリッドクラウド（半分オンプレ、半分クラウド）」を実現するための最も代表的なサービスです。

1. なぜこれが必要なの？（抱えている課題）

企業には、「どうしても社内に置いておかなければならない古いサーバー」や「大容量のファイルサーバー」があります。しかし、ハードディスクを買い足すのは高いし、バックアップも面倒です。
「容量無制限のクラウド（S3）を使いたいけど、社内のシステムを書き換えるのは無理…」という悩みに対し、このサービスが解決策となります。

2. Storage Gateway の解決策： 「騙して繋ぐ」

Storage Gateway は、社内に「仮想のアプライアンス（ソフトウェア）」として設置します。

ハイブリッド接続のイメージ
🏢 オンプレミス

☁️ AWS (S3)

プロトコル変換を行い、裏側でクラウドに転送する。

  • 社内のPCから見ると: 普通の「ファイルサーバー」や「外付けハードディスク」に見えます。いつも通りにファイルを保存します。
  • 裏側の動き: Gatewayが受け取ったデータを、裏でこっそりAWS（S3など）に転送・保存します。ユーザーはクラウドを使っている意識なく、容量無制限のストレージを使えるようになります。

3. 試験に出る「3つのタイプ」（超重要！）

試験では、「プロトコル（通信の言葉）」と「用途」でどのタイプを使うか選ばされます。

ゲートウェイの種類と用途
📂 S3 File Gateway
NFS / SMB
普通のファイルサーバーとして使う。
保存したファイルは、そのままS3のオブジェクトになる。
データ分析の前処理などに最適。

iSCSI

① S3 File Gateway （ファイルゲートウェイ）

社内のファイルサーバーをS3に移行したい場合に使います。ファイルとして読み書きでき、S3上でもそのまま扱えるのが特徴です。

② Volume Gateway （ボリュームゲートウェイ）

サーバーのディスク容量拡張や、災害対策（DR）に使います。

  • 保管型 (Stored): データ全部をローカルに持ち、AWSはバックアップ用。
  • キャッシュ型 (Cached): データの実体はS3に置き、よく使うデータだけローカルに持つ（容量節約）。

③ Tape Gateway （テープゲートウェイ）

物理的なテープの管理をやめたい場合に使います。既存のバックアップソフトを変更せずに導入できます。

4. Snow Family との違い

「データを運ぶ」という意味では AWS Snow Family (Snowballなど) と似ていますが、使い分けは明確です。

サービス 接続方法 頻度 たとえ
AWS Storage Gateway オンライン (ネット回線) 日常的・継続的 「水道管」 (常に流れている)
AWS Snow Family オフライン (配送トラック) 一回限り (大容量) 「給水車」 (大量に運んでくる)

まとめ

  • AWS Storage Gateway: オンプレミスから、プロトコルを変換してAWSストレージを使うための橋渡し。
  • S3 File Gateway: ファイル (SMB/NFS) として読み書きし、S3に置く。
  • Volume Gateway: ディスク (iSCSI) として使い、バックアップする。
  • Tape Gateway: テープ の代わりに Glacier に保存する。

「社内のサーバー容量が足りない！でもクラウド移行は大変！」という時に、「とりあえずこのゲートウェイを置いておけば、裏側は無限のS3に繋がりますよ」と提案するサービスです！
