AWS S3の基礎知識と特徴

  • AWS
  • S3
  • クラウドストレージ
  • 試験対策
  • オブジェクトストレージ

AWSの中で「最も使われている」「最も試験に出る」サービスの一つ、Amazon S3（Simple Storage Service）。一言で言えば、「容量無制限、インターネット経由で出し入れ可能な『巨大なファイル保管庫』」です。

1. Amazon S3 とは？

EC2（サーバー）にくっつける「EBS」と混同しやすいですが、役割が明確に異なります。

構造のイメージ

S3には厳密な「フォルダ」という概念はありません。「バケット（バケツ）」という大きな容器を作り、その中に「オブジェクト（データ）」を放り込みます。

バケット名は世界で唯一（全AWSユーザーの中で重複しない名前）である必要があります。

2. 試験に出る「S3の3つの特徴」

AWS認定試験では以下の3点が頻出です。

① 驚異の耐久性「イレブン・ナイン」 データは自動的に最低3箇所のAZ（データセンター群）にコピーされます。耐久性は99.999999999%（9が11個）。「1万個のファイルを1000万年保存して、やっと1個失われるかどうか」というレベルで、事実上データは消えません。
② 静的ウェブサイトホスティング EC2などのサーバーを立てなくても、S3だけでHTML/CSS/JSを公開してWebサイトにできます。
試験ポイント：「サーバーレスで安価にWebサイトを公開するには？」→ 答えは S3 です。
③ バージョン管理 ファイルを「上書き保存」してしまっても、過去の状態に戻せる機能です。誤って削除した場合の復活も可能です。

3. ストレージクラス（料金プランの使い分け）

S3にはアクセスの頻度に応じて「料金プラン（ストレージクラス）」を選べます。ここが試験の最頻出ポイントです。

クラス名 特徴 向いている用途 取り出し速度
S3 Standard
(標準)		 デフォルト。一番高いが性能も最高。 頻繁にアクセスするWeb画像、動画。 ミリ秒
S3 Intelligent-Tiering 「迷ったらこれ」。AIがアクセス頻度を見て、自動で安い階層へ移動。 アクセスパターンが読めないデータ。 ミリ秒

4. ライフサイクルポリシー（自動化）

いちいち手動でプランを変更するのは大変です。「最初はStandardに保存して、30日経ったらIAに移して、1年経ったらGlacierに入れて、3年経ったら削除する」といったルールを事前に設定できます。これをライフサイクルポリシーと言い、コスト最適化の切り札となります。

5. セキュリティ（バケットポリシー）

S3はインターネットに面しているため、設定を間違えると情報漏洩事故につながります。

  • デフォルト： 「パブリックアクセスをすべてブロック」になっています（安全）。
  • バケットポリシー： 「誰がこのバケットにアクセスできるか」を定義するルール（JSON形式）です。特定のIPアドレスや、CloudFrontからのアクセスのみを許可する際などに使います。

6. まとめ

  • S3 = オブジェクトストレージ（ファイル置き場）。
  • イレブン・ナイン（データは消えない）。
  • 静的サイトが作れる（EC2いらず）。
  • Glacierは安いけど取り出しが遅い（アーカイブ用）。
  • ライフサイクルポリシーでクラス移動を自動化。

コンピューティング(EC2)、ネットワーク(VPC)、セキュリティ(IAM)、ストレージ(S3)、データベース(RDS)。これらを組み合わせることで、AWS上で「どういう構成が一番お金がかからないか（コスト最適化）」という議論ができるようになります。
