AWSのオートスケーリング入門

  • Auto Scaling
  • EC2
  • クラウド
  • コスト最適化
  • AWS

Amazon EC2 Auto Scaling（オートスケーリング）は、クラウドならではの「伸縮自在（エラスティック）」な性質を最も象徴する機能です。サーバーの台数を自動調整することで、機会損失の回避とコスト削減を同時に実現します。

1. オートスケーリングとは？

一言で言えば、「アクセス数（負荷）に合わせて、EC2の台数を自動で増やしたり減らしたりする機能」です。

  • 混んでいる時（スケールアウト）： サーバーを勝手に増やして、ダウンしないように処理能力を高めます。
  • 空いている時（スケールイン）： サーバーを勝手に減らして、無駄な料金を節約します。

これは「居酒屋のシフト管理」に例えると分かりやすいでしょう。

金曜の夜（アクセス増）： バイトを急いで3人呼んで対応する。
平日の閑散時（アクセス減）： 「もう上がっていいよ」と早上がりさせて人件費を抑える。

これを人間が判断するのではなく、システムが自動で行うのがオートスケーリングです。

2. 仕組みの図解（ELBとの連携）

オートスケーリングは単独ではなく、通常はELB（ロードバランサー）とセットで使用されます。増えたサーバーにアクセスを振り分ける「司令塔」が必要だからです。

3. 試験に出る「スケーリング」の用語

試験では「サーバーを増やすこと」を専門用語で何と呼ぶか、その違いが問われます。オートスケーリングは「水平スケーリング」が基本です。

① 水平スケーリング
(Scale Out / In)
🚗 🚗 🚗

やり方： 台数を増やす
メリット： 無停止で拡張可能
イメージ： 軽自動車をもう一台呼んでくる。
※AWSの推奨パターン

② 垂直スケーリング
(Scale Up / Down)
🏎️💨

やり方： スペック(CPU等)を上げる
デメリット： 再起動(停止)が必要
イメージ： エンジンをフェラーリのものに積み替える（大変）。

4. 試験でのひっかけポイント

クラウドプラクティショナー試験などでよく問われるポイントです。

  • 予測スケーリング： 「CPUが上がったら」という反応だけでなく、過去のデータから「明日の12時は混むはず」と予測して事前に増やすことも可能です。
  • ヘルスチェック（自己修復）： 負荷対策だけでなく、「故障したEC2を自動で捨てて、新品を作り直す」機能もあります。これによりシステムの健康状態が保たれます。

5. まとめ

オートスケーリングによって、「必要な時に、必要な分だけ」リソースを使うことが可能になります。これがクラウドの最大のメリットである「コスト最適化」に繋がります。
