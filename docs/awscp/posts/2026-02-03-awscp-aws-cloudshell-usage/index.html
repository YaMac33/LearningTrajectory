AWS CloudShellの使い方と特徴

#AWS #CloudShell #コマンドライン #無料 #開発環境

AWS CloudShellはブラウザで即使用可能な無料のコマンド環境。特徴やCloud9との違いを解説。

AWS CloudShell（クラウドシェル）は、AWSの管理画面（ブラウザ）から直接起動できる、「コマンド入力専用の黒い画面（ターミナル）」です。
一言で言うと、「準備ゼロですぐに使える、ブラウザ版のコマンドプロンプト」です。AWS Cloud9 と似ていますが、もっと「手軽さ」に特化しています。

1. 何が便利なのか？（3つの特徴）

わざわざPCに黒い画面（ターミナル）を用意しなくてもいい理由がこれです。

CloudShellの3つの「楽」
📦 インストール不要 AWS CLI, Python, Node.js等が最初から入っている「全部入り」。
💾 1GBストレージ ファイルを保存して閉じても、次回ログイン時にデータが残る。
🆓 完全無料 EC2等の料金は一切かからない。タダで使えるLinux環境。

① インストール不要（全部入り）

AWSを操作するためのコマンド（AWS CLI）や、Python、Node.jsなどがプレインストールされています。「自分のパソコンにツールを入れるのは面倒だけど、ちょっとだけコマンドで確認したい」という時に、クリック一発で起動できます。

② 1GBの無料ストレージ付き

CloudShellを閉じても、ホームディレクトリにあるデータ（最大1GB）は消えずに残ります。
ちょっとしたスクリプト（プログラム）や設定ファイルを保存しておけるので、次回ログインした時にすぐに作業を再開できます。
※ただし、このデータは「リージョンごと」に保存されます。

③ 完全無料

CloudShell自体は無料です。（※Cloud9は裏でEC2が動くので料金がかかりますが、CloudShellはかかりません）

2. Cloud9 との違い（試験のポイント）

「どっちもブラウザで動く開発環境じゃないの？」と迷いますが、「道具としての重さ」で使い分けます。

道具としての使い分けイメージ
🔪
AWS CloudShell
スイスアーミーナイフ
・ポケットから出して即使う
・コマンド操作のみ
・無料
🛠️
AWS Cloud9

・腰を据えて作業する
・コード編集＋デバッグ
・有料 (EC2料金)

主な機能 コマンド実行のみ (CUI) コード編集 + 実行 (IDE)
画面構成 黒い画面 (ターミナル) だけ エディタ、ファイルツリー、ターミナル

短いスクリプト実行。		 アプリをガッツリ開発。
デバッグやペアプログラミング。

3. 具体的な使い方

  1. AWSコンソールの画面左下（または右上）にある「ターミナルのアイコン（>_）」をクリックします。
  2. ブラウザの下半分に黒い画面が出てきます。
  3. すでにログイン中の権限を持っているので、すぐに aws s3 ls （S3の中身を見る）などのコマンドが打てます。

まとめ

  • AWS CloudShell: ブラウザですぐ使える無料のコマンド環境。
  • 特徴: AWS CLIインストール済み、1GBの永続ストレージ。
  • 使い分け:
    • 開発（コーディング）なら Cloud9。
    • 操作（コマンド）なら CloudShell。

「EC2を立てるまでもないけど、コマンドでAWSを操作したい」という時に使う便利な道具です！
