AWS Artifactの概要

  • AWS
  • Artifact
  • セキュリティ
  • コンプライアンス
  • 監査

AWS Artifact（アーティファクト）は、試験の「セキュリティとコンプライアンス」分野で、知っているだけで1点取れるラッキー問題になりやすいサービスです。一言で言うと、「AWSの『監査レポート』や『契約書』をダウンロードできる、無料のセルフサービス資料室」です。役所にある「証明書発行機」や、企業の「重要書類保管庫」をイメージすると分かりやすいです。

1. AWS Artifact とは？

監査人や取引先から**「AWSって本当に安全なの？ 証拠を見せて」**と言われた時に駆け込む場所です。AWSは、ISOやSOCといった厳しい国際的なセキュリティ基準をクリアしていますが、口で言うだけでは誰も信じてくれません。そこで、「第三者機関がAWSを監査した結果（合格証）」をPDFとしてダウンロードできるようにしてあるのが、このサービスです。

2. 2つの主要機能（試験に出るのはここ！）

AWS Artifactには、大きく分けて「Reports（レポート）」と「Agreements（契約）」の2つの棚があります。この違いが試験で問われます。

📄 Reports (レポート)

「AWSは安全です」という
第三者からの証明書

  • 🏅
    ISO認定証 品質・セキュリティの国際規格
  • 📊
    SOCレポート 財務・セキュリティの詳細報告
  • 💳
    PCI DSS準拠証明 クレカ情報の取り扱い基準
🤝 Agreements (契約)

AWSとあなたの間で結ぶ
特別な契約書

  • 🏥
    BAA (HIPAA対応) 米国の医療情報を扱う契約
  • 🤐
    NDA 秘密保持契約

① AWS Artifact Reports（レポート）

  • いつ使う？
    • 自社の監査で、監査法人から「AWSのセキュリティチェック結果を出せ」と言われた時。
    • 上司や顧客に「AWSの安全性を証明しろ」と言われた時。

② AWS Artifact Agreements（契約）

  • いつ使う？
    • 「医療データをAWSで扱いたい」など、特定の法律に基づいた契約を結ぶ必要がある時。
    • 画面上でポチッと「同意する」を押すだけで、契約が適用されます。

3. 試験で狙われるポイント（キーワード）

以下の単語が問題文に出たら、AWS Artifact が正解である確率が非常に高いです。

Artifactを示唆するキーワード集
コンプライアンスレポート 第三者監査レポート SOCレポート ISO認定 PCI DSS 監査人 (Auditor) への提出 オンデマンドでダウンロード

※「オンデマンド」とは、サポートに電話しなくてもWeb画面から自分で好きな時に取れる、という意味です。

4. 責任共有モデルとの関係

以前学んだ「責任共有モデル」を思い出してください。Artifactは何を証明しているのでしょうか？

ユーザーの責任範囲
アプリのバグ、設定ミスなど
Artifactが証明するのはここ！
AWSの責任範囲
データセンター、物理セキュリティ
  • AWS Artifact が証明するのは、「AWSの責任範囲（クラウド『自体の』セキュリティ）」が安全であることです。
  • あなたが作ったアプリの安全性（クラウド『内の』セキュリティ）を証明するものではありません。あなたのアプリが安全かどうかは、あなたが第三者機関に依頼して監査してもらう必要があります。

5. まとめ

  • 役割： AWSの「身の潔白（安全性）」を証明する書類置き場。
  • Reports： 監査レポート（ISO, SOC）を取る。監査人用。
  • Agreements： 契約書（BAAなど）を結ぶ。法務用。
  • コスト： 無料。

これで、もし試験中に「監査」「レポート」「準拠状況」という言葉が出てきても、迷わず「Artifact」を選べるはずです。
