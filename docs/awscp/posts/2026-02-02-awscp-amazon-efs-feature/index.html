Amazon EFSの基本と特徴

#EFS #ストレージ #Linux #共有アクセス #EC2

Amazon EFS (Elastic File System) は、一言で言うと「クラウド上の『共有フォルダ（NAS）』」です。

これまで学んだストレージ（S3, EBS）と比較すると、EFSの立ち位置は明確です。「複数のEC2（サーバー）から、同時に、一つのフォルダを読み書きしたい」という時に使うサービスです。

試験では、「Linux専用」「共有アクセス」「自動拡張」の3点が頻出です。

2. 試験に出る「3つの強み」

① 複数台からの同時アクセス

これが最大の採用理由です。「Webサーバーが3台あるけど、画像ファイルは全部同じ場所（EFS）に置いて共有したい」といった構成（WordpressなどのCMS）で使われます。

② Linux 専用 (NFSプロトコル) ★超重要

EFSは、Linux OS（Amazon Linux, Ubuntuなど）でしか使えません。

  • 注意: Windows Serverからは使えません。
  • もし「Windowsで共有フォルダを使いたい」という問題が出たら、正解は Amazon FSx for Windows File Server になります。

③ 容量が無制限＆自動伸縮 (Elastic)

EBSのように「最初に100GB確保する」という必要がありません。

  • ファイルを入れたら勝手に大きくなり、消したら勝手に小さくなります。
  • 使った容量分だけ課金されます（事前プロビジョニング不要）。

耐久性の違い: EFSは、S3と同じようにデータを「複数のAZ（データセンター）」に自動保存してくれます。だから、もし1つのAZが停電してもデータは無事です。

4. コスト最適化機能（EFS IA）

S3と同じように、EFSにも「あまり使わないデータを安くする機能」があります。

  • EFS Standard: よく使うファイル用（標準）。
  • EFS Infrequent Access (EFS IA): たまにしか使わないファイル用（低頻度アクセス）。標準クラスより安価です。
  • ライフサイクル管理: 「30日間アクセスがないファイルを、自動的にIAクラス（安い方）に移動する」という設定が可能です。

まとめ

  • 役割: 複数のEC2でデータを共有するための「共有ファイルシステム」。
  • OS: Linux専用（NFSプロトコル）。WindowsはNG。
  • 特徴: 容量の確保不要（自動伸縮）、複数AZにデータ保存（高耐久）。
  • キーワード: 「共有アクセス」「Linux」「NFS」が出たらEFS。

これで、主要なストレージサービス（S3, EBS, EFS）の違いもクリアになったはずです。
