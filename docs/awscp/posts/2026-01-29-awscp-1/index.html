AWS Cloud Practitioner向け：主要サービスと責任共有モデル

2026/01/29

AWS Cloud Practitioner（AWS認定クラウドプラクティショナー）では、 「クラウドの基本概念」「主要サービスの役割」「セキュリティの考え方」を ざっくり説明できるレベルが求められます。本ページでは、頻出のサービスと 責任共有モデル（Shared Responsibility Model）をまとめます。

1. クラウドの基本：オンデマンドと従量課金

  • オンデマンド：必要なときに必要な分だけ利用する
  • 従量課金：使った分だけ支払う（初期投資を抑えやすい）
  • スケール：需要に応じて増減できる（スケールアウト/イン）

2. 主要サービスの役割（頻出）

コンピューティング：EC2 / Lambda

  • Amazon EC2：仮想サーバー。OSやミドルウェアを自由に構成できる
  • AWS Lambda：サーバーレス実行。イベントをきっかけにコードを実行する

ストレージ：S3 / EBS

  • Amazon S3：オブジェクトストレージ。高耐久・バックアップや静的サイトに便利
  • Amazon EBS：EC2に接続するブロックストレージ。OSディスクなどに使う

データベース：RDS / DynamoDB

  • Amazon RDS：マネージドRDB。MySQLなどを運用負荷少なく使える
  • Amazon DynamoDB：マネージドNoSQL。高速・スケーラブルなキー値型DB

ネットワーク：VPC / CloudFront

  • Amazon VPC：AWS内に論理的なネットワーク空間を作る（サブネット等）
  • Amazon CloudFront：CDN。コンテンツをエッジで配信し高速化する

3. 責任共有モデル（Shared Responsibility Model）

AWSでは「クラウドのセキュリティはAWSと利用者で分担する」という考え方を取ります。 よくある誤解は「AWSを使えば全部AWSが守ってくれる」です。実際は利用者側にも責任があります。

AWSの責任（セキュリティ “of” the cloud）

  • データセンターの物理セキュリティ
  • ハードウェア、ネットワーク基盤の保護
  • マネージドサービスの基盤部分（例：RDSのOS管理の一部など）

利用者の責任（セキュリティ “in” the cloud）

  • IAM設定（最小権限、MFA、ロールの適切な利用）
  • データ保護（暗号化、バックアップ、公開範囲の管理）
  • OS/アプリの運用（EC2のパッチ適用、設定ミス防止など）
  • ログ監視（CloudTrail/CloudWatch等の活用）

4. まとめ

AWS CPでは、サービスの細かい設定よりも「それぞれの役割」「課金の考え方」 「責任共有モデル」を説明できることが重要です。まずはEC2/S3/IAM/VPCの 関係を押さえ、次にRDSやCloudFrontなど周辺サービスへ広げると理解しやすいです。
