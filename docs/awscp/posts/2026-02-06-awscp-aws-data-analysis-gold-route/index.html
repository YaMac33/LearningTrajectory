AWSデータ分析の黄金の流れ

#AWS #データ分析 #QuickSight #Redshift #Glue #Athena

S3からRedshift、QuickSightまでのデータ分析の流れを解説します。

AWSでビッグデータ分析基盤を構築する際には、いくつかのサービスを組み合わせる「定石（黄金ルート）」が存在します。
「貯める(S3) → 加工する(Glue) → 保管する(Redshift) → 見る(QuickSight)」
この流れを理解しておくと、個々のサービスの役割がパズルのようにカチッとハマります。

1. データの旅（完全版フロー）

データが生まれてから、人間が意思決定に使うまでの流れを図解します。

S3を直接SQLで覗く
(簡易分析ルート)

  1. Amazon S3 (データレイク):
    • あらゆる生データ（ログ、CSV、画像）をとりあえずここに投げ込みます。容量無制限で安価です。
  2. AWS Glue / Amazon EMR (加工):
    • S3からデータを取り出し、汚れを取り除いたり、分析しやすい形に整えたりして、次の場所へ運びます。これをETL（抽出・変換・ロード）と呼びます。
  3. Amazon Redshift (データウェアハウス):
    • きれいに整理されたデータを大量に保管し、超高速で集計できるように待機させます。
  4. Amazon QuickSight (可視化):
    • Redshiftのデータを読み込み、人間が見やすいグラフやダッシュボードにして表示します。

2. QuickSight の役割： 「レストランのメニュー表」

Redshift（DWH）は、下ごしらえが済んだ食材が並ぶ「巨大な冷蔵庫」です。
しかし、冷蔵庫の中身（数字の羅列）を直接見ても、ビジネスの状況は分かりにくいものです。

Amazon QuickSight は、その食材を使って、経営者やマネージャーがひと目で状況を理解できる「きれいな盛り付け（グラフや表）」にしてテーブルに運んでくれる役割（BIツール）です。

3. 「Athena」と「QuickSight」の使い分け

どちらも「分析」に使われますが、用途が明確に違います。

サービス Amazon Athena Amazon QuickSight
対象データ S3上のファイルを直接 S3, Athena, Redshift, RDSなど様々
操作方法 SQLクエリを書く マウスでドラッグ＆ドロップ
出力結果 表計算のような「文字データ」 棒グラフや円グラフなどの「絵」
シチュエーション 「とりあえずS3の中身をサクッと確認したい」「エラーログを調査したい」 「売上の推移をグラフにして、チームで共有したい」「経営ダッシュボードを作りたい」

まとめ：試験対策キーワード

  • Amazon QuickSight: 「ダッシュボード」「視覚化」「BI（ビジネスインテリジェンス）」という言葉が出たらこれです。
  • AWS Glue: 「ETL（Extract/Transform/Load）」「データカタログ（データの索引）」という言葉が出たらこれです。
  • Amazon Athena: 「サーバーレス」「S3上のデータに対して標準SQLを実行」という言葉が出たらこれです。

この「データの流れ」を頭の中に描けるようになれば、分析系の問題は怖くありません！
