AWS Organizationsの基礎知識

  • AWS
  • Organizations
  • SCP
  • 一括請求
  • クラウド管理

AWS Organizations（オーガニゼーションズ）は、「複数のAWSアカウントをまとめて管理する親玉サービス」です。これまでは「1つのAWSアカウントの中の話（IAMユーザーやVPC）」をしてきましたが、企業規模が大きくなると、「部署ごとにAWSアカウントを分けたい」「本番環境と開発環境でアカウントごと分けたい」という要望が出てきます。

そうやって増えてしまった大量のAWSアカウント（子）を、1つの親アカウントが統括管理する仕組みがこれです。試験では「一括請求（Consolidated Billing）」と「SCP（ガードレール）」の2点が超頻出です。

1. なぜアカウントを分けるのか？

「1つのアカウントの中にVPCを分ければいいのでは？」と思うかもしれません。しかし、アカウントごと分けることには、VPC分割にはない強力なメリットがあります。

  • 最強のセキュリティ境界： アカウントが違えば、IDもパスワードも全く別物です。万が一、開発用アカウントが乗っ取られても、本番アカウントは無傷で済みます。
  • コストの明確化： 「経理部のアカウント」「営業部のアカウント」と分ければ、タグ付け等をしなくても、どの部署がいくら使ったか一目瞭然です。

2. AWS Organizations の階層構造

Organizationsの構造は、一般的な「会社の組織図」と全く同じです。

管理アカウント
(親会社・支払い担当)
開発部 OU
(Organizational Unit)
メンバーアカウント A
(開発用)
経理部 OU
(Organizational Unit)
メンバーアカウント B
(本番用)
  • 管理アカウント (Management Account)： 親玉です。全ての支払い義務を持ち、全権限を握ります。ここが乗っ取られると全滅するので、最強のセキュリティ（MFAなど）が必要です。
  • メンバーアカウント (Member Account)： 子アカウントです。管理アカウントに招待されて加入します。
  • OU (Organizational Unit)： 部署（フォルダ）のようなものです。アカウントをグループ化して、まとめてルールを適用できます。

3. 試験に出る「2大メリット」

Organizationsを使う理由は、以下の2点に集約されます。

① 一括請求 (Consolidated Billing) ★お金の話

複数のアカウントの請求書を、管理アカウント（親）に一本化できます。

  • 支払いが楽： 子アカウントごとにクレジットカードを登録する必要がありません。親がまとめて払います。
  • ボリュームディスカウント（重要）： AWSは「使えば使うほど安くなる（S3やデータ転送など）」料金体系です。Organizationsを使うと、全アカウントの使用量を「合算」して計算してくれます。
    • 例：A社とB社が別々に10TB使うより、合算して20TB使った扱いにしたほうが、GB単価が安くなります。

② SCP (Service Control Policies) ★守りの話

IAMポリシーに似ていますが、もっと強力な「ガードレール（禁止令）」です。

  • 特徴： 「特定のアカウント（またはOU）全体に対して、AWSサービスの使用を制限する」機能です。
  • IAMとの違い：
    • IAM：「AさんはS3を使ってよし」と許可するもの。
    • SCP：「そもそもこのアカウントでは、誰であろうと（ルートユーザー含め）、絶対にS3を使ってはならん！」と禁止するもの。
  • 用途： 「開発環境OUでは、高額なEC2インスタンス（GPU付きなど）の起動を禁止する」「日本リージョン以外での操作を禁止する」といった統制に使います。

4. IAMポリシーとSCPの関係（図解イメージ）

SCPは「フィルター（ふるい）」のようなものです。いくらアカウント内のIAMで「管理者権限（AdministratorAccess）」を持っていても、親組織（SCP）が「禁止」と言えば使えません。

AWS Organizations (親)
SCP: 「高額なEC2 (p3.xlarge) は禁止！」 👮‍♂️
↓
❌ ここで弾かれる (Deny)
↓
メンバーアカウント (子)
IAMポリシー: 「EC2 フルアクセス許可」 😎
↓

「よーし、p3.xlarge を起動するぞ！」

5. 試験で狙われるシナリオ

このパターンの問題は非常によく出ます。

問題 1

あなたの会社では10個のAWSアカウントを運用しており、請求処理が煩雑です。また、各アカウントのストレージ使用量を合計して、ボリュームディスカウントを受けたいと考えています。どのサービスを使うべきですか？

正解：AWS Organizations の一括請求（Consolidated Billing）機能

問題 2

開発部のAWSアカウントでは、間違って高額なサービスを使わないように、Redshiftの使用を強制的に禁止したいです。そのアカウントのルートユーザーであっても使えないようにするにはどうすればいいですか？

正解：AWS Organizations の SCP (Service Control Policies) を適用する

6. まとめ

  • 複数アカウント管理： 増えすぎたアカウントを階層構造（OU）で管理する。
  • 一括請求： 請求を親に一本化し、ボリュームディスカウントでお得になる。
  • SCP： アカウント全体にかける強力な制限（ガードレール）。ルートユーザーも逆らえない。
  • 無料： このサービス自体は無料で使えます。

これで、AWSアカウント管理とコスト削減の切り札についてもバッチリです。
