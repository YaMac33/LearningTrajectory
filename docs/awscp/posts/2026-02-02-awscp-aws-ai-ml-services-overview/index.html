AWSのAI/MLサービス全体像

  • AWS
  • AI
  • ML
  • 生成AI
  • 機械学習

AWS認定試験（CLF-C02）において、AI（人工知能）とML（機械学習）の分野は近年非常に重要視されています。最新の試験範囲に基づき、従来の「特定の機能を持つAIサービス」に加えて、注目度の高い「生成AI」も含めて体系的に整理しました。

1. AWS AI/ML サービスの全体像（3層構造）

AWSのAI/MLサービスは、利用者が必要とする「専門知識のレベル」に応じて、大きく3つの層（レイヤー）に分かれています。

Layer 1: AIサービス (アプリ層) すぐに使えるAI 専門知識不要。APIを呼ぶだけで「目・耳・口」の機能や「生成AI」を利用可能。
(Bedrock, Rekognition, Polly etc.)
Layer 2: MLプラットフォーム モデルを作る・育てる データサイエンティスト向け。独自のAIモデルを開発・管理する環境。
(SageMaker)
Layer 3: インフラストラクチャ パワーを支える 超高性能な計算リソース。学習や推論に特化した専用チップ。
(Trainium, Inferentia)

2. Layer 1: 生成AI・アプリケーション層

エンジニアが機械学習の理論を知らなくても、APIを介して最先端のAI機能をアプリに組み込めるグループです。試験では「〇〇をしたい時、どのサービスを使うか？」というマッチング問題が出ます。

サービス名 概要（一言でいうと） 主な利用シーン
🧠Amazon Bedrock 生成AIの基盤モデル利用 チャットボット、文書要約、画像生成
※サーバーレスでモデルを選んで使える
💬Amazon Q AIアシスタント AWSコンソールでの質問、コード生成、ビジネス分析
👁️Amazon Rekognition 画像・ビデオ分析 顔認証、不適切なコンテンツの検出、物体検出
🗣️Amazon Polly テキストを音声に変換 文章の読み上げ、ナレーション作成
👂Amazon Transcribe 音声をテキストに変換 会議の文字起こし、動画の字幕作成
🤖Amazon Lex 対話型AI（チャットボット） コールセンターの自動応答 (Alexaの技術)
📖Amazon Comprehend テキスト解析（自然言語処理） 顧客レビューの感情分析（ポジ・ネガ判定）
🌐Amazon Translate 高精度な機械翻訳 多言語サイトの自動翻訳、チャット翻訳
📄Amazon Textract 文書からのデータ抽出 スキャンした請求書の表データを読み取り (OCR)

3. Layer 2: 機械学習プラットフォーム層

自分たちで独自のAIモデルを構築したい開発者向けのグループです。

  • Amazon SageMaker： 機械学習の「一連の流れ（準備→学習→デプロイ）」を1つのサービスで完結できるプラットフォームです。
  • 試験ポイント： 「自社データを使い、独自のアルゴリズムでモデルをトレーニングしたい」と言われたら SageMaker が正解です。

4. Layer 3: インフラストラクチャ層

AIを動かすための「足腰」となる、専用チップや高速な計算機グループです。EC2インスタンスのタイプとして提供されます。

名称 役割
AWS Trainium AIの学習（トレーニング）を高速化・低コスト化する専用チップ。
AWS Inferentia AIの推論（実行）を効率化する専用チップ。
※作ったモデルを使って予測することを「推論」と呼びます。

5. 実務イメージ：カスタマーサポートの改善

これらのサービスは単独で使うだけでなく、組み合わせることで強力なソリューションになります。

6. まとめ：試験攻略のコツ

  • 専門知識の有無： 「知識なしですぐ使いたい」ならAIサービス、「独自に作りたい」ならSageMaker。
  • キーワード一致： 「感情分析＝Comprehend」「顔認証＝Rekognition」「チャットボット＝Lex」。
  • 生成AI： 「基盤モデル」「生成AI」なら Bedrock。

AI系サービスは名前が機能を表していることが多い（Polly＝オウム、Translate＝翻訳など）ので、名前から連想できるようにしておきましょう。
