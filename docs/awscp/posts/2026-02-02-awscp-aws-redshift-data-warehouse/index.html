Amazon Redshiftの徹底解説

  • AWS
  • Redshift
  • データ分析
  • データウェアハウス
  • OLAP

データ分析の王道サービス「Amazon Redshift（レッドシフト）」を徹底解説します。Athenaが「生データをサクッと見る」サービスだったのに対し、Redshiftは「大量のデータをガッツリ溜めて、高速に分析する」ための巨大な倉庫（データウェアハウス）です。

1. Amazon Redshiftとは？

Amazon Redshiftは、「データウェアハウス（DWH）」と呼ばれる、分析専用の超高速なデータベースサービスです。ペタバイト規模（テラバイトの1000倍）のデータに対しても、複雑なクエリを高速に実行できます。

イメージ：図書館の「巨大な索引システム」

普通のデータベース（RDSなど）が「本屋さん（1冊ずつ売買する）」なら、Redshiftは「巨大な図書館（何万冊もの本から特定の傾向を調べる）」です。「昨日売れた本は？」と聞くのではなく、「過去10年で、雨の日に20代の男性が買った本の傾向は？」というような、膨大で複雑な質問（クエリ）に答えるのが得意です。

2. Redshiftが速い「2つの秘密」

試験でもこの「仕組み」の名前がヒントになることがあります。

RDS (行指向)
1人分のデータをまとめて保存
1田中25東京
2佐藤30大阪

「年齢」だけ欲しくても、全部読む必要がある。
(遅い)

Redshift (列指向)
項目ごとにまとめて保存
253045...
東京大阪福岡...

「年齢」の列だけ読めば終了。
(爆速！)

① 列指向（カラムナ）ストレージ

分析では「全社員の平均年齢を知りたい」といった、特定の項目（列）だけを集計することが多いです。Redshiftは列ごとにデータを固めて保存しているため、不要な列（名前や住所）を読み飛ばすことができ、劇的に速くなります。

② MPP（Massively Parallel Processing：超並列処理）

1つの重い作業を、たくさんの計算機（ノード）で分担して同時に処理します。「1人で100枚の皿を洗う」のではなく「10人で10枚ずつ一斉に洗う」イメージです。

リーダーが命令を分割し、部下が一斉に計算して結果を返す。

3. RDS（普通のDB）との違い

ここが一番の「試験出る出るポイント」です！

項目 Amazon RDS Amazon Redshift
主な用途 OLTP (オンライントランザクション)
日々の取引、注文処理		 OLAP (オンライン分析処理)
複雑な分析、集計
得意なこと データの追加・更新・削除
(書き込み頻度が高い)		 大量データの読み取り・集計
(書き込みはバッチでまとめて)
データの単位 行（レコード）単位 列（カラム）単位
例え話 銀行のATMやレジ 経営会議のための分析レポート

4. 試験で問われやすいポイント（CLF-C02）

  • キーワード： 「データウェアハウス」「OLAP」「ペタバイト規模」「列指向」「BIツールとの連携」。
  • Redshift Spectrum： 前回の「Athena」のように、データをRedshiftに読み込ませなくても、S3にあるデータを直接クエリできる機能です。S3（データレイク）とRedshift（ウェアハウス）の橋渡しをします。
  • フルマネージド： セットアップ、管理、バックアップなどはAWSがやってくれます。

5. 実務イメージ：全国チェーンの売上分析

コンビニチェーンの本部を想像してください。

  1. 全国数千店舗のレジから、毎秒「何が売れたか」というデータが **RDS（各店舗のDB）**に送られます。
  2. 夜間にそのデータを **ETL（AWS Glueなど）**を使って整理し、**Redshift**へ移動させます。
  3. 翌朝、マーケティング担当者が「先週の新商品の売上推移と、天気の関係」を **Redshift**上で数秒で集計し、次の戦略を立てます（QuickSightで可視化）。

6. まとめ

  • Redshift = データウェアハウス（分析専用の巨大倉庫）。
  • 高速化の秘密 = 列指向 ＆ MPP（超並列）。
  • RDSとの違い = 「記録（OLTP）」ならRDS、「分析（OLAP）」ならRedshift。

これで、AWSのデータ分析基盤の主役について理解できました。
