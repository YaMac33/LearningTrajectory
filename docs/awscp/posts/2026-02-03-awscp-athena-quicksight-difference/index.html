AthenaとQuickSightの違い

#AWS #データ分析 #可視化 #Athena #QuickSight

S3データを扱うAthenaとQuickSightの使い分けを解説します。

「S3にあるデータを見る」という目的は同じでも、その「見え方」と「見るための手段」が全く違います。
一言で言うと、以下の違いがあります。

  • Amazon Athena: S3の中身を「文字（表）」として、細かく検索・抽出するツール。
  • Amazon QuickSight: S3の中身を「絵（グラフ）」として、全体を可視化・分析するツール。

エクセル（Excel）に例えると非常に分かりやすくなります。

1. エクセルで例える決定的な違い

S3に「1年分の売上データ（巨大なCSVファイル）」が入っていると想像してください。

エクセル機能でのたとえ
📋
Athena
(フィルター・関数)
「A商品の売上だけ抽出して」
「合計値を計算して」

見えるもの：表 (Table)

(グラフ作成)
「売上の推移を棒グラフにして」
「割合を円グラフにして」

見えるもの：絵 (Chart)

① Amazon Athena（アテナ）

エクセルの「フィルター機能」や「関数」です。
「4月のデータだけ見せて」「A商品の売上合計を計算して」と命令（SQL）を書いて、条件に合う行や計算結果を表示します。
特定のデータを探し出したり、生データの中身を確認したりするのに向いています。

② Amazon QuickSight（クイックサイト）

エクセルの「グラフ作成機能」です。
マウスで項目を選んで、カラフルなダッシュボードを作ります。
売上の傾向（トレンド）や、異常値を目で見て直感的に理解するのに向いています。

2. それぞれの詳細解説

① Amazon Athena： 「SQLで直接問いかける」

S3にあるファイル（CSV, JSON, Parquetなど）に対して、標準的なSQL（データベースを操作する言語）を使って直接検索をかけます。

  • 「見る」プロセス:
    • コンソール画面で SQL を書く（例：SELECT * FROM sales WHERE date = '2023-01-01'）。
    • 数秒後、S3の中から条件に合うデータだけが「表形式」で抽出されて表示される。
  • メリット:
    • 準備不要: データをデータベース（RDS等）に移す必要がない。S3のまま検索できる。
    • 安い: スキャンしたデータ量（検索した分）だけ課金。

② Amazon QuickSight： 「可視化して分析する」

いわゆる BI（ビジネスインテリジェンス）ツールです。データを人間にわかりやすい形に変換します。

  • 「見る」プロセス:
    • ダッシュボードを開く。
    • S3のデータが既に「グラフ」や「地図」、「推移表」になっている。
  • メリット:
    • 直感的: SQLが書けない営業担当や経営層でもデータを見られる。
    • 気づき: 数字の羅列では気づかない「あ、夏だけ売上が落ちてる」といった傾向が一瞬でわかる。

3. 実は「協力関係」にある（ここが重要！）

実は、この2つは「どっちを使うか？」と喧嘩するものではなく、「組み合わせて使う」のがAWSの黄金パターンです。
なぜなら、QuickSight自体は「S3にある生のCSVファイル」を直接読み解くのが少し苦手だからです。

そこで、「AthenaがS3からデータを整理して取り出し、それをQuickSightに渡してグラフにする」という連携プレイをよく行います。

まとめ：使い分けの判断基準

観点 Amazon Athena Amazon QuickSight
主な利用者 エンジニア、データアナリスト 経営者、マーケター、営業
見る形式 テキスト、表 (Table) グラフ、チャート (Chart)
操作方法 SQLコードを書く マウスでドラッグ＆ドロップ
目的 「あのエラーログを見つけたい」
「正確な数値リストが欲しい」		 「売上の傾向を知りたい」
「異常を一目で発見したい」
  • 「具体的なデータそのもの（文字）」を見たいなら Athena。
  • 「データの意味や傾向（絵）」を見たいなら QuickSight。
