Amazon Detectiveの役割と機能

#セキュリティ #Amazon Detective #調査 #ログ分析 #AWS

Amazon Detectiveは、セキュリティ事故の調査を自動で行う「名探偵」の役割を担います。

Amazon Detective（ディテクティブ）は、その名の通り「名探偵」です。
セキュリティ上の問題が起きた時に、「何が起きたのか？」「犯人はどこから来たのか？」「被害はどこまで広がったのか？」という原因究明（調査）を、グラフや図を使って助けてくれるサービスです。

一言で言うと、「大量のログを自動で分析して、犯行現場の相関図を作ってくれる『科学捜査班』」です。

1. なぜこれが必要なの？（GuardDutyとの違い）

よく Amazon GuardDuty（ガードデューティ） と混同されますが、役割が完全に違います。
「警備員が通報し、刑事が現場検証する」という流れで覚えるとスムーズです。

セキュリティ対応の連携プレー
🚨 GuardDuty 警備員 / センサー

「あ！泥棒が入ったぞ！」

異常を検知してアラームを鳴らすのが仕事。
(詳しい被害状況まではわからない)

➡
🕵️‍♂️ Detective 刑事 / 科学捜査

「よし、証拠を集めよう」

アラーム後に現場へ急行。
ログを集めて犯行ルートを特定する。
(根本原因をグラフで可視化)

2. Detective のすごいところ： 「自動で相関図作成」

通常、セキュリティ事故の調査は、何千行ものログ（VPCフローログやCloudTrail）を目で見て、「このIPアドレスがこの時間にここに繋いで…」と手作業でパズルを組み立てる必要があり、数日かかります。

Detectiveを有効にしておくと、これらのログを勝手に吸い上げて、機械学習で分析し、以下のような情報を「グラフ（絵）」にしてくれます。

  • 「このインスタンスは、普段このIPとしか通信しないのに、今日は急に知らない国と大量に通信している（異常な振る舞い）」
  • 「このマルウェアは、AサーバーからBサーバーへ感染が広がっている」

3. AWSセキュリティサービスの「4兄弟」

試験では、以下の4つの使い分けが頻出です。混乱しやすいので整理しましょう。

セキュリティ4兄弟の役割分担
👮 GuardDuty 警備員 (検知)
脅威検知、機械学習、自動。
「何か変だ！」と気づく。
👷 Inspector 点検業者 (診断)
脆弱性診断、CVE。
「この鍵、壊れそうですよ」と事前に直す。
🏢 Security Hub 管理センター (集約)
一元管理、スコア化。
全情報をまとめて報告する。
🕵️ Detective 名探偵 (調査)
根本原因調査、可視化。
「犯人はお前だ！」と突き止める。

4. 試験対策キーワード

試験で以下の文脈が出たら、Amazon Detective が正解です。

  • 「セキュリティインシデントの『根本原因』を特定したい」
  • 「調査プロセスを効率化・可視化したい」
  • 「VPCフローログやCloudTrailを分析して、不審な振る舞いの『グラフ』を見たい」

まとめ

  • Amazon Detective: セキュリティ事故の「調査・原因究明」をするツール。
  • イメージ: 散らばった証拠（ログ）をまとめて、犯人の行動を「グラフ（相関図）」にしてくれる名探偵。

「GuardDutyで見つけて、Detectiveで調べる」とセットで覚えておけば完璧です！
