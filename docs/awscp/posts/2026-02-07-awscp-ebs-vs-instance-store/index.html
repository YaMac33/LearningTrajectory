EBSとインスタンスストアの違い

カテゴリ: awscp 2026-02-07 15:00

  • #インスタンスストア
  • #クラウドストレージ
  • #AWS
  • #データ保存

1行要約

EBSは「消えにくい外付けディスク」、インスタンスストアは「電源を切ると消える一時ディスク」。

やさしい解説（まずは感覚）

EC2 には 2種類の保存場所があります。

  • EBS: USB外付けHDD・SSDのイメージ
  • インスタンスストア: ノートPC内蔵の作業用メモ帳のイメージ

図でイメージ

【EBS】
   EC2インスタンス
        │
        │（ネットワーク越し）
        ▼
   EBSボリューム（別サービス）
        ↑
   インスタンス停止しても残る
      
【インスタンスストア】
   EC2インスタンス
        │
        ▼
   ローカルディスク
   （インスタンスと運命共同体）

それぞれを詳しく

EBS（Elastic Block Store）

特徴

  • EC2とは別サービス
  • インスタンスを停止・再起動してもデータは残る
  • スナップショットをS3に保存できる
  • SSD / HDD など種類がある

実務イメージ

  • WebサーバのOSディスク
  • アプリケーションデータ
  • データベースの保存領域

👉 「消えたら困るもの」は基本EBS

インスタンスストア

特徴

  • EC2ホストに物理的に直結
  • 停止・終了するとデータは消える
  • 高速だが永続性なし
  • バックアップ不可

実務イメージ

  • 一時ファイル
  • キャッシュ
  • 一時的な計算結果

👉 「消えてもいい作業場」

超重要：試験で問われやすいポイント（ここ出る）

  • ✅ インスタンス停止時のデータ
    • EBS → 残る
    • インスタンスストア → 消える
  • ✅ バックアップ
    • EBS → スナップショット可能
    • インスタンスストア → 不可
  • ✅ 用途
    • 永続データ → EBS
    • 一時データ → インスタンスストア

ひっかけ・間違えやすい点 ⚠️

  • ❌「EC2に付いてるから、どっちも同じ」→ 違います
  • ❌「高速＝EBSじゃない」→ 高速なのはインスタンスストア
  • ❌「再起動＝消える」→ 再起動では消えない。消えるのは 停止 or 終了

覚え方（試験直前用）

  • 🧠 EBS = External（外付け）
  • 🧠 インスタンスストア = 一時メモ帳
