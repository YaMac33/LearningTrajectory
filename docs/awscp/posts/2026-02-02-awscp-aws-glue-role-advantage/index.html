AWS Glueの役割と利点

  • AWS
  • Glue
  • ETL
  • データカタログ
  • サーバーレス

AWS Glue（グルー）は、一言でいうと「データの引越し・お掃除・台帳作成を自動化する、サーバーレスの便利屋さん」です。データ分析を行うための「下準備（ETL）」を専門に行うサービスであり、AthenaやRedshiftが活躍するための土台を作ります。

1. AWS Glueの3つの主な役割

試験では、Glueが「具体的に何をするサービスか」を問われます。以下の3つの機能をセットで覚えましょう。

🕷️ ① クローラー
(Crawler) 自動調査員
バラバラに保存されているデータ（S3など）を自動で見に行き、「どんな形式か」「どんな項目があるか」を調べ上げます。
📖 ② データカタログ
(Data Catalog) 共通の台帳
クローラーが調べた情報を保存する「共通の索引」。これがあるおかげで、AthenaやRedshiftはデータの場所や構造を即座に知ることができます。
⚙️ ③ ETLジョブ
(ETL Jobs) 加工・引越し
データを別の場所へ移したり、形を変えたりする実際の「作業」です。プログラム（スクリプト）を自動生成して実行します。

例え話：図書館の整理

  • クローラー： 新しく届いた大量の段ボール箱（S3）を勝手に開けて、中身をチェックするロボット。
  • データカタログ： 「どの箱に何の本が入っているか」が書かれた検索用パソコン（台帳）。
  • ETLジョブ： 本の汚れを拭き取り、分類ラベルを貼って、本棚（Redshift）に並べる作業員。

2. なぜAWS Glueがすごいの？

従来のETL作業は、専用のサーバーを立てて、複雑なプログラムを一から書く必要がありました。Glueはここを劇的に変えました。

  • サーバーレス： サーバーの管理は一切不要。使った分だけ支払う（従量課金）。
  • 自動コード生成： 「ここからここへデータを移したい」とポチポチ選ぶだけで、Pythonなどのコードを自動で作ってくれます。
  • Glue DataBrew： プログラムが全く書けない人でも、Excelのような画面で直感的に「データのお掃除（クリーニング）」ができる機能もあります。

3. 試験で問われやすいポイント

CLF-C02などの試験で出るポイントです。

機能・特徴 試験のポイント
データカタログ AthenaやRedshift Spectrumがデータを読み込むために参照する「メタデータのリポジトリ」。
サーバーレス 「ETLのためにEC2を立てる必要があるか？」→ いいえ、Glueは完全マネージドです。
クローラー スキーマ（データの構造）を自動的に推論・作成してくれる機能。

5. まとめ

  • Glue = サーバーレスETLサービス。
  • データカタログ = データの住所録（メタデータ管理）。
  • クローラー = データ構造の自動調査員。
  • DataBrew = ノーコードでデータ加工。

Glueは、派手な分析ツール（QuickSight）や倉庫（Redshift）を裏で支える、「データ分析の前準備（下ごしらえ）を全部やってくれる執事」とイメージしてください。
