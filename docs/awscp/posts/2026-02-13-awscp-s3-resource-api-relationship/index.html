S3のリソースとAPIの関係

#S3 #API #バケット #オブジェクト #AWS

S3のバケットとオブジェクトはリソースで、APIにより操作される仕組みを解説します。

Q. 「開発・実装面では、バケットとオブジェクトはリソースであり、S3はバケットやオブジェクトを管理するためのAPIを提供します。」とはどういう意味？
A. これは「操作されるモノ（リソース）」と「操作する命令（API）」の関係を表しています。

S3を理解する上で、この「主従関係」を整理すると一気に視界がクリアになります。

1. 構造図：バケットとオブジェクトの関係

まず、「モノ」の関係性を整理しましょう。
S3の世界には、大きく分けて「バケット（箱）」と「オブジェクト（中身）」の2種類のリソースしかありません。

S3のリソース階層
Amazon S3 サービス
バケット A
バケット B
Obj-1.jpg
Obj-2.txt
  • バケット: データを入れるための「箱」。リージョン単位で作られ、名前は世界中で一意である必要があります。
  • オブジェクト: 実際のデータ（画像、動画、ログファイルなど）。キー（ファイル名）を持ちます。

AWSでは、これら「APIで操作できる対象物」のことを総称して「リソース」と呼びます。

2. APIとの関係：どうやって操作する？

私たちがS3を使うとき、実は直接ハードディスクを触っているわけではありません。必ず「S3 API」という窓口を通して命令しています。

具体的な命令（API）の例

  • PUT Bucket: 「新しいバケットを作ってください」という命令。
  • PUT Object: 「このデータを、あのバケットに入れてください」という命令。
  • GET Object: 「あのバケットの、このデータを取り出してください」という命令。

普段使っている「AWSマネジメントコンソール（Web画面）」も、裏側ではポチポチするたびに、このAPIを呼び出しているだけなのです。

3. わかりやすいメタファー（倉庫管理）

現実世界に置き換えると、よりイメージしやすくなります。

S3の役割分担
🏭 Amazon S3 巨大な倉庫
🗃️ バケット 整理棚
📦 オブジェクト 荷物
🤖 S3 API 倉庫番ロボット

あなたは倉庫（S3）に入って直接荷物を置くことはできません。必ず入り口にいる倉庫番ロボット（API）に、「この荷物（オブジェクト）を、あの棚（バケット）に置いて！」と指示を出します。

4. RESTful設計によるリソース識別

S3は「RESTful」という設計思想で作られています。これは「すべてのリソースにURL（住所）がある」という意味です。

https://bucket-name.s3.amazonaws.com/object-key

  • このURLを見れば、どのバケットのどのオブジェクトか一発でわかります。
  • だからこそ、これらを「識別可能なリソース」と呼ぶのです。

まとめ

試験対策として覚えるべきポイントは以下の通りです。

  • バケットとオブジェクトは「リソース」（操作される対象）。
  • S3はそれらを操作するための「APIを提供するサービス」。
  • コンソール操作もプログラムも、結局は「API」を叩いている。
