サブネットとサブネットマスクの違い

#ネットワーク #サブネット #サブネットマスク #IPアドレス #AWS

サブネットはネットワークの場所、サブネットマスクはその境界を定義する数値です。

「サブネット」と「サブネットマスク」は、名前が似ていますが、指しているモノが全く違います。
一言で言うと、「サブネットは『場所（土地）』、サブネットマスクは『定規（境界線）』」です。
先ほどの「家」の例えを使って、決定的な違いを解説します。

似ているようで別物
🏞️ Subnet
「場所・区画」
ネットワークそのもの。
サーバーが住む住所の範囲。
📏 Subnet Mask
「定規・境界線」
範囲を決めるための数値。
どこまでが住所かを示すルール。

1. サブネット（Subnet）

意味： 小分けにされたネットワークそのもの（場所）

  • 役割: 「ここは営業部のエリア」「ここは開発部のエリア」という「空間」や「グループ」のことです。
  • 家で例えると: 家の中にある「子供部屋」や「寝室」という部屋そのものを指します。
  • 実態: VPC（大きな土地）を切り分けた、一つ一つの区画です。

2. サブネットマスク（Subnet Mask）

意味： どこまでがそのネットワークかを定義する数値（定規）

  • 役割: 「この部屋の広さはここからここまでですよ」「この住所のここまでがエリア名ですよ」という「範囲を決めるためのルール（数値）」です。
  • 家で例えると: 部屋の広さを測る「メジャー（巻尺）」や、部屋を仕切る「壁の位置」のことです。
  • 実態: IPアドレスの「どこまでがグループ名（ネットワーク部）で、どこからが個人の番号（ホスト部）か」をコンピュータに教えるための数字です。

3. 具体例で見ると一発でわかります

例えば、AWSでよく見る 10.0.1.0/24 という表記があります。これを分解してみましょう。

10.0.1.0/24 の中身
青色：共通部分（マスク） / 灰色：自由部分
Network Part (/24) 10 . 0 . 1
Host Part . 0 ~ 255
「サブネット」は、この範囲全体（10.0.1.0 〜 10.0.1.255）のIPの集まり。
「サブネットマスク」は、この仕切り線（/24）の位置を決めるルール。
  • サブネット:
    • 10.0.1.0 から 10.0.1.255 までの「IPアドレスの集まり」のこと。
    • これが「部屋（場所）」です。インスタンスはこの中に住みます。
  • サブネットマスク:
    • /24 （または 255.255.255.0）のこと。
    • これが「定規（ルール）」です。
    • 「先頭の24ビット（10.0.1まで）が部屋の名前だよ。残りの数字（最後の0〜255）が部屋の中の席番号だよ」と定義しています。

4. なぜ「マスク（仮面）」と呼ぶの？

「マスク（Mask）」には「覆い隠す」という意味があります。

32桁の数字（IPアドレス）の上から覆いかぶせて、「ここまではグループ共通の番号だから、変更しちゃダメだよ！（隠す）」とし、「残りの部分は自由に使っていいよ（見せる）」とする機能だからです。

  • マスクが大きい（数字が大きい） → 変更していい部分が減る → 部屋が狭くなる（使えるIPが減る）。
  • マスクが小さい（数字が小さい） → 変更していい部分が増える → 部屋が広くなる（使えるIPが増える）。

まとめ

用語 イメージ AWSでの作業
サブネット 「土地」
インスタンスを置く場所		 EC2起動時に「どこに置くか」として選択する
サブネットマスク 「境界線・定規」
広さを決める数値		 サブネット作成時に「CIDRブロック」として入力する

AWSで「サブネットを作成する」という作業は、「サブネットマスク（CIDRブロック）を指定して、土地の広さを決めること」とイコールです。
「部屋を作る（サブネット）」ために「壁の位置を決める（マスク）」必要がある、という関係性ですね！
