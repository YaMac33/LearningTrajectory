AWSの責任共有モデル解説

  • AWS
  • 資格試験
  • 責任共有モデル
  • クラウド
  • セキュリティ

AWS資格試験で最も失点してはいけない（そして実務でも最も大事な）概念、「責任共有モデル」について解説します。これまでEC2やIAMなど個別の技術を見てきましたが、ここで「AWSとあなた（ユーザー）の役割分担」をはっきりさせます。

一言で言うと、「賃貸マンションの管理」と同じです。

  • 大家さん（AWS）： 建物の耐震構造、オートロック、エレベーターの点検までは責任を持ちます。
  • 入居者（あなた）： 自分の部屋の鍵かけ、部屋の中の火の始末、招待する客の管理は、あなたの責任です。

1. 鉄の掟：この2つを暗記してください

試験では、以下のフレーズがそのまま出ます。「Of」と「In」の違いを意識してください。

👤 ユーザーの責任 (入居者) データ、設定、OSパッチ、暗号化
Security IN the Cloud
🛡️ AWSの責任 (大家さん) ハードウェア、物理施設、ネットワーク、ハイパーバイザー
Security OF the Cloud

2. 具体的な線引き（試験に出る分類）

どこまでがAWSで、どこからが自分か。この境界線を問う問題が必ず出ます。

🛡️ AWSの責任範囲（大家さん）

ユーザーが物理的に触れない部分は、すべてAWSの責任です。

  • 物理セキュリティ： データセンターへの侵入防止、監視カメラ、警備員。
  • ハードウェア： サーバー、ストレージ機器の故障対応、廃棄処理（HDDの物理破壊など）。
  • グローバルインフラ： リージョン、AZ、エッジロケーションの維持管理。
  • ハイパーバイザー： 物理サーバー上で仮想マシン（EC2）を動かすための土台ソフト。

👤 ユーザーの責任範囲（入居者）

AWSから渡された「後のこと」は、すべてあなたの責任です。

  • 顧客データ： データの暗号化、バックアップ。（AWSは中身を勝手に見ません）
  • プラットフォーム等の設定： OSのパッチ適用（Windows Updateなど）、ファイアウォール（セキュリティグループ）の設定。
  • アクセス管理： IAMユーザーの作成、パスワードの強度設定、MFAの有効化。

3. 【超重要】サービスによって責任範囲が「動く」

ここが試験のひっかけポイントです。「EC2」を使うか、「RDS」を使うか、「S3」を使うかで、ユーザーがやるべき作業（責任）の量が変わります。

パターンの違い

  • EC2 (IaaS)： OSの更新（パッチ当て）は「ユーザーの責任」です。自由度は高いですが管理が大変です。
  • RDS (マネージド)： OSの更新は「AWSの責任」になります。面倒な作業をAWSに押し付けられます。
  • S3 (抽象化サービス)： サーバーのOSなんて意識しません。ユーザーは「データの管理と暗号化」だけに集中します。

4. 試験でよく出る「どっちの責任？」クイズ

これらが即答できれば合格レベルです。

Q1. 故障した物理ディスクの交換は？
AWSの責任
Q2. EC2上のゲストOSのパッチ適用は？
ユーザーの責任
Q3. S3に保存したデータの暗号化設定は？
ユーザーの責任
Q4. データセンターの入館管理は？
AWSの責任
Q5. IAMパスワードのポリシー設定は？
ユーザーの責任

5. まとめ

「ハードウェアや建物、インフラ自体が壊れたらAWSの責任」、「設定ミス、パスワード流出、OSの更新忘れでハッキングされたらユーザーの責任」と覚えましょう。このモデルを理解していれば、AWSが提供するセキュリティツールが「誰のために」「何を守るために」あるのかが明確になります。
