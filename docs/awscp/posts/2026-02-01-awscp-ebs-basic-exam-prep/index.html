EBSの基本と試験対策

  • EBS
  • EC2
  • ストレージ
  • 試験対策
  • AWS

Amazon EBS（Elastic Block Store）は、EC2（サーバー）にとっての「外付けハードディスク（またはSSD）」です。EC2を「パソコン本体」とするなら、EBSはそこにケーブルで繋がっている「Cドライブ」や「Dドライブ」にあたります。非常に地味ですが、これがなくてはEC2は動きません（OSをインストールする場所だからです）。試験では、「S3との違い」と「インスタンスストアとの違い」が必ず問われます。

1. EBSのイメージ：ネットワーク越しのドライブ

EBSは、EC2の筐体の中に物理的に入っているわけではなく、ネットワークを経由して接続されています。

EC2 インスタンス CPU / メモリ
EBS ボリューム OS / データ
  • 永続性： EC2を「停止」しても、EBSの中のデータは消えません。
  • 付け替え： EC2から取り外して、別のEC2に付け替えることができます（USBメモリのような感覚）。

2. 試験に出る「EBSの鉄の掟」3選

ここを理解していないと、設計問題で間違えます。

① AZ（アベイラビリティーゾーン）の壁

EBSは、作成したAZから出られません。「東京リージョンのAZ-a」で作ったEBSは、「AZ-a」にあるEC2にしか接続できません。

AZ-a
EBS
❌
AZ-c
EC2

※ 移動させるには「スナップショット（バックアップ）」を経由する必要がある。

② 1対1の関係（基本）

  • ルール： 基本的に、「1つのEBSは、1台のEC2にしか接続できない」。
  • 例外： 「マルチアタッチ」という機能を使えば複数台に接続できますが、試験の基本問題では「EBSは1台のEC2専有」と考えてOKです。
  • ※逆に、1台のEC2に複数のEBS（Cドライブ、Dドライブ…）を付けることは可能です。

③ 削除時の挙動（Delete on Termination）

EC2インスタンスを「削除（Terminate）」した時、くっついているEBSをどうするか設定できます。

  • ルートボリューム（Cドライブ）： デフォルトでは「一緒に削除される」設定になっています。
  • データボリューム（Dドライブ）： デフォルトでは「残る」設定になっています。
  • 試験対策： 「EC2を削除してもデータを残したい」と言われたら、「終了時の削除（Delete on Termination）」設定をオフにします。

3. ライバルとの比較：インスタンスストア & S3

ここが最大の山場です。データ保存場所の違いを整理しましょう。

VS インスタンスストア (Instance Store)

インスタンスストアは、EC2が動いている物理サーバーに内蔵されているディスクです。

特徴 EBS (外付け) インスタンスストア (内蔵)
データの寿命 永続的 (EC2を停止しても残る) 一時的 (EC2を停止すると消える)
速度 普通 (ネットワーク経由) 爆速 (物理直結)
用途 OS、データベース、重要データ 一時ファイル、キャッシュ
試験の合言葉 「永続ストレージ」 「エフェメラル (儚い) ストレージ」

注意： 試験で「EC2を停止・再起動したらデータが消えた。なぜか？」と聞かれたら、答えは「インスタンスストアを使っていたから」です。

VS Amazon S3 (オブジェクトストレージ)

特徴 EBS (ブロックストレージ) S3 (オブジェクトストレージ)
データの扱い ブロック単位
(ファイルの「一部」だけ書き換え可)		 オブジェクト単位
(ファイル丸ごと置き換え)
頻繁な書き換え 得意 (OSやDBに向く) 苦手 (毎回全アップロードが必要)
アクセス方法 EC2にマウントして使う (Cドライブ) インターネット経由 (HTTP/HTTPS)
コスト 高め (確保した容量分払う) 安い (使った容量分払う)

4. バックアップ：スナップショット

EBSのデータをバックアップするには、「スナップショット」を取ります。

  • 保存先： 実は裏側で S3 に保存されています（安くて頑丈だから）。ユーザーからは見えません。
  • 増分バックアップ： 2回目以降は「変更があった部分だけ」を保存するので、容量とお金を節約できます。

5. EBSボリュームの種類（松竹梅）

EBSには性能によっていくつかタイプがあります。

  • 汎用SSD (gp2 / gp3)： バランス型。迷ったらこれ（デフォルト）。Webサーバーなど。
  • プロビジョンドIOPS SSD (io1 / io2)： 超高性能。高い。IOPS（速度）を指定して買う。大規模DBなど。
  • HDD (st1 / sc1)： 安いが、ブート（OS起動）には使えない。ログ分析など。

6. まとめ

  • EBS = EC2の外付けハードディスク。
  • 永続性 = EC2を止めてもデータは消えない（インスタンスストアは消える）。
  • AZロック = 作ったAZから移動できない（移動にはスナップショットが必要）。
  • ブロックストレージ = OSやDBのように、頻繁に中身を書き換える用途に使う。

これで、ストレージ周りの知識（S3とEBSの違い）は完璧です。
