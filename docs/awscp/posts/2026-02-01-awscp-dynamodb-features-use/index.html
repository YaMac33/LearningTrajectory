DynamoDBの特長と使い方

  • DynamoDB
  • NoSQL
  • データベース
  • AWS
  • スキーマレス

Amazon DynamoDB（ダイナモディービー）は、RDSと並んで試験に必ず出るデータベースですが、その性質は真逆です。一言で言うと、「超高速・容量無制限の『NoSQL』データベース」です。RDSが「Excel（表）」なら、DynamoDBは「JSONの詰め合わせ」のようなイメージです。

1. RDSとの決定的な違い（NoSQLとは？）

RDS（リレーショナルデータベース）は、「列」と「行」がきっちり決まった表です。一方、DynamoDB（NoSQL）は、「キー（名札）」と「バリュー（値）」さえあれば、中身は何でもありです。

RDS (SQL)

事前に枠（スキーマ）の定義が必要。
全員が同じ項目を持つ。

IDNameAgeAddress
1Tanaka25Tokyo
2Sato30Osaka
DynamoDB (NoSQL)

IDさえあれば中身は自由（スキーマレス）。
人によって項目がバラバラでOK。

{
  "ID": "001",
  "Name": "Tanaka",
  "Age": 25,
  "Hobby": "Soccer"
}
{
  "ID": "002",
  "Name": "Sato",
  "Address": "Osaka",
  "IsPremium": true
}

2. 試験に出る「3つの最強ポイント」

DynamoDBが選ばれる理由は、RDSにはできない以下の特徴があるからです。

① サーバーレス（完全マネージド）

  • RDS： サーバーのサイズ（t3.microなど）を選んだり、容量を気にする必要があります。
  • DynamoDB： サーバーが存在しません。勝手に容量が増え、勝手に性能が上がります。あなたはテーブルを作るだけです。

② 桁違いのパフォーマンス

  • RDS： データが増えると検索が遅くなりがちです。
  • DynamoDB： データが100件でも1億件でも、「ミリ秒単位（一桁ミリ秒）」で応答します。
  • キーワード： 「一桁ミリ秒のレイテンシー（遅延）」という言葉が出たらDynamoDBです。

③ スケーラビリティ（拡張性）

  • アクセスが急増しても、自動で性能を拡張（オートスケーリング）します。
  • 用途： ゲームのハイスコアランキング、SNSの「いいね」機能、Amazonのショッピングカート（アクセスが爆発する系）。

3. DynamoDB専用の「加速装置」: DAX

試験では「もっと速くしたい」という要求が出ます。DynamoDB自体も速い（ミリ秒）ですが、Amazon DynamoDB Accelerator (DAX) という専用のキャッシュ機能を使うと、更に高速になります。

DynamoDB + DAX = マイクロ秒

（ミリ秒の1000分の1の速さ）

  • 試験対策： 「DynamoDBの読み取り負荷を下げたい」「ミリ秒をマイクロ秒にしたい」とき選ぶ。

4. グローバルテーブル（世界展開）

ボタン一つで、世界中のリージョンにデータを自動複製（レプリケーション）できます。日本のユーザーは東京のDynamoDBへ、アメリカのユーザーは米国のDynamoDBへアクセスさせることで、世界中どこでも爆速で表示できます。

6. まとめ

DynamoDBは、「細かいことはいいから、とにかく速く、大量の単純データをさばきたい！」という時に使う、AWSならではのクラウドネイティブなデータベースです。これで、データベース系（RDS, Aurora, DynamoDB, Redshift）の違いは完璧です。
