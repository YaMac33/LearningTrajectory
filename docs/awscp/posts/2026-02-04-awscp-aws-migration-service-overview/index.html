AWS Application Migration Serviceの概要

#AWS #MGN #サーバー移行 #リフト＆シフト #クラウド

AWS MGNはサーバーをそのままAWSに移行するサービス。簡単かつ迅速な移行を実現します。

AWS Application Migration Service (MGN) は、オンプレミスや他のクラウドにあるサーバーを、「中身（OS・アプリ・データ）を丸ごとそのまま」AWS（EC2）にコピーして引っ越しさせるサービスです。
一言で言うと、「サーバーの『丸ごとコピー機』」です。
以前は「Server Migration Service (SMS)」や「CloudEndure」と呼ばれていたサービスの最新・最強版です。現在、AWSへのサーバー移行といったら、基本的にはこれを使います。

1. なぜこれが必要なの？（「リフト＆シフト」の実現）

サーバーの引っ越しには、大きく2つのやり方があります。

  • 作り直し: AWS上で新品のEC2を立ち上げ、OSをインストールし、アプリを入れ直し、データを移す。（手間がかかるし、設定ミスのリスクがある）
  • そのまま移行 (Lift and Shift): 今動いているサーバーのディスクの中身を、そのままAWSにコピーする。

MGNは、この「2. そのまま移行（リフト＆シフト）」を自動で行うためのツールです。OSの設定も、インストールされているアプリも、デスクトップの壁紙も、そのままEC2になります。

2. 仕組み： 「裏でこっそりコピーし続ける」

MGNのすごいところは、「稼働中のサーバーを止めずにコピーできる」点です。

移行のステップ
1
エージェント導入
引っ越したいサーバー（ソースサーバー）に、専用のソフト（エージェント）を入れます。
2
継続的レプリケーション
サーバーが動いている裏で、データの変更をリアルタイムでAWS側の「ステージングエリア（準備室）」にコピーし続けます。システムを止める必要はありません。
3
カットオーバー（本番切り替え）
準備ができたら、ボタンを押します。すると、数分でAWS上に本番用のEC2が立ち上がります。

3. 他のサービスとの違い（試験の引っかけ）

名前が似ているサービスや、役割が近いサービスとの区別が重要です。

サービス名 役割 対象
Application Migration Service (MGN) 「サーバー丸ごと」引っ越し OS、アプリ、データ全部 (Windows/Linux)
Database Migration Service (DMS) 「データベースだけ」引っ越し DBの中身 (Oracle, SQL Serverなど)
Application Discovery Service 引っ越し前の「調査」 移行はしない。「サーバーが何台あるか」を調べるだけ
  • 「サーバーを移行したい」 → MGN
  • 「DBを移行したい」 → DMS
  • 「移行計画のために現状を把握したい」 → Discovery Service

4. 試験対策キーワード

試験で以下の文脈が出たら、Application Migration Service (MGN) が正解です。

  • 「オンプレミスのサーバーを、AWSへ『リホスト（Rehost）』したい」
    • ※リホスト ＝ そのまま移行すること。
  • 「物理サーバー、VMware、Azure上のVMなどを、最小限のダウンタイムでEC2に移行したい」
  • 「Lift and Shift（リフト＆シフト）戦略を取りたい」

まとめ

  • MGN: サーバーを「OSごと丸っ切り」AWSにコピーするサービス。
  • メリット: アプリの書き換え不要。移行作業中もサーバーを止めなくていい。
  • キーワード: 「リホスト (Rehost)」「リフト＆シフト」。

「引っ越し荷物をダンボールに詰め直すのではなく、家ごとトラックに積んで移動させるサービス」というイメージです！
